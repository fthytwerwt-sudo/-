# 执行注意事项：短视频自动流的最简单流程

## 1. 素材使用边界

- `已确认` 火山引擎素材未打码不得入片。
  - 原因：素材含手机号、短信验证码、API Key 管理页和资源 ID 痕迹。
  - 处理：如果口播提 API，只用信息卡，不使用该原画面。
- `已确认` `创建文件夹.mp4` 只能作为弱 B-roll，不作为主线素材。
- `已确认` `录屏2026-04-30 03.25.28.mov` 是历史长录屏，不默认进入本条视频。
- `已确认` Codex 素材入片时需裁切或遮挡右侧分支详情、底部路径、文件名、巨大 diff 数字和本地任务信息。
- `已确认` Trae 素材入片时需避开不必要本地路径、文件夹名和项目列表。

## 2. 状态边界

- `已确认` 豆包输出方案不等于工程跑通。
- `已确认` 豆包输出 `Trae SOLO` prompt，只能证明 prompt 目标是让 Trae 搭架构，不能证明系统已运行。
- `已确认` Trae 生成项目骨架不等于代码运行成功。
- `已确认` `settings.py` / `base_module.py` 可见，只能证明文件生成或正在生成，不能写成完整项目验收通过。
- `待验证` Trae 代码运行状态；画面出现过环境 / 生成风险，不得写满。
- `待验证` 阿里云剪辑只能写成云端总装方向 / 技术验证候选，不得写成正式稳定链路。
- `已确认` Codex 检查不等于内容过线。
- `已确认` 技术验证跑通不等于内容可发。

## 3. 剪辑结构边界

- `已确认` 这条不是教程，不要剪成手把手安装课。
- `已确认` 不做正反面展示结构。
- `已确认` 不做即梦评测，不攻击即梦，只做定位对比。
- `已确认` 不做 Codex 教程、Trae 教程、阿里云剪辑教程。
- `已确认` 不要把画面切成“工具介绍合集”；主线是“短视频自动流是可复用流程，不是一键生成”。

## 4. 文案与画面对齐

- 豆包段必须保留“一句简单需求”这一点，不要一上来剪复杂方案。
- 豆包 prompt 段必须保留“可复制到 Trae SOLO”的证据，但要在口播里说明这不等于系统已跑通。
- Trae 段必须保留 `Updating Tasks...`、`11 待办`、`vlog_automation_workflow` 项目骨架。
- API 段不要使用火山引擎原画面，使用信息卡解释。
- 阿里云剪辑段使用信息卡说明“装配台”角色，不要放成稳定链路证明。
- Codex 段只证明执行检查、路径确认、命令、文件和报告，不证明内容过线。

## 5. 禁用表达

- 禁用：`今天教你`
- 禁用：`手把手`
- 禁用：`保姆级教程`
- 禁用：`底层逻辑`
- 禁用：`抓手`
- 禁用：`赋能`
- 禁用：`闭环`
- 禁用：`破局`
- 禁用：`正式稳定链路`
- 禁用：`已经全自动跑通`

## 6. 执行前复审

- `待验证` 后续剪辑执行前必须先由 ChatGPT 复审本执行包。
- `待验证` 如果 ChatGPT 调整文案节奏，Codex 再按复审后的脚本执行剪辑，不得直接把本包当成已可剪最终片。
