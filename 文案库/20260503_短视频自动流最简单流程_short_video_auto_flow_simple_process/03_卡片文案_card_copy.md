# 卡片文案：短视频自动流的最简单流程

## 1. 卡片使用原则

- 一屏最多 2-3 个信息模块。
- 只做流程解释和状态边界，不做教程步骤说明。
- 不强卖课，不写“手把手”“保姆级”“今天教你”。
- 不把待验证事项写成已稳定。

## 2. 流程总览卡

### 主标题

短视频自动流，不是一键生成

### 信息模块

1. 一句需求
2. 拆成流程
3. 再交给工具执行

### 小字

先把顺序理出来，自动化才有地方落脚。

## 3. 豆包 -> Trae prompt 卡

### 主标题

豆包不是直接做视频

### 信息模块

1. 把一句需求拆成方案
2. 生成 Trae SOLO 能接住的 prompt
3. 再交给 Trae plan

### 小字

关键不是继续聊天，而是把想法翻译成任务说明。

## 4. API 解释卡

### 主标题

API 先别想复杂

### 信息模块

1. 文字生成可以接
2. 配音 / 图片 / 卡片可以接
3. 剪辑总装也可以接

### 小字

它的作用是把工具变成系统能调用的能力。

### 边界提示

`待验证` 本条不能写成 API 已全部接通。

## 5. 阿里云剪辑总装卡

### 主标题

阿里云剪辑不是总控脑

### 信息模块

1. 不负责想创意
2. 不负责写文案
3. 更像最后的装配台

### 小字

录屏、卡片、音轨放进时间线，最后导出视频。

### 边界提示

`待验证` 只能写成云端总装方向 / 技术候选，不得写成正式稳定链路。

## 6. Codex / Claude / Trae 执行层卡

### 主标题

流程在，工具可以换

### 信息模块

1. Trae：生成项目骨架
2. Codex：执行检查和结果回报
3. Claude Code：可补代码和工作流执行

### 小字

重点不是某个软件最强，而是每一步都有人负责、有结果能查。

### 边界提示

`已确认` Codex 检查不等于内容过线。

## 7. 即梦 vs 自动流对比卡

### 主标题

即梦像抽素材，自动流像搭流程

### 信息模块

1. 即梦：适合快速出单个画面
2. 自动流：适合把需求拆成可复用生产线

### 小字

不是说即梦没用，是它和自动流解决的问题不一样。

## 8. 最后总结卡

### 主标题

别一上来追求一键生成

### 信息模块

1. 先拆生产流程
2. 再接工具能力
3. 最后检查结果边界

### 收束句

顺序对了，自动化才有地方落脚。
