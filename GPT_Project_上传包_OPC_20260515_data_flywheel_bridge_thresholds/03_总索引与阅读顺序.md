本文件是当前 `10 份基础执行包 + OPC 总纲 + 状态动作总控器 + 参考到执行落地契约 + 目标驱动数据飞轮与文案执行闭环` 的总索引。

当前根目录目标非常明确：
- 保留 10 份高频基础执行包文件
- 新增 1 份 OPC 上位机制文件
- 新增 1 份最高状态动作总控机制文件
- 新增 1 份参考到执行落地契约机制文件
- 新增 1 份目标驱动数据飞轮与文案执行闭环机制文件
- 让 GPT 与 Codex 在 `10 份基础执行包 + OPC 总纲 + 状态动作总控器 + 参考到执行落地契约 + 目标驱动数据飞轮与文案执行闭环` 内完成 80% 路由判断、reference 执行保真和数据驱动文案前置判断
- 不再默认深读旧 10-37 散文件

### 2.3 1 份最高状态动作总控机制文件
12. `11_项目状态动作总控器_机制推理层.md`

`11_项目状态动作总控器_机制推理层.md` 是 GPT Project / ChatGPT 侧最高机制入口。每轮先用它判断当前项目状态、事实源裁决、触发机制、下一动作、完成标准和反馈更新，再进入文案、剪辑、复盘、供料或执行。

`11_项目状态动作总控器_机制推理层.md` 也承载当前三大机制推理函数：

- `editing_inference_function（剪辑推理函数）`
- `content_route_inference_function（内容路由推理函数）`
- `quality_issue_classifier（质量短板分类器）`

这三个函数统一使用 `input_signal -> observed_evidence -> state_inference -> action_policy -> validation_rule -> blocked_if -> feedback_update`，分别约束 `editing_decision_pack（剪辑决策包）`、`content_route_card（内容路由卡）` 和 `quality_lock_card（质量锁卡）`，不推进动态状态。

`content_route_card V2（内容路由卡 V2）` 的标准输出模板写在 `05_文案路由规则.md`，用于每条真实视频执行前统一判断 `opening_route_decision（开头路由判断）`、`middle_carrier_decision（中段承载判断）`、`focusee_middle_editing_decision（FocuSee 中段剪辑判断）`、`card_placement_decision（卡片位置判断）`、`core_evidence（核心证据）`、API 生成人物、少量 PPT、Prompt 尾卡和 `blocked_if（阻断条件）`。

`script_anchor_extraction_function（文案锚点提取函数）` 的标准输出模板也写在 `05_文案路由规则.md`，用于每条真实视频执行前把最终文案补成句子级 `script_to_timeline_map（文案到时间线映射表）`、`tts_prosody_anchor_map（TTS 韵律锚点表）`、`opening_visual_hook_spec（开头视觉钩子规格，条件触发）` 和 `forbidden_visual_map（禁用画面表）`。缺这些前置字段时，不得进入视频执行、TTS 生成或高情绪开头生成。

### 2.4 1 份参考到执行落地契约机制文件
13. `12_参考到执行落地契约_reference_to_execution_contract.md`

`12_参考到执行落地契约_reference_to_execution_contract.md` 是 GPT Project / ChatGPT 侧 reference 保真入口。凡用户给 reference / 样片 / 参考图 / 参考视频 / 参考声音 / 原感稿 / 外部资料，必须先把它转换成 `reference_anchor -> effect_targets -> function_fields -> deviation_check -> done_when`，再下发 Codex 执行。

### 2.5 1 份目标驱动数据飞轮与文案执行闭环机制文件
14. `13_目标驱动数据飞轮与文案执行闭环_goal_driven_data_flywheel_and_copy_execution_loop.md`

`13_目标驱动数据飞轮与文案执行闭环_goal_driven_data_flywheel_and_copy_execution_loop.md` 是 GPT Project / ChatGPT 侧目标、阈值、文案修改前置读取、内容结构反馈、单主变量和 Codex 动态执行 prompt 的正式机制入口。

它承载：

- `goal_driven_data_flywheel_spec_v1（目标驱动数据飞轮规格 V1）`
- `threshold_config_v1（阈值配置 V1）`
- `lead_score_model（客资评分模型）`
- `data_goal_copy_revision_gate（数据目标驱动文案修改闸门）`
- `content_structure_feedback_engine（内容结构反馈引擎）`
- `single_primary_variable_rule（单主变量规则）`
- `next_video_execution_prompt（下一条视频执行 prompt）`
- `data_flywheel_memory（数据飞轮记忆）`

本文件写入只代表机制已落地，不代表目标飞轮已真实跑通，不代表阈值已被真实样本验证。
