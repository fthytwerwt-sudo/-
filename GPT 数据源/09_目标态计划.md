# 目标态计划

## 1. 文件定位

本文件只保存目标态和后续验证方向。

它不是当前事实清单，也不保存当前状态值。

## 2. 目标态使用规则

- 目标态只能写“希望验证什么”。
- 目标态不能写成“现在已经完成什么”。
- 目标态升级为事实前，必须先有 GitHub 当前文件证据。
- 若目标态与 GitHub 当前文件冲突，以 GitHub 当前文件为准。

## 3. 稳定目标方向

### 协作目标

- 让 ChatGPT 先按协作规则接手，再去 GitHub 读取当前事实。
- 让 Codex 负责维护 GitHub 文件、日志和同步口径。
- 让 GPT Project 静态资料不再污染当前事实判断。

### 内容目标

- 形成 `真实 AI 使用经验 + 工作提效实录` 的长期内容系统。
- 让每条视频更稳定地承担真实经验记录 / 提效证明 / 方法沉淀入口。
- 让工作包作为可选沉淀路径，而不是每条内容默认必须生成的产品单元。
- 让场景稳定作为内容切口 / 证据容器 / 问题发生地。
- 让用户录制素材承担真实证据链。
- 让少量 PPT 或卡片只承担辅助解释。

### 执行目标

- 让每轮执行留下可复读 GitHub 证据。
- 让审片入口、日志和状态文件互相一致。
- 让状态判断可被 ChatGPT 和 Codex 重复读取。
- 让所有最终上传包、复审包、素材包、manifest、README、contact sheet、zip 和导出包都默认落在 `/Users/fan/Documents/视频工厂` 内部。
- 让 GPT Project 静态协作包唯一稳定在 `/Users/fan/Documents/视频工厂/GPT 数据源/`。

### 声音目标

- 声音方向可以继续朝低压、清楚、轻陪伴的向导音探索。
- 具体试配结果和是否通过必须从 GitHub 当前文件读取。

## 4. 不得写成事实

- 不得把目标态写成当前完成。
- 不得把待验证路线写成已验证路线。
- 不得把协作规则写成项目事实。
- 不得把静态资料里的目标描述写成当前状态。
- 不得把桌面、下载目录或 `/private/tmp` 写成最终项目产物默认路径。

## 5. 一句话规则

目标态只负责指明后续要验证的方向；当前事实必须去 GitHub 读取。
