# 主线锚点与职责边界

## 1. 文件定位

本文件保存稳定主线和职责边界。

它不保存当前 round、当前声音试配结果或当前审片状态。

## 2. 稳定主线

当前项目讨论视频结构时，默认按下面主线理解：

**API 生成真人 + 用户录制素材 + 少量 PPT + 云端剪辑**

这是一条职责分配和方向边界，不代表某个具体当前样片已经完成验证。

## 3. 四个组成部分

### `API 生成真人`

负责开头进入、关键判断、转折衔接或结尾收束。

### `用户录制素材`

负责中段主体推进、过程证据和结果差证明。

### `少量 PPT`

负责关键词显影、结构整理、总结和必要的辅助解释。

### `云端剪辑`

表示正式组装方向；是否已经稳定跑通必须从 GitHub 当前执行记录读取。

## 4. 职责边界

- 结构跟着文案走，不先固定人像段次数。
- 中段主体证据优先由用户录制素材承担。
- 卡片、PPT、尾卡只做辅助，不替代证据链。
- 目标方向不等于当前执行事实。

## 5. 视觉与声音方向

### 视觉

`Minecraft-inspired` 只能理解为原创体素风格方向，不得复用官方 logo、字体、贴图、模型、音效或其他受保护资产。

### 声音

“可爱女生向导音”只能作为目标方向或听感锚点。

具体声音样本、声音试配、声音复刻、供应商和验证状态必须从 GitHub 当前文件读取。

声音试配存在不等于声音通过。

## 6. 一句话规则

本文件只保存主线职责边界；当前样片、声音和发布状态都去 GitHub 当前文件读取。
