# 主线锚点与职责边界

## 1. 文件定位

本文件保存稳定主线和职责边界。

它不保存当前 round、当前声音试配结果或当前审片状态。

## 2. 稳定主线

当前项目讨论视频结构时，默认按下面主线理解：

**API 生成真人 + 用户录制素材 + 少量 PPT + 云端剪辑**

这是一条职责分配和方向边界，不代表某个具体当前样片已经完成验证。

## 3. 四个组成部分

### `API 生成真人`

负责开头进入、关键判断、转折衔接或结尾收束。

### `用户录制素材`

负责中段主体推进、过程证据和结果差证明。

在 `真实 AI 使用经验 + 工作提效实录` 口径下，`用户录制素材` 更核心，必须优先承载真实使用过程、页面、操作、真实提效差值、真实踩坑、真实判断和真实试错。

### `少量 PPT`

负责关键词显影、结构整理、总结和必要的辅助解释。

### `云端剪辑`

表示正式组装方向；是否已经稳定跑通必须从 GitHub 当前执行记录读取。

## 4. 职责边界

- 结构跟着文案走，不先固定人像段次数。
- 中段主体证据优先由用户录制素材承担。
- 卡片、PPT、尾卡只做辅助，不替代证据链。
- 目标方向不等于当前执行事实。

## 4.1 真实经验主线如何挂到稳定主线

`已确认` 内容中心价值是 `真实 AI 使用经验 + 工作提效实录`，但稳定技术主线仍是 `API 生成真人 + 用户录制素材 + 少量 PPT + 云端剪辑`。

- `API 生成真人` 负责开头进入、关键判断、转折或收束。
- `用户录制素材` 负责真实经验、真实过程、真实提效差值和结果差证据。
- `少量 PPT` 只做关键词、结构和结尾辅助。
- `云端剪辑` 只表示组装方向，不写成已稳定跑通。
- 元素娃娃 / 主持娃娃 / 骚萌卡不替代真实证据，只做表达壳、人格化入口、节奏钩子和反转标点。

## 4A. 方案 B 骚萌卡在正式主线中的职责边界

三类骚萌卡已进入项目口径，但不改变当前稳定主线：`API 生成真人 + 用户录制素材 + 少量 PPT + 云端剪辑`。

骚萌卡不是装饰图，也不是固定贴片，而是“文案功能触发的搞笑人格分身”。

它只承担：
- 问题钩子。
- 反转触发器。
- 情绪标点。
- 停顿处的轻笑点。

它不承担：
- 不替代 用户录制素材 的真实操作证据。
- 不替代反面 / 正面录屏里的结果差展示。
- 不替代 API生成真人（API 生成人像 / 主持壳） 的正式判断职责。
- 不替代 结果差提示卡 的收束判断职责。
- 不替代 Prompt引用尾卡 的工作包承接职责。

三类骚萌卡与主线的关系：

| card | 所在位置 | 主线职责 |
| --- | --- | --- |
| `problem_hook_sassy_card（问题钩子骚萌卡）` | 元素娃娃 slogan 后，反面展示前 | 把用户痛点讲得好笑，给反面展示一个进入理由 |
| `negative_reversal_sassy_card（反面反转骚萌卡）` | 反面录屏里，旧做法结果刚露馅之后 | 把“看起来有东西，实际不能用”的翻车落差讲得好笑 |
| `positive_reversal_sassy_card（正面反转骚萌卡）` | 正面录屏里，新做法结果明显变好之后 | 把“这回真的能用了”的正向反转讲得好笑 |

节奏预算关系：

- 如果启用三张骚萌卡，必须优先从后半段回收时长。
- `shot07_clean_host（回场主持壳）` 默认删除或极短化。
- `shot08_judgment_card（判断总结卡）` 默认合并进 `shot06_result_diff_card（结果差提示卡）`。
- `shot03_negative_recording（反面真实录屏）` 与 `shot05_positive_recording（正面真实录屏）` 不作为优先压缩对象。
- `shot09_prompt_tail（Prompt 引用尾卡）` 保留必要展示，但不得重复讲结论。

## 5. 视觉与声音方向

### 视觉

`Minecraft-inspired` 只能理解为原创体素风格方向，不得复用官方 logo、字体、贴图、模型、音效或其他受保护资产。

### 声音

“可爱女生向导音”只能作为目标方向或听感锚点。

具体声音样本、声音试配、声音复刻、供应商和验证状态必须从 GitHub 当前文件读取。

声音试配存在不等于声音通过。

## 6. 一句话规则

本文件只保存主线职责边界；当前样片、声音和发布状态都去 GitHub 当前文件读取。
