## 3. 状态标记

### `已确认`

只能用于已有文件、执行记录、用户明确反馈或可复读 GitHub 证据支持的内容。

### `部分成立`

用于方向成立但链路、证据、同步或验证尚未完整闭环的内容。

### `待验证`

用于需要继续复审、执行、听感判断、用户确认或 GitHub 读取的内容。

### `推测`

用于风险判断、可能原因或还没有证据的判断。

## 4. 验证字段边界

### `technical_validation`

只表示技术链路、脚本、产物生成、扫描或基础检查结果。

它不等于 `content_validation`。

### `content_validation`

只表示内容表达、结构、证据链、节奏、声音和用户复审层面的内容判断。

它必须从 GitHub 当前状态读取。

### `send_ready`

只表示是否可发送或发布。

它必须从 GitHub 当前状态读取，不能由静态资料推断。

### 声音状态

声音样本、声音试配、声音复刻、TTS 供应商和最终音色都必须按 GitHub 当前证据读取。

试配存在不等于声音通过。

## 5. 禁止误写

- 不得把历史样片写成当前最新样片。
- 不得把技术验证写成内容验证。
- 不得把用户未确认写成可发送。
- 不得把声音试配写成声音通过。
- 不得把目标态写成当前事实。
- 不得把本包静态资料写成当前事实本体。
- 不得把桌面、下载目录或 `/private/tmp` 写成项目最终交付路径。
- 不得把无空格 `GPT数据源/` 写成 GPT Project 静态包默认上传来源。

## 6. 一句话规则

术语可以放在本包稳定保存；状态值必须去 GitHub 当前文件读取。
