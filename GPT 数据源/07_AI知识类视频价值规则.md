# AI 知识类视频价值规则

## 1. 文件定位

本文件保存稳定价值判断和验收框架。

它不保存某个样片是否通过、某条声音是否通过或某个执行轮次是否完成。

## 2. 价值底线

AI 知识类视频的价值不在于“讲了很多”，而在于：

- 帮用户更快拿到可用初稿。
- 让用户看到为什么结果更专业。
- 用真实过程、结果差或可复核证据支撑判断。
- 能自然连接到工作包、模板、prompt 包或服务单元。

## 3. 强证据优先级

优先级从高到低：

1. 同一任务在普通输入和工作包输入下的结果差。
2. 用户录制素材中的真实过程。
3. 前后对比。
4. 步骤截图或结果截图。
5. 口头说明。

## 4. 验收维度

内容复审至少看：

- 主题是否具体。
- 开头判断是否成立。
- 中段证据是否真实。
- 结构是否清楚。
- 节奏是否压得住。
- 声音是否符合目标听感。
- 结尾是否收住。
- 工作包承接是否自然。

这些是验收维度，不是当前通过结论。

## 5. 爆款与变现分开判断

- 更容易出爆，通常靠强情绪、强对比、强开头。
- 更容易变现，通常靠具体任务、结果差、可复用交付物。

两者不能互相冒充。

## 6. 不得写成当前事实

- 不得在本文件写某个样片已通过。
- 不得在本文件写某个声音试配已通过。
- 不得在本文件写某个供应商已经最终确定。
- 不得在本文件写某个商业化路径已经验证成立。

## 7. 一句话规则

本文件只保存价值判断框架；当前内容是否过线必须从 GitHub 当前审片状态和复审记录读取。
