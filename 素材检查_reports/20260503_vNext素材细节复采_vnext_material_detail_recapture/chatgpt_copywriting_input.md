# ChatGPT 文案输入：vNext 豆包到 Trae 流程证据

## 主结论

`已确认` 这轮素材可以支持的新叙事不是“某个工具一键已经全跑通”，而是：

> 用户给豆包一句简单需求，豆包把它翻译成 Trae 可执行的方案和 SOLO prompt；用户再把 prompt 放进 Trae SOLO，Trae 自动 plan 并生成项目骨架；后续再接 API、云端剪辑和 Codex 检查。

## 必须写出的真实流程

1. `需求入口`：一句简单需求。
   - 屏幕原文：`我想用 trae 做一个短视频自动流`
   - 文案可规范写成：`我想用 Trae 做一个短视频自动流`
   - 状态：`已确认`

2. `豆包输出方案`：豆包先给 Trae 短视频自动流方案。
   - 标题：`用 Trae 搭建短视频自动流：从 0 基础轻量版到无人值守全自动化版`
   - 核心流程：选题策划 → 脚本生成 → 分镜制作 → 视频生成 / 剪辑 → 配音字幕 → 封面标题 → 自动发布
   - 两套方案：0 基础零代码轻量自动流、无人值守全自动化版
   - 状态：`已确认`

3. `豆包生成 Trae prompt`：用户要求豆包给一个 Vlog 自动流 prompt。
   - 用户问题：`我主要是想做一个 Vlog 的视频自动流，你先给我一个 prompt 让 trae 帮我把架构搭建出来（脚本）`
   - 豆包标题：`Trae Vlog 自动流 核心搭建 Prompt（直接复制粘贴到 Trae SOLO 即可一键生成完整架构 + 可运行脚本）`
   - 模块包括：全局人设锚定模块、Vlog 选题与叙事线生成模块、Vlog 分镜与标准化脚本生成模块、素材智能匹配与调度模块、Vlog 专属自动化后期模块、成片与运营物料导出模块、总控调度与异常处理模块
   - 状态：`已确认`

4. `Trae 执行`：Trae SOLO 接住 prompt。
   - 画面先出现 Builder / IDE 协作入口，再进入 `SOLO Coder`
   - `SOLO Coder` 输入区出现豆包 prompt 的模块文本
   - Trae 回复：`让我先规划一下任务，然后逐步实现`
   - 出现 `Updating Tasks...` 和 `11 待办`
   - 创建项目名：`vlog_automation_workflow`
   - 文件树可见：`modules`、`templates`、`workflows`、`config`、`assets`、`frontend`、`logs`
   - 文件名可见：`settings.py`、`base_module.py`
   - 状态：`已确认`

5. `API`：只写成后续接工具能力。
   - 可以写：后续这个项目骨架还要接素材、API、剪辑、发布等能力。
   - 不可以写：API 已经全部接通。
   - 状态：`待验证`

6. `阿里云剪辑`：只写成用户选择的云端总装方向。
   - 可以写：后续云端总装验证会看阿里云剪辑能否接住用户录制素材、卡片素材、音轨和导出。
   - 不可以写：阿里云剪辑正式稳定。
   - 状态：`待验证`

7. `Codex`：写成执行检查和结果回报。
   - Codex 素材可见 `ffprobe`、命令执行、文件变更、路径索引、Git 操作、报告文件等。
   - 可以写：Codex 负责检查执行结果、路径、文件、命令和 Git 状态。
   - 不可以写：Codex 证明内容过线。
   - 状态：`已确认`

8. `即梦对比`：写成“单点素材生成 vs 可复用流程”。
   - 可写方向：即梦更像单点素材生成；这条流程更像把需求拆成可复用生产线。
   - 不要写成：即梦绝对不可用，或本流程已经稳定替代所有工具。
   - 状态：`部分成立`

## 不得写成

- 不得写成豆包已经工程跑通。
- 不得写成 Trae 代码已运行成功。
- 不得写成阿里云剪辑正式稳定。
- 不得写成 Codex 证明内容过线。
- 不得使用未打码火山引擎素材。
- 不得把 Trae 生成项目骨架写成完整交付验收。
- 不得把看不清的小字补成确定原文；看不清处统一标记 `uncertain_need_human_check`。

## 需要人眼复核

- `部分成立` Trae 画面能确认 prompt 已进入 `SOLO Coder` 并提交，但未清晰看到物理复制 / 粘贴快捷键动作。
- `部分成立` 豆包第二轮思考过程可见语义方向，但部分小字和截断句不宜逐字引用。
- `待验证` Trae 代码运行状态；画面出现 Python 扩展未安装提示和局部失败提示，因此只能写“生成结构 / 文件”，不能写“运行成功”。
