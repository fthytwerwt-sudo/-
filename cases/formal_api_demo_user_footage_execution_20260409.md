# 主题
把 AI 执行前的目标、边界、验收先压清，再交给 AI 推进

## 基础参数
- 总时长：15秒
- 视频比例：9:16

## 目标场景
AI 项目讲解

## 目标用户
正在用 AI 推项目、手里有真实录屏过程，但不想把整条视频做成 demo 感或说明书感的人

## 全局质量要求
- 开头 3 秒必须先用 API 真人把判断打出去
- 中段主体必须由用户录制素材承担，不能退回整条 PPT
- 中段必须有过程感、操作感或证据感
- 结尾只允许用少量轻量信息卡收束，不抢主叙事
- 看完后观众至少能拿走一个动作：先压清目标、边界、验收，再交给 AI
- 整条不能像 demo、不能像说明书、不能像乱切素材
- 组装继续按北京区 OSS + 云剪 cloud-only 理解

## 展示主线
- 路由画像：api_human_local_footage_light_ppt_cloud_editing
- 主策略：hybrid
- 路由理由：API 真人负责开头判断，用户真实录屏负责主体推进和证据感，结尾只用一张轻量收束卡把动作点压实。

## Hook
AI 最耗时间的，不是写，是你一开始没把需求压清。

## 结尾落点
先压清，再让 AI 干活，你会快很多。
