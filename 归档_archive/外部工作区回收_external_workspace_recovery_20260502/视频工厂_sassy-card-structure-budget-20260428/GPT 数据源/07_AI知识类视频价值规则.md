# AI 知识类视频价值规则

## 1. 文件定位

本文件保存稳定价值判断和验收框架。

它不保存某个样片是否通过、某条声音是否通过或某个执行轮次是否完成。

## 2. 价值底线

AI 知识类视频的价值不在于“讲了很多”，而在于：

- 帮用户更快拿到可用初稿。
- 让用户看到为什么结果更专业。
- 用真实过程、结果差或可复核证据支撑判断。
- 能自然连接到工作包、模板、prompt 包或服务单元。

## 3. 强证据优先级

优先级从高到低：

1. 同一任务在普通输入和工作包输入下的结果差。
2. 用户录制素材中的真实过程。
3. 前后对比。
4. 步骤截图或结果截图。
5. 口头说明。

## 4. 验收维度

内容复审至少看：

- 主题是否具体。
- 开头判断是否成立。
- 中段证据是否真实。
- 结构是否清楚。
- 节奏是否压得住。
- 声音是否符合目标听感。
- 结尾是否收住。
- 工作包承接是否自然。

这些是验收维度，不是当前通过结论。

## 4A. 方案 B 骚萌卡价值边界与内容验证前置规则

### 4A.1 价值边界

三类骚萌卡的价值不是“画面更热闹”，而是服务 AI 知识类视频的三件事：

- 帮观众看懂问题。
- 帮观众看懂反转。
- 帮观众看懂结果差。

搞笑必须增强理解，不得破坏证据链和可观看性。

没有文案钩子 / 反转 / 停顿时，不得为了热闹插卡。

当前不是为了每个位置都加好看的东西，而是在不降低可观看性的前提下，把“问题、反面反转、正面反转”做得更有梗、更好看。

### 4A.2 三类骚萌卡的价值职责

| card | 价值职责 | 不能越权 |
| --- | --- | --- |
| `problem_hook_sassy_card（问题钩子骚萌卡）` | 把真实用户痛点讲得好笑，让观众进入“这说的不就是我吗”的状态 | 不讲完整方法，不解释正反流程，不做总结，不做产品介绍 |
| `negative_reversal_sassy_card（反面反转骚萌卡）` | 把旧做法“看起来有东西，实际不能用”的落差变成笑点 | 不讲原因，不讲方法，不提前讲正面答案，不破坏反面录屏证据链 |
| `positive_reversal_sassy_card（正面反转骚萌卡）` | 放大新做法“这回真的能用了”的正向反转和爽点 | 不继续吐槽旧做法，不重复解释方法，不抢结果差提示卡的总结职责 |

### 4A.3 内容验证前置规则

进入新一轮 `content_validation（内容验证）` 前，若使用三张骚萌卡，必须先检查：

1. 是否真的存在对应文案钩子或反转。
2. 是否有足够停顿承接骚萌卡。
3. 是否仍由 `用户录制素材` 承担反面 / 正面主体证据。
4. 是否没有让骚萌卡替代 `API生成真人（API 生成人像 / 主持壳）` 的正式判断职责。
5. 是否没有让骚萌卡替代 `结果差提示卡（结果差卡）` 的收束判断职责。
6. 是否没有让骚萌卡和真实录屏争夺主体。
7. 是否从后半段回收了足够时长，而不是把全片继续拉长。
8. 三张骚萌卡是否同一角色体系、同一视觉风格、同一搞笑方向。

一票否决：
- 只可爱不搞笑。
- 只说明不反转。
- 像普通信息卡。
- 像固定贴纸。
- 抢真实录屏证据。
- 为了热闹插入，导致节奏变散。
- 后半段不回收时长，导致整片可观看性下降。

## 5. 爆款与变现分开判断

- 更容易出爆，通常靠强情绪、强对比、强开头。
- 更容易变现，通常靠具体任务、结果差、可复用交付物。

两者不能互相冒充。

## 6. 不得写成当前事实

- 不得在本文件写某个样片已通过。
- 不得在本文件写某个声音试配已通过。
- 不得在本文件写某个供应商已经最终确定。
- 不得在本文件写某个商业化路径已经验证成立。

## 7. 一句话规则

本文件只保存价值判断框架；当前内容是否过线必须从 GitHub 当前审片状态和复审记录读取。
