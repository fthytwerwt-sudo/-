# 文案路由规则

## 1. 文件定位

本文件保存稳定的文案路由和承载分配规则。

它不保存当前 round、当前样片状态或当前发布状态。

## 2. 文案类型

默认先分四类：

1. `内容表达文案`
2. `分发包装文案`
3. `关系建立文案`
4. `变现承接文案`

若用户没有特别说明，优先按 `内容表达文案` 判断。

## 3. 内容表达文案默认交付结构

内容表达文案至少要能拆出：

1. 完整脚本。
2. block 到 segment 的承载分配。
3. 录制素材清单。
4. prompt 包或工作包结构。
5. 结果差证明点。
6. 总结卡或收束文案。
7. 回审清单。

## 4. 承载分配原则

| 内容段 | 默认承载 |
| --- | --- |
| 开头进入 | `API 生成真人` 或清楚的判断段 |
| 问题点破 | 判断句、对比句、短证据 |
| 过程演示 | `用户录制素材` |
| 结构整理 | 少量 PPT / 卡片 |
| 结尾收束 | 总结句、少量 PPT、必要时人像段 |
| 工作包引用 | 尾卡或补充说明 |

## 5. 禁止路由

- 不允许让 PPT 替代主体证据。
- 不允许让人像段替代真实过程证据。
- 不允许先定视觉次数，再硬塞文案。
- 不允许把尾卡做成第二个主结尾。
- 不允许把云端剪辑方向写成已经稳定执行的事实。

## 6. Docker / 工作台类任务

若任务命中 Docker、工作台、compose、录制减负、工作包生成、prompt 生成或结果差对比：

1. 先确认是不是中间层 scaffold 或规则问题。
2. 再检查 GitHub 当前文件、脚本和日志。
3. 只做最小可逆修正。
4. 不把“有默认修正路径”写成“链路已稳定跑通”。

## 7. 方案 B 三类骚萌卡结构机制与节奏预算

### 7.1 用户确认口径

用户确认的新结构不是“多加几张好看的卡”，而是把 `骚萌卡` 写成文案功能触发机制。

- 正面也需要一张骚萌卡，因为正面之后文案也会有反转。
- 有反转的阶段，就可以触发一张骚萌卡。
- 元素娃娃 slogan 后面还有一个骚萌卡位置。
- 三张骚萌卡必须风格统一，不能变成不同角色。
- 三张骚萌卡都必须搞笑一点，不能只可爱、不能只说明、不能变成普通信息卡。
- 每张骚萌卡主要表现什么、承载什么，必须在文案路由阶段先写清。
- 当前不是为了每个位置都加好看的东西，而是要保证视频可观看性。
- 骚萌卡必须和文案、视频节奏搭配。

### 7.2 总机制

- 骚萌卡不是装饰图。
- 骚萌卡不是固定贴片。
- 骚萌卡是“文案功能触发的搞笑人格分身”。
- 三张骚萌卡必须统一风格，但承担不同叙事职责。
- 三张骚萌卡都必须有搞笑感。
- 骚萌卡必须和文案反转 / 问题钩子 / 视频停顿配合，不能只因为画面好看就插。
- 如果使用三张骚萌卡，必须从后半段回收时长，优先压缩或删除 `回场主持壳（API 生成真人回场段）` 和 `judgment_card（判断总结卡）`。

默认结构顺序：

1. 元素娃娃 slogan（开头定性）
2. `problem_hook_sassy_card（问题钩子骚萌卡）`
3. 反面展示 / 反面录屏
4. `negative_reversal_sassy_card（反面反转骚萌卡）`
5. 正面展示 / 正面录屏
6. `positive_reversal_sassy_card（正面反转骚萌卡）`
7. 结果差提示卡
8. `Prompt 引用尾卡`

### 7.3 `problem_hook_sassy_card（问题钩子骚萌卡）`

位置：
- 元素娃娃 slogan 后，反面展示前。

核心职责：
- 点出问题。
- 把观众从“听介绍”拉到“这说的不就是我吗”。
- 用骚萌搞笑的方式把问题讲出来。

它承载：
- 当前视频的真实问题。
- 用户痛点。
- 一句轻吐槽。
- 进入反面展示的理由。

它不承载：
- 不讲完整方法。
- 不解释正反流程。
- 不展示结果差。
- 不做总结。
- 不做产品介绍。

情绪：
- 发现问题。
- 眯眼吐槽。
- 轻微嫌弃但可爱。
- 必须搞笑，不能只是卖萌。

表情动作：
- 挑眉。
- 侧眼。
- 捂嘴偷笑。
- 手指轻点。
- 像发现大家都在犯同一个小毛病。

文案例子：
- “我发现一个问题：很多人用 AI，不是不会问，是第一句就把它问懵了。”
- “你不是不会用 AI，你是开头就把任务丢成一团。”
- “很多人用 AI，第一步就把路问歪了。”

建议时长：
- `1.6s-2.2s`

失败线：
- 变成普通解释卡。
- 文案太正经。
- 不搞笑。
- 像课程标题页。
- 和元素娃娃 slogan 重复。

### 7.4 `negative_reversal_sassy_card（反面反转骚萌卡）`

位置：
- 反面录屏里，旧做法结果刚露馅之后。

核心职责：
- 替观众吐槽翻车结果。
- 把“看起来有东西，实际不能用”的落差变成笑点。

它承载：
- 反面结果的反转。
- 空泛 / 泛泛而谈 / 看起来对但交不了的落差。
- 观众的无语感。
- 第一处强笑点。

它不承载：
- 不讲原因。
- 不讲方法。
- 不提前讲正面答案。
- 不做产品介绍。

情绪：
- 贱萌吐槽。
- 得瑟拆台。
- 欠揍可爱。
- 必须搞笑，不能只是可爱。

表情动作：
- 挑眉。
- 小坏笑。
- 捂嘴偷笑。
- 手一摊。
- 像在说“你看，我就知道会这样”。

文案例子：
- “方案很满，用起来空。”
- “看着像方案，实际像空气。”
- “字挺多，但没一处能直接用。”
- “这不是初稿，这是礼貌废话。”

建议时长：
- `1.2s-1.6s`

失败线：
- 反面结果还没露出来就插。
- 只卖萌不吐槽。
- 把观众骂了。
- 破坏录屏证据链。
- 时长太长导致反面展示断掉。

### 7.5 `positive_reversal_sassy_card（正面反转骚萌卡）`

位置：
- 正面录屏里，新做法结果明显变好之后。

核心职责：
- 放大“这回真的能用了”的正向反转。
- 把正面结果的可用度提升变成第二个轻笑点。

它承载：
- 正面结果突然变清楚。
- 工作包 / prompt 调整后的差值。
- “终于像能交的初稿了”的爽点。
- 第二处轻笑点。

它不承载：
- 不继续吐槽旧做法。
- 不重复解释方法。
- 不抢结果差提示卡的总结职责。

情绪：
- 小得意。
- 小惊喜。
- 轻微炫耀。
- 像在说“这才对嘛”。
- 必须搞笑，不能变成普通鼓励卡。

表情动作：
- wink（眨眼）。
- 点头。
- 小骄傲。
- 小手势。
- 歪头得瑟。
- 轻微小扭动。

文案例子：
- “诶，这回像能交的初稿了。”
- “这才像回事嘛。”
- “你看，顺序一对，结果就不飘了。”
- “终于不是空气方案了。”

建议时长：
- `1.2s-1.6s`

失败线：
- 正面结果没有明显反转却硬插。
- 还在继续吐槽旧做法，导致情绪重复。
- 变成炫技。
- 抢走正面录屏的证据主体。

### 7.6 统一风格锁定

- 原创 Q 版 AI 向导 / 可爱元素主持娃娃。
- 高质量 3D / GIF 感。
- 9:16 竖屏整页 reaction card（整页反应卡）。
- 橙黄爆炸 / 放射线 / 表情包式背景。
- 大字 punchline（大字笑点）清楚可读。
- 表情骚萌、得瑟、戏精、欠揍可爱。
- 必须搞笑。
- 同一个角色体系，不能每张像换了角色。
- 不能写实人物。
- 不能普通贴纸。
- 不能低幼卖萌。
- 不能性感擦边。
- 不能出现胸口 AI 标识。
- 不能使用已有 IP / meme 角色 / 平台 UI。
- 不能回到本地程序绘图感。

可以：
- 挑眉。
- wink（眨眼）。
- 捂嘴偷笑。
- 歪头。
- 侧眼。
- 吐槽脸。
- 小得意。
- 小坏笑。
- 轻微扭动。

不要：
- 大哭。
- 惊恐。
- 抱头崩溃。
- X 眼。
- 旋涡眼。
- 尖叫嘴。
- 撒娇。
- 擦边。
- 幼态。
- 软色情。
- 低幼装可爱。

### 7.7 触发规则

骚萌卡不是按段落固定出现，而是按文案功能触发。

触发条件：

1. 问题钩子触发：
   - 当元素娃娃 slogan 后，需要用一句人话点出用户痛点时，触发 `problem_hook_sassy_card（问题钩子骚萌卡）`。
2. 反面反转触发：
   - 当旧做法结果暴露出“空泛 / 不能用 / 看着对但交不了”时，触发 `negative_reversal_sassy_card（反面反转骚萌卡）`。
3. 正面反转触发：
   - 当新做法结果暴露出“突然清楚 / 能交 / 前后差明显”时，触发 `positive_reversal_sassy_card（正面反转骚萌卡）`。

禁止触发：
- 没有文案钩子。
- 没有反转。
- 没有停顿。
- 只是为了画面热闹。
- 会打断录屏证据链。
- 会让视频超过节奏预算。
- 会让骚萌卡和真实录屏争夺主体。

### 7.8 节奏预算

- 当前 full 约 `56 秒`，精确记录为 `55.92s`。
- 当前中段约 `28.5 秒`，精确记录为 `28.52s`。
- 三张骚萌卡预计总时长约 `4.0s-5.4s`。
- 如果启用三张骚萌卡，必须从后半段回收至少 `4-6 秒`。

三张骚萌卡建议预算：

| card | 建议时长 |
| --- | --- |
| `problem_hook_sassy_card（问题钩子骚萌卡）` | `1.6s-2.2s` |
| `negative_reversal_sassy_card（反面反转骚萌卡）` | `1.2s-1.6s` |
| `positive_reversal_sassy_card（正面反转骚萌卡）` | `1.2s-1.6s` |
| 合计 | `4.0s-5.4s` |

优先压缩 / 删除：
- `shot07_clean_host（回场主持壳）`
- `shot08_judgment_card（判断总结卡）`

不压缩：
- `shot03_negative_recording（反面真实录屏）`
- `shot05_positive_recording（正面真实录屏）`
- `shot09_prompt_tail（Prompt 引用尾卡）` 的必要展示

节奏收束规则：
- `shot06_result_diff_card（结果差提示卡）` 负责收住判断，不再继续搞笑。
- 如果保留三张骚萌卡，`回场主持壳（API 生成真人回场段）` 默认删除或极短化。
- `judgment_card（判断总结卡）` 默认合并进 `结果差提示卡（结果差卡）`。
- `Prompt引用尾卡（Prompt 引用尾卡）` 保留，但不得重复讲结论。

## 8. 一句话规则

文案路由先拆任务、证据和承载，再决定脚本和视觉；动态执行状态去 GitHub 读取。
