# block / segment 承载分配表

## 1. 使用边界

`已确认` 本表只做计划分配，不生成视频、音频、图片。
`已确认` 用户录制素材承担主体推进。
`已确认` 信息卡只做辅助，不替代中段主叙事。
`已确认` HyperFrames 动效不得抢真实录屏证据主体。

## 3. 不生成信息卡的原则

`已确认` 当真实录屏本身能证明动作和证据点时，不额外插信息卡。
`已确认` 若信息卡会压缩豆包 / Trae / Codex 真实流程证据，则不生成。
`已确认` Prompt 引用尾卡只显示 1-2 行关键词，不承担主叙事。
