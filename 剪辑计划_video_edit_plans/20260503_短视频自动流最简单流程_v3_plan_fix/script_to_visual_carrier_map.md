# 完整文案到画面承载映射表

## 1. 使用边界

`已确认` 完整文案语义必须保真。
`已确认` 本表不是删稿表，而是画面承载分配表。
`已确认` 不得把完整文案全压到信息卡，也不得全交给 TTS 而无画面推进。

## 3. 承载硬规则

- 主持壳承担开头、判断、转折、收束。
- 用户录制素材承担豆包 / Trae / Codex 主体证据。
- 信息卡只显影关键词、边界和总结。
- HyperFrames 只用于卡片动效，不接录屏。
