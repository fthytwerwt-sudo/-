# 动态资料源同步规则

## 1. 文件定位

本文件只解决一件事：

- GPT 项目源 / 动态资料源 / ChatGPT 侧实时更新内容，什么时候必须桥接到 Codex 执行层

它不负责：

- 改项目主线
- 代替项目脑正文
- 代替执行偏差回写规则

## 2. 什么叫动态资料源

只要内容满足“更新发生在仓库外，或只存在于 ChatGPT / GPT 侧，但会影响执行判断”，就属于动态资料源。

### GPT 项目源

- GPT 项目里的说明
- GPT 持有的项目背景
- GPT 侧新增的长期规则

### ChatGPT 项目数据源

- ChatGPT 当前线程里新收束的判断
- ChatGPT 侧维护的最新拍板
- 只存在于 ChatGPT 侧的数据整理结果

### Drive / Docs / 项目脑动态更新

- Google Docs / Drive 中新增或更新的项目资料
- 不在仓库里，但被 ChatGPT 引用的新文档
- 项目脑外部资料库的新增结论

### 只存在于 ChatGPT 侧、但会影响执行的新内容

- 新的执行前提
- 新的任务边界
- 新的主路径口径
- 新的禁止项
- 新的验收口径

## 3. 三档同步分级

### A. 仅参考更新

定义：

- 只提供参考
- 不改变本轮执行
- 不改变长期前提

### B. 本轮执行更新

定义：

- 会影响本轮执行
- 但不一定升格为长期默认事实

### C. 长期前提更新

定义：

- 会影响后续默认执行
- 会改变执行前提、主路径、默认读取或长期背景

## 4. 各档命中条件与桥接动作

### A. 仅参考更新

- 什么情况下命中：
  - 只是补充资料
  - 只是表达参考
  - 只是备选路线
  - 当前轮不据此修改执行动作
- 要不要桥接进 Codex：
  - 不强制
- 要写到哪里：
  - 可只保留在 ChatGPT 侧
  - 若需要留痕，可写进本轮执行单的“参考信息”

### B. 本轮执行更新

- 什么情况下命中：
  - 会改变本轮任务做法
  - 会改变本轮禁止项或验收口径
  - 会让 Codex 本轮不能继续按旧理解执行
- 要不要桥接进 Codex：
  - 必须
- 要写到哪里：
  - 优先写入 `codex_source/03_research_findings_bridge.md`
  - 或在本轮执行单中显式带入

### C. 长期前提更新

- 什么情况下命中：
  - 会改变当前阶段长期有效的执行前上下文
  - 会改变当前正式主路径
  - 会改变当前明确不做事项
  - 会改变默认读取顺序或长期桥接规则
- 要不要桥接进 Codex：
  - 必须
- 要写到哪里：
  - 先写入 `codex_source/03_research_findings_bridge.md`
  - 再同步更新 `codex_source/02_current_execution_context.md`
  - 若执行规则本身被改变，再同步更新相关 `codex_source/*`

## 5. 硬规则

- 只存在于 GPT 项目源里的内容，不得默认视为 Codex 已知。
- 只存在于 ChatGPT 侧的新内容，不得默认视为 Codex 已知。
- 会影响本轮执行的，必须写进 `codex_source/03_research_findings_bridge.md` 或本轮执行单。
- 会影响长期前提的，必须同步更新 `codex_source/02_current_execution_context.md`。
- 若动态资料源已经改变执行现实，还要同时遵守 `codex_source/05_execution_deviation_and_reality_sync.md`。

## 6. 一句话规则

只在 GPT / ChatGPT / 外部动态资料源里更新过、但还没桥接进执行层的内容，Codex 一律不得默认当成已知并据此执行。
