# AI 知识类视频价值桥接

## 1. 文件定位

本文件用于把项目脑里的价值底线、内容类型差异与结尾总结卡映射，桥接进 Codex 执行层。

## 2. 当前正式价值底线

当前 AI 知识类视频默认必须满足：

- 不能只说问题
- 不能只给观点
- 不能没有动作
- 不能没有证据
- 不能让用户看完后仍不知道下一步做什么
- generation / assembly 成功，不等于内容过线

## 3. 四类内容的默认交付与结尾

1. `AI 项目讲解`
   - 默认交付：判断项目卡点、关键取舍和最小下一步
   - 默认结尾：`judgment_card + 最小行动`
2. `AI 方法分享`
   - 默认交付：可直接试的方法 + 易错点
   - 默认结尾：`steps_error_card`
3. `AI 学习实操`
   - 默认交付：最小步骤 + 自检
   - 默认结尾：`steps_card + 自检句`
4. `AI 案例拆解`
   - 默认交付：案例为什么成立 + 如何迁移
   - 默认结尾：`judgment_card + 可迁移句`

## 4. 样片执行前必须先锁什么

凡是准备进入样片执行，至少先锁：

1. 当前内容类型
2. 用户看完后能做什么
3. 用户看完后能判断什么
4. 最关键证据是什么
5. 最小行动 / 自检句是什么
6. 结尾总结卡类型是什么

未锁清前：

- 不得把文案直接压进样片执行
- 不得把 generation / assembly 成功写成内容已过线

## 5. 当前执行含义

当前 Codex 执行层必须默认应用这套桥接到以下动作：

- 写脚本
- 做 block 路由
- 选结尾总结卡
- 判断是否值得进样片
- 做样片验收

必须同时明确：

- 这套桥接已是仓库正式价值口径
- 但这不等于“新主线样片已验证成立”

## 6. 一句话规则

当前 Codex 执行层必须按“看完后用户能带走有用东西”理解 AI 知识类视频质量线；4 类内容不再共用一种价值交付、证据结构与结尾总结卡，进入样片前必须先锁清“能做什么 / 能判断什么 / 证据是什么 / 最小行动或自检句是什么 / 结尾卡是什么”。
