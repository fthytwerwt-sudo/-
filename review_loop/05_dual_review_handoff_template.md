# 双审核交接模板

## 1. 文件定位

本模板用于承接“Codex 初检 → ChatGPT 质量判断”的双审核流程。

分工必须写死：

- Codex 先做结构化初检
- ChatGPT 做最终判断拍板

Codex 不越权下最终结论，只负责把记录、缺失项、疑似问题层和交接信息压清楚。

## 2. A. 执行审核交接

请按下面结构填写：

- 本轮真实记录了什么：
- 数据是否齐：
- 模板是否按要求填：
- 哪些字段缺失：
- 是否已归档：

推荐补充说明：

- `数据是否齐`
  - 只回答齐 / 部分齐 / 不齐
- `模板是否按要求填`
  - 重点检查是否有：
    - 视频编号
    - 主场景
    - 主结构
    - 本条唯一改动变量
    - 结果字段
    - 当前判断状态
    - 下一轮唯一最优先改点
- `是否已归档`
  - 只回答已归档 / 未归档，并注明落点文件

## 3. B. 质量审核交接

请按下面结构填写：

- 当前最可能问题层：
- 次可能问题层：
- 当前判断状态：
- 建议 ChatGPT 重点看哪一层：
- 当前不要越权下最终结论：

推荐填写口径：

- `当前最可能问题层`
  - 只写一个主层
- `次可能问题层`
  - 只写一个次层
- `当前判断状态`
  - 固定写 `已确认 / 部分成立 / 待验证`
- `建议 ChatGPT 重点看哪一层`
  - 只给一个重点，不平均发力
- `当前不要越权下最终结论`
  - 明确写：`以下仅为 Codex 初检收口，最终问题层与下一轮唯一最优先改点由 ChatGPT 拍板`

## 4. 固定提醒

- Codex 先做结构化初检，不做最终拍板
- ChatGPT 负责判断问题层、纠偏和决定下一轮只最该改什么
- 如果字段明显缺失，先把缺失写清，再交质量判断
- 如果本轮一次改了多个变量，必须在交接里明确说明诊断价值下降

## 5. 当前一句话规则

双审核交接的目标不是让 Codex 抢判断，而是先把结构、字段、缺失项和疑似问题层收干净，再把最终判断交给 ChatGPT。
