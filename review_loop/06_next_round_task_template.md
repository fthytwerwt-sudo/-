# 下轮视频执行单草稿模板

适用场景：

- 上一条视频已经完成记录和初检
- 需要给下一条生成执行单草稿
- 当前只允许收成一个最优先改点

使用要求：

- 模板语气偏执行，不偏分析
- 下一轮只改一个变量
- 其余变量保持不变
- 不允许顺手扩成多变量联动测试
- 必须承接上一轮 `review_variable_card（复盘变量卡）`
- 如果上一轮没有锁定主变量，先写 `待验证`，不得硬生成执行任务

## 执行备注

- 主场景默认沿用上一条，除非本轮明确决定改场景
- 主结构默认沿用上一条，除非本轮唯一改动变量就是结构
- 如果上一条结果字段还不齐，这一轮草稿状态默认写保守
- 如果上一条一次改了多个变量，这一轮先收窄，不继续扩散
- 如果 24h / 72h / 7 天数据不足，不能硬定变量，先写 `待验证`
- 默认只改 1 个变量；特殊情况下最多 2 个，但必须说明主变量和辅助变量
- 如果 `review_variable_card（复盘变量卡）` 缺主变量、观察窗口或目标信号，本模板不得写成已可执行

## 固定落款

请明确写上：

- `本草稿仅用于下一轮执行准备`
- `最终是否采用、下一轮到底只改哪一个点，由 ChatGPT 拍板`

## 当前一句话规则

下轮执行单草稿的任务只有一个：把“上一条发生了什么、下一条只改什么”收清楚，不要顺手把别的变量一起改掉。
