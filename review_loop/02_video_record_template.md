# 单条视频记录模板

适用场景：

- 一条视频已经准备发布或已经发布
- 需要把“本轮到底测了什么、结果是什么、下一轮只改什么”固定落下来

使用原则：

- 一条视频一份记录
- 先把发布前变量写清，再填结果字段
- 当前优先使用项目内主场景与主结构口径
- 不在这份模板里做跨视频总结

## 填写提醒

- 如果本轮一次改了多个变量，必须在“最主要现象描述”或“当前判断状态”里明确写出诊断价值下降
- 如果结果字段不齐，不要硬写强判断，优先用 `待验证`
- 如果已经能看出方向，但仍不够稳，优先用 `部分成立`
- 24h / 72h 数据未回填前，不得写成内容通过或规律成立
- 异常样本可以记录、可以参考，但不得作为规律沉淀主证据

## 截图优先录入规则

用户可以直接提交平台数据截图，Codex 负责提取字段并写入记录。

每条视频必须独立建档。

每条视频至少分三个时间窗：

- 24h
- 72h
- 7d

每个时间窗的数据不得互相覆盖。

### 最低统计频率

- 最低必须统计：24h、72h
- 标准建议统计：24h、72h、7d

### 时间窗含义

- 24h：看开局，判断初始分发、标题 / 封面 / 开头是否接住。
- 72h：看趋势，判断内容层、账号层、私信咨询是否出现稳定信号。
- 7d：封账，判断是否达到 6000 播放基础门槛，以及是否可以进入下一条文案。

### 写入规则

- 截图录入前必须先确认 `video_id`。
- 截图录入前必须先确认 `24h / 72h / 7d` 时间窗。
- 平台数据、留存完播、互动、账号增长、评论、私信、咨询截图必须分开归类。
- 截图看不清或字段识别不确定时，必须标记 `uncertain_need_human_check`。
- 截图没有提供的字段，必须标记 `missing`，不得硬猜。
