# 复盘状态规则

## 1. 文件定位

本文件用于固定当前复盘系统里的状态口径，避免把小样本阶段的现象写得过满。

当前只使用三种状态：

- `已确认`
- `部分成立`
- `待验证`

## 2. 三种状态怎么用

### 已确认

适用场景：

- 当前条目对应的是已经真实发生、已被记录的执行事实
- 或者某个判断已经有足够直接证据支撑，当前轮不再把它当成猜测

使用边界：

- 如果说的是“字段是否齐”“本条改了哪个变量”“视频是否已发布”“数据是否已回填”，可以直接用 `已确认`
- 如果说的是“某个内容规律已经成立”“某个结构必然更好”，不能只凭一两条样本就写成 `已确认`

### 部分成立

适用场景：

- 当前看起来已经有方向
- 但仍夹杂其他变量干扰
- 或者证据还不足以把它写成稳定规律

典型用法：

- 某次改动后结果明显变好或变差，但样本仍少
- 现象和问题层大体对得上，但还不能排除其他原因

### 待验证

适用场景：

- 当前只是猜测
- 结果字段还没齐
- 只有单条样本
- 或者本轮改动不够干净，暂时无法判断

典型用法：

- 视频刚发，数据还没跑出来
- 只有“感觉像这样”，但没有结果字段支撑
- 本轮一次改了不止一个变量，诊断价值明显下降

## 3. 小样本阶段的硬规则

### 规则 A：2 条样本通常只能看方向

当前小样本阶段，`2` 条样本通常只能帮助我们看方向，不足以直接写成稳定规律。

这意味着：

- 两条都变好，不等于已经证明规律成立
- 两条都变差，不等于已经证明根因唯一
- 两条方向一致时，更合理的写法通常是 `部分成立`

### 规则 B：不允许把单条偶然结果写成规律

以下写法都不允许直接成立：

- “这类主场景一定更好”
- “这个钩子一定有效”
- “这个结构已经被验证是最佳”

单条视频更适合写成：

- 本轮现象是什么
- 这更像哪一层的问题
- 下一轮最值得继续试哪个单变量

### 规则 C：一次改 2 个以上变量，会降低诊断价值

复盘的核心不是一次把所有问题都改完，而是让下一轮可判断。

因此默认要求：

- 每条视频尽量只改 `1` 个变量
- 若一次改了 `2` 个以上变量，必须明确标注“诊断价值下降”
- 在这种情况下，状态默认更保守，优先写 `待验证` 或 `部分成立`

## 4. 推荐判断顺序

每次填状态前，先问四个问题：

1. 这是执行事实，还是效果规律
2. 结果字段是否够支撑当前写法
3. 本轮是否只改了一个变量
4. 当前写成 `已确认` 会不会过满

只要其中任一项回答不稳，就优先使用更保守的状态。

## 5. 当前一句话规则

当前复盘系统宁可把状态写保守，也不要把单条偶然结果写成规律；小样本阶段先看方向，不抢着下结论。
