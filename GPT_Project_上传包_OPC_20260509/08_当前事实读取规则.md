# 当前事实读取规则

## 1. 文件定位

本文件不保存当前事实本体。

它只说明：每次需要当前事实时，从哪里读、如何裁决冲突、读不到时怎么办。

## 2. 默认事实源

每次需要当前事实时，优先读 GitHub 当前文件。

默认主读取分支：
- `main`

默认读取顺序：

1. `AGENTS.md`
   - 仓库顶层入口规则。
2. `codex_log/latest.md`
   - 最新执行摘要。
3. `codex_log/current_local_artifact_paths.md`
   - Codex 已本地验证存在的规范路径索引。
4. `dist/latest_review_pack/summary.json`
   - 最新审片包状态摘要。
5. `dist/latest_review_pack/review_manifest.md`
   - 最新审片入口。
6. `GPT数据源/08_当前正式事实.md`
   - 当前正式事实正文。
7. 如涉及项目总述、OPC 机制、读取顺序，再读 GitHub 当前 `GPT数据源/00_项目总述.md`、`GPT数据源/03_总索引与阅读顺序.md`、`GPT数据源/10_OPC一人公司闭环与多AI协作机制.md`。
8. 如涉及当前发布目标、声音、样片、具体执行或历史判断，再读对应 `codex_log/*.md`。

## 3. 需要从 GitHub 读取的动态事实

以下内容不得由本上传包静态资料保存或推断：

- 当前 round。
- 当前 `latest_review_pack`。
- 当前 `content_validation`。
- 当前 `send_ready`。
- 当前 `publish_status`。
- 当前声音 trial 状态。
- 当前 TTS / voice cloning 状态。
- 当前审片入口。
- 当前执行产物状态。
- 当前 remaining blockers。

## 4. 冲突裁决

- GitHub 当前文件 > 本上传包静态资料。
- 本上传包只给协作方式，不给当前事实。
- 若本上传包与 GitHub 说法冲突，以 GitHub 当前 `main` 为准。
- 若无法读取 GitHub，必须保守接手，不得用本上传包旧内容补当前事实。

## 5. 禁止误写

- 不得把历史样片写成当前最新样片。
- 不得把技术验证写成内容验证。
- 不得把用户未确认写成可发送。
- 不得把声音试配写成声音通过。
- 不得把目标态写成当前事实。
- 不得把本上传包静态文字写成 GitHub 已同步动态事实。

## 6. 读不到 GitHub 时怎么办

如果无法读取 GitHub：

1. 明确标记 `待验证`。
2. 说明缺少哪个路径或分支。
3. 不补写具体当前状态。
4. 不用静态资料里的历史描述替代当前事实。
5. 优先让 Codex 同步当前事实文件或重新给出本地可读 canonical path。

## 7. 一句话规则

当前事实永远从 GitHub 当前 `main` 文件读取；本上传包只提供协作规则和接手骨架。
