# 03｜以后修改下发模板

这些模板不是让用户直接丢给 Codex（写入执行器）硬跑，而是先复制给 ChatGPT（总控脑），由 ChatGPT（总控脑）补齐证据、边界和验收后，再生成 Codex（写入执行器）执行单。

## 模板 1：我要修工程状态地图

```markdown
【我要修哪一层】
目标层 / 状态层。重点是 `engineering_state_map（工程状态地图）` 和 `acceptance_contract（验收契约）`。

【当前状态】
请先判断每个模块是 `formal（正式）`、`candidate（候选）`、`probe_only（仅探测）`、`documented_only（仅文档）`、`missing（缺失）` 还是 `conflict（冲突）`。

【这次只完成什么】
只完成工程状态地图和验收契约，不修 `runtime（运行时）`，不修 `RAG（检索增强生成）`，不启动 `service（服务）`。

【允许改哪些文件】
- codex_log/engineering_line_audit/*
- codex_log/engineering_line_plain_manual/*
- codex_log/latest.md
- 如另行授权，才允许新增 schema（结构）或 fixture（样例）

【禁止改哪些文件】
- codex_source/adapter_integration/runtime_entry.py
- codex_source/adapter_integration/service_boundary.py
- .env*
- dist/**
- media/**
- GPT数据源/**

【必须先读取】
- AGENTS.md
- codex_log/latest.md
- codex_log/engineering_line_audit/20260617_engineering_line_audit_report.md
- codex_log/engineering_line_audit/20260617_engineering_line_gap_matrix.md
- codex_source/22_工作流入口归位索引_workflow_entry_routing_index.md
- codex_source/adapter_integration/workflow_registry.py

【执行步骤】
1. 列出所有工程模块。
2. 给每个模块标状态。
3. 给每个状态绑定证据路径。
4. 给每个状态写 `done_when（做到哪算完成）`。

6. 单列 `conflict（冲突）` 项。

- 必须改 `runtime（运行时）` 才能继续。
- 必须启动服务或调用外部 API（外部接口）。

【最后怎么回报】
回报状态地图文件、验收契约文件、冲突登记、禁止状态推进扫描、commit（提交）、push（推送）和远端 HEAD（远端分支头）验证。
```
