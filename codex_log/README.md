# codex_log 说明

`codex_log/` 是《视频工厂》仓库的执行日志目录。

这里记录的是 Codex 的真实执行过程、交付清单、命令结果、阻塞点和交接点，不是项目脑正文，也不替代 `project_source/`。

## 目录规则

- `codex_log/latest.md` 永远保存最近一次执行的简版摘要。
- 每次需要落日志时，都新增一条独立日志文件。
- 文件名格式固定为 `YYYYMMDD_task_name.md`。
- 文件名使用英文和下划线，不用空格。

## 什么时候必须写日志

命中以下任一情况，就必须写：

- 改了仓库文件
- 跑了命令
- 生成了产物
- 完成了 commit / push / PR
- 形成了新的阻塞点或下一步交接点

以下情况可以不写：

- 只有纯读取，且没有修改、没有执行、没有新结论
- 只有纯聊天判断，未进入执行层

## 每条日志的固定模板

每条日志至少包含以下栏目：

1. 本轮目标
2. 执行前已确认事实
3. 实际读取
4. 实际改动
5. 实际执行
6. 当前结果
7. 下一步建议

## latest.md 的作用

`latest.md` 只保留交接最需要的信息，不写成长文。

至少保留：

- 当前项目执行状态
- 最近一次完成了什么
- 当前最关键的下一步
- 新会话接手时建议先读哪些文件

## 真实性要求

- 只写已确认的仓库事实、执行结果和阻塞点。
- 不得把聊天中的猜测写成已执行。
- 不得只在聊天里汇报、不落仓库。
- 如需补写历史日志，只能基于已验证的 git 历史、仓库状态和已确认执行结果回填。
