No-Service Graph Probe 解释页｜这次 Codex 到底验证了什么

这轮是 no_service_graph_probe（无服务图探测）：Codex 没有启动服务，没有安装依赖，也没有真实调用向量库或外部 API。 它做的是用 fake_graph_runner_no_dependency（无依赖假图运行器） 把冻结的契约样例串起来，确认“路由、清洗、检索清单、原文回读、交接、完成真实性检查”这条链路可以被检查。 这不是 runtime（运行时） 接入完成，也不是 LangGraph（图运行框架） 已经可用。 下一步安全动作是 retrieval_cleaning_adapter_probe_after_user_chatgpt_review（用户 / ChatGPT 复审后进入检索与清洗适配探测）。

one_sentence_truth（一句话真相）：这轮证明的是：契约链路能被 fake graph 跑通；没有证明正式 LangGraph runtime 已经可用。

graph_flow_diagram（图流程图）

这张图说明 fake graph 跑过的 7 个节点。它像一次“排练”：排练能证明台词和走位能被检查，但不能证明正式剧场已经开演。

展开看细节：7 个 graph_nodes（图节点）分别检查什么

route_decision_node（路由判断节点）

  • 检查什么: 项目路由、workflow 归位、runtime 是否允许写仓库。
  • 防什么 bug: 防止 LangGraph（图运行框架）替代 Project State Action Router（项目状态动作总控器）。
  • 通过后代表什么: 本轮可以按机制修补流做 no-service probe。
  • 不代表什么: 不代表正式 runtime 已接管项目路由。

cleaning_adapter_node（清洗适配节点）

  • 检查什么: secret_scan_before_ingestion（入库前密钥扫描）、metadata（元数据）、source_path（来源路径）、chunk_id（切片编号）。
  • 防什么 bug: 防止清洗后丢掉来源，导致后面无法回读原文。
  • 通过后代表什么: fixture 里的清洗样例保留了追溯字段。
  • 不代表什么: 不代表真实文档清洗器已经处理过生产数据。

retrieval_manifest_node（检索清单节点）

  • 检查什么: provider（检索来源）、source_path（来源路径）、chunk_id（切片编号）、source_readback_required（必须原文回读）。
  • 防什么 bug: 防止只拿 page_content（正文片段）就当事实。
  • 通过后代表什么: DashVector fixture 能表达完整检索清单。
  • 不代表什么: 不代表已经真实调用 DashVector。

source_readback_node（原文回读节点）

  • 检查什么: 检索命中能否回到当前分支原文件。
  • 防什么 bug: 防止旧摘要、旧分支、旧口径冒充当前事实。
  • 通过后代表什么: 本地 fixture 指向的源文件存在且可读。
  • 不代表什么: 不代表所有未来资料都能自动回读。

retrieval_gap_report_node（检索缺口报告节点）

  • 检查什么: gap_status（缺口状态）是否为 none（无缺口）。
  • 防什么 bug: 防止低置信度、空检索或冲突检索被静默放行。
  • 通过后代表什么: 本轮 fixture 没有制造检索缺口。
  • 不代表什么: 不代表真实 RAG（检索增强生成）永远无缺口。

executor_handoff_node（执行器交接节点）

  • 检查什么: active_write_executor（当前激活写入执行器）仍是 Codex；service 和 memory 都不能写仓库。
  • 防什么 bug: 防止 service（服务）或 runtime memory（运行时记忆）绕过 Codex 写文件。
  • 通过后代表什么: 写入边界在 fixture 中被保留。
  • 不代表什么: 不代表 FastAPI 服务已经可以启动。

completion_truth_check_node（完成真实性检查节点）

  • 检查什么: required_outputs_exist（必交付物存在）、source_readback_passed（原文回读通过）、forbidden_status_promotion_scan（禁止状态推进扫描）。
  • 防什么 bug: 防止技术成功、文件存在、sandbox 成功被包装成最终完成。
  • 通过后代表什么: 本轮 no-service probe 的完成声明有边界。
  • 不代表什么: 不代表内容、声音、视觉、发布状态被推进。

glossary（术语表）

no_service_graph_probe（无服务图探测）: 不启动服务，只用本地脚本和样例验证图节点契约。
fake_graph_runner_no_dependency（无依赖假图运行器）: 不依赖 LangGraph 的模拟执行器，用来验证节点顺序和阻断规则。
LangGraph（图运行框架）: 用于图式 workflow 编排的上游能力，本轮没有实际启用。
runtime（运行时）: 真正执行 workflow 的运行环境；本轮状态是未启用。
service（服务）: 如 FastAPI 后端服务；本轮没有启动，未来也不得直接写仓库。
retrieval_manifest（检索清单）: 记录检索来源、source_path、chunk_id、score 等字段的清单。
source_readback_map（原文回读映射）: 把检索结果映射回仓库原文的证据表。
cleaning_adapter（清洗适配器）: 把源文件安全加载、扫描、规范化、切片并保留元数据的桥接层。
fixture（验证样例）: 用来测试通过或阻断规则的静态样例。
schema（结构契约）: 规定字段、硬规则和阻断条件的结构文件。
DashVector（正式主检索路线）: 当前视频工厂保留的主向量检索路线。
Chroma（沙盒向量库）: 上游本地向量库，只能作为 sandbox / 对照。
RAG（检索增强生成）: 先检索再生成的技术路线；检索结果不是仓库事实。
completion_truth_check（完成真实性检查）: 防止假完成、状态误推进和检索摘要冒充事实的最终检查。
