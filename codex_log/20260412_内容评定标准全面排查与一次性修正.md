## 3. 本轮排查出的 6 类核心问题

### A. 前后打架

涉及文件：

- `project_source/21_topic_selection_and_copywriting_rules.md`
- `project_source/22_copy_mode_routing_rules.md`
- `project_source/25_ai_knowledge_video_value_rules.md`
- `project_source/15_distribution_and_commercialization_rules.md`
- `codex_source/03_research_findings_bridge.md`

主要问题：

- `21 / 22 / 25` 还停在上一轮更泛的内容矩阵，没有把当前更优先的内容路线收口到一致口径。
- `15` 仍以“抖音优先 + 再兼顾小红书 / B站 / 视频号”的旧分发框架表达，和这轮“抖音 + 小红书双线优先”的阶段性建议不够一致。
- `bridge` 里的旧条目仍把“结构 / 开头 / 结尾矩阵”写得过满，容易和本轮更聚焦的阶段性建议打架。

最稳修法：

- 用“当前阶段更适合 / 当前优先 / 当前不优先”收口，不改正式事实文件。
- 把 `21 / 22 / 25 / 15` 的口径改成一套同向表达。
- 把 `bridge` 改成“外部观察 + 建议判断 + 阶段性建议”的桥接写法。

### B. 职责串线

涉及文件：

- `project_source/21_topic_selection_and_copywriting_rules.md`
- `project_source/22_copy_mode_routing_rules.md`
- `project_source/15_distribution_and_commercialization_rules.md`

主要问题：

- `21` 里旧的开头 / 结尾类型矩阵偏多，已经开始吃掉 `22` 的部分职责。
- `22` 里旧的四类内容矩阵过重，把当前阶段真正要解决的“爆款型文案 vs 承接型文案”遮住了。
- `15` 里旧的平台与商业化部分太泛，容易把分发策略、产品路径、平台优先级混在一起。

最稳修法：

- `21` 只保留选题、优先方向、前置锁定问题。
- `22` 负责结构、开头、中段证据、时长和爆款 / 承接的文案路由。
- `15` 负责双线平台建议、30 天测试建议、内容信号到承接路径。

### C. 旧口径残留

涉及文件：

- `project_source/21_topic_selection_and_copywriting_rules.md`
- `project_source/22_copy_mode_routing_rules.md`
- `project_source/15_distribution_and_commercialization_rules.md`

主要问题：

- `21` 没有明确把 `AI 日常工作流分享` 收成当前不优先。
- `22` 仍保留过多旧的四类内容矩阵，弱化了当前阶段更适合的主结构与主开头。
- `15` 仍保留 B站 / 视频号过早进入第一轮主测试面的旧感觉。

最稳修法：

- 把当前优先、不优先、备选路线明确写出来。
- 把旧矩阵降为次要，突出当前阶段真正优先的结构建议。
- 把 B站 / 视频号改为“当前不优先先测的平台”。

### D. 研究吸收错位

涉及文件：

- `project_source/22_copy_mode_routing_rules.md`
- `project_source/25_ai_knowledge_video_value_rules.md`
- `project_source/15_distribution_and_commercialization_rules.md`
- `project_source/08_quality_baseline_and_90_score_rules.md`

主要问题：

- “爆款潜力”和“变现潜力”之前没有强制拆开写。
- `45–55` / `25–30` / `7:3` 没有形成稳定落点。
- “用户真实操作录屏”作为中段强证据的重要性，没有被同时写进文案路由、价值判断和质量边界。

最稳修法：

- `22` 写结构、开头、中段证据、时长和爆款 / 承接的文案路由。
- `25` 专门拆开“更容易出爆”和“更容易变现”。
- `15` 写 30 天测试建议与双线平台策略。
- `08` 只补和质量判断直接相关的阶段性建议，不让它越界成运营文件。

### E. 阶段状态写漂

涉及文件：

- `codex_source/03_research_findings_bridge.md`
- `project_source/15_distribution_and_commercialization_rules.md`

主要问题：

- 旧桥接条目里“正式吸收”的说法过满，容易让人误读成正式事实或市场已验证。
- 平台建议、配比建议、时长建议如果不标注阶段性，很容易被看成长期硬规则。

最稳修法：

- 统一标注：
  - `外部观察`
  - `建议判断`
  - `阶段性建议`
- 明确哪些内容没有升级成正式事实。

### F. 重复与缺口

涉及文件：

- `project_source/21_topic_selection_and_copywriting_rules.md`

- `project_source/25_ai_knowledge_video_value_rules.md`
- `codex_log/latest.md`

主要问题：

- `21` 与 `22` 的一部分旧内容有重复。
- `25` 缺“为什么当前把中段证据看得更重”这一层。
- `latest` 还停在样片素材阻断态摘要，没有记录本轮内容评定标准的全面修正。

最稳修法：

- 删掉重复矩阵和过时表达。
- 在 `25` 补“中段证据为何关键”。
- 用新的 `latest` 收口本轮项目口径同步结果。
