## 当前为什么判素材阻断

### 已成立的部分

- 竖版可读性已经明显改善
- `目标 / 边界 / 验收` 三个方向已经能被观众看到
- 观众能知道你在整理交接槽位

### 仍然不够的部分

- 缺同一任务的“旧状态”硬证据：
  - 不是抽象地“还没压清”
  - 而是能一眼看出：现在这份任务还不能直接交给 AI
- 缺同一任务的“新状态”硬证据：
  - 不是抽象地“已经在整理”
  - 而是能一眼看出：现在已经是可直接交接 / 可直接推进的清单状态
- 当前 `1.mov` 更像同一段聊天录屏的滚动浏览，而不是同一任务的明确前后差值记录

## 补录目标

补一段只服务 `seg02` 的新素材，长度控制在 `9s` 左右，目标只有一个：

**让观众静音也能一眼看懂：同一任务已经从“不能直接交给 AI”变成“压成可直接交接状态”。**

## 补录结构（必须按同一任务连续成立）

### Beat A｜旧状态（约 `1.5–2s`）

必须出现：

- 同一任务的旧版本
- 明显还是模糊 / 混乱 / 不可直接交接
- 画面里最好能看到：
  - 杂糅需求
  - 没分层的输入
  - 没有 `目标 / 边界 / 验收` 的旧状态

禁止出现：

- 只是一屏普通聊天
- 只是一堆字但看不出哪里混乱
- 与后续不是同一任务

### Beat B｜压清动作（约 `4–5s`）

必须出现：

- 光标真的在动
- 同一任务被压成：
  - `目标`
  - `边界`
  - `验收`
- 至少有一个动作是明确可见的：
  - 新建 3 条
  - 把原内容分栏
  - 把原句子挪到三块区域
  - 把旧的混乱输入重排成 3 个槽位

最佳做法：

- 浏览器宽度收窄
- 页面缩放 `125%–140%`
- 去掉没必要的侧栏和大面积空白
- 同一屏内尽量让 `目标 / 边界 / 验收` 同时可见

### Beat C｜新状态（约 `2–3s`）

必须出现：

- 同一任务的“压清后状态”
- 一眼能看出：
  - 已可直接交接
  - 已可直接交给 AI
  - 已可直接推进

最佳画面：

- 三块清单结构稳定停住 `1.5s` 以上
- 或一个已经压成明确模板 / checklist / handoff note 的最终画面

## 必须出现的 UI / 差值

至少要同时出现这 4 件事中的 3 件：

1. 同一任务标题或同一任务上下文
2. 旧状态里没有 `目标 / 边界 / 验收` 三分
3. 压清动作里真的把内容迁移 / 拆分 / 归位到三块
4. 新状态里三块结构稳定停住，能直接看懂“这是交接清单”

## 录制要求

- 录同一任务，不要跳任务
- 不要频繁切标签页
- 不要把系统栏 / 无关窗口当主体
- 鼠标动作要慢，留 `0.5–0.8s` 停顿
- 每个 beat 至少留出可读停顿
- 不要再录成超宽屏主体，尽量把主内容收窄到中间

## 最省时的补录方式

1. 打开同一任务的旧版本或混乱版本
2. 录 `1.5–2s` 停住
3. 开始把它压成 `目标 / 边界 / 验收`
4. 录下光标移动、内容归位、三块成形
5. 在成形后的稳定画面停住 `2–3s`

一句话：

**补的不是“更多操作”，而是同一任务的强前后差值。**
