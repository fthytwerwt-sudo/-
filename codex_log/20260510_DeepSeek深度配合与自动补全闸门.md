本轮目标是把《视频工厂》的 DeepSeek 配合机制从“按需文件地图 / 风险报告”推进到可执行的深度协作流程，并修复 Codex 只完成显性任务、不自动补齐上下游机制链的问题。

## 3. 新增机制

### 3.1 Auto-completion gate（自动补全闸门）

`已确认` 已写入 `codex_source/01_execution_rules.md`。

触发范围包括机制修补、路由修补、项目文件修改、DeepSeek / 多 AI 协作、视频样片 / 成片、文案生产、发布复盘、reference / locked reference / visual route、多文件任务，以及用户要求“完整流程 / 深度配合 / 自动补全 / 不要只做点名任务”的任务。

后续触发后必须输出：

```text
auto_completion_gate:
  triggered:
  reason:
  explicit_user_task:
  implicit_mechanism_chain:
    goal_layer:
    judgment_layer:
    route_layer:
    trigger_layer:
    supply_layer:
    execution_layer:
    feedback_layer:
    validation_layer:
    sync_layer:
  missing_links:
  must_complete_now:
  blocked_if_not_completed:
  auto_completion_plan:
  auto_completion_result:
```

硬边界：

- 只完成用户点名任务，不等于完成。
- 若隐性机制链有缺口，必须补齐或 blocked。
- 若需要 DeepSeek 供料而未触发，不能写完整完成。
- 若供料包生成但 Codex 没读，不能写完整完成。
- 若读供料包但没复核原文件，不能写完整完成。
- 若执行后没有风险复核或日志回流，不能写完整完成。

### 3.2 editing_decision_pack（剪辑决策包）

`已确认` 已新增 DeepSeek action：`editing_decision_pack（剪辑决策包）`。

它用于视频剪辑、录屏放大、卡片插入、中段承载、画面证据链保护等任务，回答：

- 哪里放大。
- 哪里保留原画面。
- 哪里插卡。
- 哪里框选 / 高亮。
- 哪里定格或左右对比。
- 哪里不动以保护证据链。
- 哪些 reference 只继承质量，不照搬流程。
- 哪些剪辑动作会导致 demo 感、说明书感、硬拼接感。

硬边界：

- DeepSeek 不直接剪视频。
- DeepSeek 不直接读取视频、音频、图片或媒体文件。
- DeepSeek 不拍板最终画面好不好。
- DeepSeek / fallback 只基于 Codex 提供的文字化素材样料生成建议。
- Codex 执行前仍必须复核素材证据和原文件。
- 最终内容判断仍由 ChatGPT / 用户完成。

### 3.3 DeepSeek deep collaboration flow（DeepSeek 深度配合流程）

`已确认` 已写入 `codex_source/17_deepseek_supply_controller_protocol.md`。

标准链路：

```text
Codex route_decision（路由判断）
-> Auto-completion gate（自动补全闸门）
-> 判断 supply_required（三卡 / DeepSeek 供料）
-> 生成 supply_request（供料请求任务卡）
-> controller 运行 DeepSeek / fallback
-> 输出 supply_pack（供料包）
-> Codex 读取 supply_pack
-> Codex 复核原文件
-> Codex 执行
-> 执行后 risk_report / after_read_gap 复核
-> latest / dated log 回流
```
