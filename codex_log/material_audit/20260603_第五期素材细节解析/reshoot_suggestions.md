# 补录建议｜第五期素材细节解析

## 如果下一轮要做“候选商品单独立项判断”内容

- `建议` 录一段 30-60 秒的清晰屏幕录制，只展示候选商品判断表：样本数、可用视觉时长、点击 / 咨询 / 成交信号、佣金 / 成本、供应履约风险。
- `建议` 对每个候选商品单独录：商品视觉、平台数据、供应信息、素材来源，避免多个候选在 Finder 中快速切换导致后续无法判断。
- `建议` 对 M04 中关键阈值重新录一版放大画面，或直接整理成表格截图，避免小字阈值只能标 `待验证`。

## 如果下一轮要做“执行流程 / Codex 质量闸门”内容

- `建议` 录制时保留清晰的 task checklist、validator 输出、contact sheet、failed/fixed 对照；每个状态停留 3-5 秒。
- `建议` 单独录一次“render 成功但 visual check 失败”的对照：日志通过、contact sheet 暴露问题、修复后再验证。
- `建议` 对 API / provider / model 只展示脱敏 route，不展示任何 key、token、账号、余额或敏感路径。

## 如果下一轮要做“候选媒体池”内容

- `建议` 每个候选资产录 5-8 秒独立片段，命名时包含 `candidate_id / product_type / source_status / usage_status`。
- `建议` 对可能含真人、婚礼、数字人或品牌包装的素材，提前标注授权状态：`real_person_authorized / synthetic_asset / public_reference_only / internal_only`。
- `建议` 对商品功效类素材增加“不能宣称真实功效，需产品证据”的安全标签。

## 不建议补录的方式

- 不建议继续只录 Finder 快速翻文件；这对素材库存有用，但对 ChatGPT 写判断型内容帮助有限。
- 不建议只录技术 metadata；它只能证明可解码或技术状态，不能证明内容或商业价值。
- 不建议把多个候选商品、多个任务、多个项目状态混在一段长屏录里；后续解析成本会显著增加。
