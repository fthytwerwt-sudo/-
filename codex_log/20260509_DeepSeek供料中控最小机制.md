# DeepSeek 供料中控最小机制

## 1. 本轮定位

- `项目`: 《视频工厂》
- `目标`: 落地 `DeepSeek supply controller（DeepSeek 供料中控）` 的最小可执行版本
- `执行边界`: 不做完整多 agent runtime，不做后台服务，不修改视频机制正文，不修改视频 / 声音 / 发布状态

## 2. 本轮新增内容

- `已确认` 新增 `scripts/deepseek_supply_controller.py（DeepSeek 供料中控脚本）`
- `已确认` 新增 `codex_source/17_deepseek_supply_controller_protocol.md（DeepSeek 供料中控协议）`
- `已确认` 小幅更新 `codex_source/16_deepseek_readonly_explorer_rules.md（DeepSeek 只读探索器规则）`
- `已确认` 小幅更新 `codex_source/00_codex_readme.md（Codex 执行入口）`

## 3. 中控支持的触发原因

controller 当前支持：

- `missing_context`
- `rule_conflict`
- `stale_context_risk`
- `large_context`
- `before_write_gate`
- `after_read_gap`
- `user_explicit_deepseek`

## 4. 中控支持的 action

controller 当前支持：

- `file_map（文件地图）`
- `risk_report（风险报告）`
- `context_summary（上下文摘要）`
- `missing_files（缺失文件）`
- `auto（自动选择上述 action）`

`auto` 不能生成自由任务，只能在前四类 action 中选择。

## 5. 输出回流路径

controller 固定输出到：

- `dist/deepseek_supply_controller/latest_supply_pack.md`
- `dist/deepseek_supply_controller/latest_supply_pack.json`
- `dist/deepseek_supply_controller/latest_supply_manifest.json`

Codex 后续执行必须读取供料包，不能只让供料结果停在日志里。
