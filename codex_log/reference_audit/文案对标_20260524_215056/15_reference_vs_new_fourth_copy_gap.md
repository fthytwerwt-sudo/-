# 15 Reference vs New Fourth Copy Gap｜对标与新第四期话语差异

## Baseline

本报告不读取或修改新第四期最终稿，因为最终稿未锁定。比较对象是当前已知新第四期方向：`Codex 直接操作电脑做选品初筛 -> 候选表 / 云盘表格 / 聊天框结论 -> 告诉用户哪些商品更适合、理由、风险、下一步复查项`。

## Core Gap（一句话）

上一轮我们学到了结构；这一轮更关键的是话语差异：reference 更像“一个人把复杂问题讲给你听”，新第四期下一版如果直接讲 Codex、字段和表格，很容易变成“项目报告”。

## Difference Analysis

1. 结构差异：reference 每段先有问题卡，新第四期容易直接进流程。
2. 话语差异：reference 多用短判断和具体场景，新第四期风险是名词密度太高。
3. 开头冲击差异：reference 先给结果/承诺感，新第四期不能先解释系统能力。
4. 细节进入时机差异：reference 在画面证据出现时讲细节，新第四期也必须让字段跟画面同步。
5. 判断卡差异：reference 用卡片承接判断，新第四期需要“问题 / 判断 / 边界 / 结果”四类卡。
6. 配比差异：reference 是过渡顺口、证明细；新第四期不能整篇都顺口，也不能整篇都像说明书。
7. 句长差异：reference 用短句制造节奏，新第四期需要减少连续长名词串。
8. 口语感差异：reference 常把复杂概念翻成人话，新第四期要把“初筛、候选、风险字段”翻成“先看哪几个、为什么留下、下一步核什么”。
9. 爽点差异：reference 的爽点是复杂事情被拆开；新第四期的爽点应是“从乱翻商品卡变成一张可复查表”。
10. 边界差异：reference 低压收束；新第四期必须明确 AI 初筛不是商业验证。

## Hard Rewrite Instructions

- 不要先解释系统，先抛具体痛点。
- 不要说“商品卡初筛动作”，要说“翻到眼睛花，还不知道哪个该测”。
- 不要平铺字段，要让字段在画面出现时逐个打出来。
- 不要只说“生成表格”，要说“云盘表格里商品名、佣金、风险、下一步都排好了”。
- 不要一直顺口，关键高光要停下来讲画面。
- 不要把 Codex 写成自动赚钱机器，要写成先帮人把复查对象和风险列清。
