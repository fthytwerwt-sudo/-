# 收束与 CTA 库 v1

## 当前主结论

- 当前用户偏好的收束，不是空喊一句结束，而是先把观点收住，再给一个低门槛动作
- 当前项目最适合复用的 CTA，是 `轻行动` 和 `继续探索`
- 强关注、强卖课、强会员导向，目前先不作为默认复用方向

## 使用边界

- 本文件只基于当前“喜欢样本”得出
- 当前属于初步判断，后续可继续补强
- 这里整理的是收束动作和 CTA 类型，不是最终统一口播尾句

## 1. 观点收束型

- 典型动作：
  - 最后把前面的判断压成一句话
  - 告诉观众“所以你现在该怎么理解”
- 样本参考：
  - `喜欢 5`：最后落到“选一个用起来就可以了”
  - `喜欢 6`：最后回到“最好的架构是……”
- 当前项目适合复用：
  - 方法论视频
  - 工具选择视频
  - 项目判断视频

## 2. 低门槛行动型 CTA

- 典型动作：
  - 给观众一个今天就能做的动作
  - 不要求他立刻完成很重的事
- 样本参考：
  - `喜欢 9`：让观众开始动手做一个很小的 AI 项目
  - `喜欢 10`：让观众直接去试一试
- 当前项目最适合复用：
  - `先试一下`
  - `先从一个最小动作开始`
  - `先把今天这一步做掉`

## 3. 继续互动型 CTA

- 典型动作：
  - 邀请观众补充自己的情况
  - 让评论区变成下一轮输入来源
- 样本参考：
  - `喜欢 9`：评论区告诉我你最想让 AI 帮你做什么
  - `喜欢 10`：评论区蹲你们的复盘成果
- 当前判断：
  - 可以用
  - 但更适合自媒体发布场景
  - 当前仓库知识库阶段先保留，不强推

## 4. 强导流 / 强关注 / 强售卖型 CTA

- 样本参考：`喜欢 1 / 3 / 7`
- 当前判断：
  - 这些收束很完整，也很像成熟内容账号
  - 但当前项目阶段先不把它设成默认重用方向
- 原因：
  - 容易把项目表达带到“账号运营话术”
  - 当前更需要内容本身的稳，而不是商业导流的重

## 当前最适合当前项目阶段的收束方式

- 第一优先：`一句话收束观点`
- 第二优先：`给一个最小行动`
- 补充位：`邀请继续探索`
- 暂不默认：`强关注 / 强售卖 / 强会员 CTA`

## 当前一句话理解

- 当前用户喜欢的收束，不是把人推出去，而是把观点收住以后，顺手给他一条路
