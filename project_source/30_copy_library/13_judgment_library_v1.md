# 判断句库 v1

## 当前主结论

- 当前用户偏好的判断句，不是中性复述，而是明显有立场
- 最常见的判断动作，是 `先定性`、`再比较`、`再给选择`
- 更像用户会发的判断句模式，是“像人说话的判断”，不是书面报告腔

## 使用边界

- 本文件只基于当前“喜欢样本”得出
- 当前属于初步判断，后续可继续补强
- 这里整理的是“判断动作”和“句型味道”，不是最终固定模板

## 1. 先把事情定性

- 典型句型：
  - `X 不是 Y，是 Z`
  - `好的架构，都是被业务约束逼出来的`
  - `这不是早期红利，而是……`
- 这类判断的作用：
  - 先把讨论框架钉住
  - 观众会更快知道你站在哪边
  - 后面解释会更顺
- 当前项目最适合复用：
  - 路线判断
  - 方法选择
  - 内容取舍

## 2. 把对象压成一眼能懂的比较

- 典型动作：
  - 把工具写成“像什么样的人”
  - 把多个选项压成一组差异明显的角色
  - 把复杂对象变成“谁更适合什么”
- 样本参考：`喜欢 5`
- 这类判断的作用：
  - 非常口语化
  - 很适合做工具评测、方案比较
  - 能快速建立用户感受，不只是信息点

## 3. 给直接建议，不装客观中立

- 典型句型：
  - `懒得想就选……`
  - `先上线……再考虑……`
  - `工具嘛，就是用得顺手才是你的`
- 这类判断的作用：
  - 让内容有“我替你做了第一轮判断”的感觉
  - 观众更容易记住结论
- 当前项目最值得保留：
  - 给默认推荐
  - 给当前阶段建议
  - 给最小行动方案

## 4. 带风险提醒的判断

- 典型动作：
  - 先承认一个东西有优点
  - 但马上补一句边界
  - 不让结论变成无脑吹
- 样本参考：
  - `喜欢 5` 里的“聊挺开心的，但最好不要当工作参考”
  - `喜欢 6` 里的“能用简单方案解决的问题，用复杂架构只会增加成本和风险”
- 这类判断的作用：
  - 会让判断更像真人经验
  - 比纯正向夸赞更可信

## 当前最像用户口吻的判断句模式

- `一句话先拍板`
- `然后用一个更生活化的比喻或比较压实`
- `最后给一个能执行的建议`

## 当前一句话理解

- 当前用户喜欢的不是“信息对”，而是“判断清楚”
- 真正能留下来的判断句，往往都能让人一听就知道：你是怎么想的，我现在该怎么做
