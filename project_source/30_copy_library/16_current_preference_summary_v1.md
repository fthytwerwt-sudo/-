# 当前偏好总结 v1

## 当前主结论

- 只看这 10 条“喜欢样本”，已经能先看出一个很明显的方向：用户偏好 `有判断、有人味、有动作` 的表达
- 当前最稳的主线，不是冷资料，也不是纯营销，而是 `先给结论 -> 再讲为什么 -> 最后给动作`
- 这已经足够支撑第二层文案知识库
- 但它还只是第一轮初拆，不是最终风格定稿

## 当前已经能先确定的

- 用户喜欢 `结论先行`
  - 很多样本第一句就拍板，不先铺背景
- 用户喜欢 `像人在说话`
  - 不是说明书腔，也不是纯报告腔
- 用户喜欢 `有主观看法`
  - 会直接推荐、直接比较、直接下判断
- 用户喜欢 `解释里带动作`
  - 不只是告诉你是什么，还会告诉你现在该怎么做
- 用户喜欢 `口语桥很顺`
  - “那问题来了”“那怎么办”“举个例子” 这种桥很常见

## 当前还不能定死的

- `只喜欢哪一段` 还没补
  - 所以现在没法精确知道是喜欢整条，还是只喜欢其中某几个动作
- 还不能定死用户对 `夸张度` 的容忍上限
  - `喜欢 2 / 8` 有很强的情绪和夸张，但不代表这就是默认主线
- 还不能定死用户对 `长叙事` 的真实偏好
  - `喜欢 3` 很强，但目前更像“可借部件”，还不能直接当主模板
- 还不能定死用户对 `CTA 强度` 的最终偏好
  - 现在只能确认轻 CTA 更稳，不能确认重导流是否被长期喜欢

## 当前从喜欢样本里最值得先用的东西

- 开头：
  - 结论先拍桌
  - 结果先交代
- 中段：
  - 判断句先压一下
  - 再用比较、举例、编号把逻辑展开
- 过渡：
  - 口语化问题推进
  - 编号展开
  - 举例切换
- 收尾：
  - 一句话收束
  - 给一个低门槛动作

## 后面最值得补的输入

- `只喜欢哪一段`
  - 这是后面最值钱的补料
- 口语原话 / 真实口播
  - 能帮助区分“文案好看”和“说出来真顺”
- 不喜欢样本深拆
  - 能帮助把当前判断的边界压实
- 当前项目里真正过线的成片口播文案
  - 能帮助判断“喜欢样本”和“可用项目表达”之间哪里已经重合，哪里还没重合

## 当前一句话理解

- 现在这套第二层知识库，已经能告诉后续 ChatGPT / Codex：
  - 什么样的表达更像用户会喜欢
  - 什么样的部件可以先拿来用
  - 什么样的东西现在还只能先观察
- 但必须继续记住：
  - `只喜欢哪一段` 还没补
  - 所以当前只是第一轮初拆，不是最终风格定稿
