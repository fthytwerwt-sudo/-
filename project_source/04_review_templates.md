# 项目回审模板

## 1. 回审总原则

本项目的回审不是为了打分，而是为了定位问题层级。

每一轮回审都必须优先回答：

1. 问题主要出在哪一层
   - 场景层
   - 结构层
   - 心理机制层
   - 表现层
   - 配音 / 字幕节奏层
2. 当前最优先应该只改哪一个点

固定原则：

- 回审不是泛泛提建议
- 每轮只允许收成一个最优先改点
- 如果问题很多，也要先找“改了最值”的那一个
- 回审必须显式接入“心理机制是否接住人”的诊断层

---

## 2. 最小可用回审模板

适用场景：

- 刚出一个最小版本
- 想快速知道最先改哪里

可直接使用模板：

```text
请帮我对这条 AI 短视频做“最小可用回审”。

你不要打分，只回答下面 5 个问题：
1. 这条内容当前最主要的问题出在哪一层：
   - 场景
   - 结构
   - 心理机制
   - 画面表现
   - 字幕与配音节奏
2. 哪一句或哪一页最拖后腿
3. 观众最可能在哪一刻失去兴趣
4. 这条内容有没有接住“身份代入 / 认知减负 / 最小行动”中的至少一个
5. 如果这轮只能改一个点，最应该改什么，为什么

输出时只保留一个最高优先级改点，不要列一长串建议。
```

## 3. 快速初审模板

适用场景：

- 有脚本或视频初稿
- 想快速判断是否值得继续打磨

可直接使用模板：

```text
请对这条 AI 视频做快速初审。

请按下面结构输出：
1. 一句话结论：值不值得继续做
2. 当前主要问题层级
3. 开头是否有效
4. 中段是否像说明书
5. 结果是否可见
6. 心理机制有没有接住人：
   - 身份命中
   - 信息缺口
   - 自我效能感
   - 最小行动
7. 这轮只改一个点的话改哪里

要求：
- 不打分
- 不平均发力
- 只抓最高优先级问题
```

## 4. 正式交付审模板

适用场景：

- 准备交付“可发布测试”版本前

可直接使用模板：

```text
请把这条 AI 短视频按“正式交付审”来审。

必须逐条判断：
1. 脚本是否成立：像不像短视频，而不是说明书或项目汇报
2. 开头是否有效：前几秒能不能让人知道值不值得看
3. 前后变化是否可见：不是口头喊变化
4. 结构是否适配当前场景
5. 心理机制是否真正接住人，而不是空喊痛点
6. 画面是否还像程序 demo
7. 字幕和配音是否协同，不出戏
8. 结尾是否有清楚落点

最后只输出：
- 是否达到“可发布测试”水位
- 如果还没达到，唯一最优先改点是什么
```

## 5. 文案专项回审模板

适用场景：

- 主要问题怀疑在脚本文案

可直接使用模板：

```text
请只从“文案层”审这条视频脚本。

重点检查：
1. 开头是不是废话开头
2. 中段是不是说明书腔
3. 结果是不是说了等于没说
4. 句子是不是太书面
5. 有没有一句能真正命中目标用户身份
6. 有没有给到最小行动或掌控感

输出格式：
- 当前最拖后腿的句子
- 为什么拖后腿
- 应该改成什么方向
- 这轮只改这一处，不扩散
```

---

## 6. 画面专项回审模板

适用场景：

- 主要问题怀疑在 PPT 信息卡、镜头组织、素材接法

可直接使用模板：

```text
请只从“画面层”审这条视频。

重点检查：
1. 画面是不是太像程序 demo
2. 页面层级是否清楚
3. 每页信息量是否过大或过平
4. 页与页之间是否有变化感
5. 辅助素材是否加分还是抢戏
6. 画面有没有承接脚本里的前后变化

输出时只回答：
- 画面层当前最大问题是什么
- 具体体现在哪一页或哪一段
- 这一轮只优先改哪个画面问题
```

---

## 7. 字幕与配音节奏专项回审模板

适用场景：

- 主要问题怀疑在节奏和听感

可直接使用模板：

```text
请只从“字幕与配音节奏层”审这条视频。

重点检查：
1. 配音是不是太像系统播报
2. 停顿是否合理
3. 字幕分段是否利于理解
4. 字幕出现与页面切换是否协调
5. 是否有哪一段显得太赶或太平
6. 节奏有没有帮助观众处理信息，而不是增加负担

输出时只保留：
- 当前节奏层最关键问题
- 它最明显出现在什么时间段
- 这轮优先怎么改
```

## 8. 场景 / 结构回审模板

适用场景：

- 怀疑问题根源不是一句话写得不好，而是场景判断或结构选错

可直接使用模板：

```text
请只从“场景与结构匹配度”审这条视频。

请判断：
1. 这条内容实际上属于哪个场景模式
2. 当前结构是不是选对了
3. 如果结构没选对，更适合换成哪一种
4. 当前内容最像：
   - 项目讲解
   - 方法分享
   - 学习实操
   - 案例拆解
   - 其他哪类
5. 结构有没有把最重要的变化讲出来

最后只输出一个结论：
- 这条视频要不要换结构
- 如果要换，换成什么结构最值
```

## 9. 改后复审模板

适用场景：

- 已按上一轮改过
- 想确认有没有真正往前走

可直接使用模板：

```text
请对这条视频做“改后复审”。

背景：
- 上一轮最优先改点是：[贴这里]
- 这一轮已经改完

请只回答：
1. 这个问题有没有真正改善
2. 改完后有没有引入新的更大问题
3. 现在新的最高优先级问题是什么

注意：
- 不回到大而全点评
- 只判断这次修改值不值
```

---

## 10. 用户只说“不对 / 有点空 / 差点意思 / 不像人话”时的诊断模板

适用场景：

- 用户反馈模糊
- 需要先把模糊感受压成问题层级

可直接使用模板：

```text
用户的反馈是：
“不对 / 有点空 / 差点意思 / 不像人话”

请不要泛泛解释，请你把这个模糊反馈拆成最可能的 4 类问题：
1. 场景没判准
2. 结构不成立
3. 心理机制没接住人
4. 表达或表现太像说明书 / demo

请对当前内容逐项判断：
- 哪一类最像真实问题
- 证据是什么
- 如果这轮只能改一个地方，最值的是哪一处

输出时只保留一个最优先诊断结论。
```

---

## 11. 心理机制诊断附加层

无论使用哪种回审模板，都建议额外补这一段：

```text
请额外判断：
这条内容有没有至少接住以下机制中的一个：
- 身份命中 / 自我代入
- 认知减负 / 处理流畅
- 信息缺口
- 自我效能感
- 最小行动
- 掌控感 / 胜任感

如果没有接住，请明确指出最值得补哪一个，而不是全补。
```

---

## 12. 结论

本项目的回审体系核心不是“越细越好”，而是：

- 先定位问题层级
- 再只抓一个最优先改点
- 并且强制把“心理机制是否接住人”纳入诊断层

这样回审才不会越审越散。
