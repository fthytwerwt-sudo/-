# 项目回审模板

## 1. 回审总原则

本项目的回审不是为了打分，而是为了定位问题层级、抓最高优先级改点，并统一质量口径。

每一轮回审都必须优先回答：

1. 当前主要问题出在哪一层
   - 场景层
   - 结构层
   - 展示路由层
   - 心理机制层
   - 画面表现层
   - 字幕 / 配音节奏层
2. 当前最优先应该只改哪一个点
3. 当前问题是不是已经触发一票否决项
4. 当前问题更像内容不过线，还是展示路由选错

固定原则：

- 回审不是泛泛提建议
- 每轮只允许收成一个最高优先级改点
- 如果问题很多，也要先找“改了最值”的那一个
- 当前不要拿旧 demo 当质量样片对照
- 命中质量判断时，默认回到项目内部质量基线，而不是假设存在平台官方 90 分规则
- 展示路由必须成为正式变量，不能只在总结里顺嘴带过

## 2. 最小可用回审模板

适用场景：

- 刚出一个最小版本
- 想快速知道最先改哪里

可直接使用模板：

```text
请帮我对这条 AI 短视频做“最小可用回审”。

你不要打分，只回答下面 5 个问题：
1. 这条内容当前最主要的问题出在哪一层：
   - 场景
   - 结构
   - 展示路由
   - 心理机制
   - 画面表现
   - 字幕与配音节奏
2. 哪一句、哪一页或哪一段最拖后腿
3. 观众最可能在哪一刻失去兴趣
4. 这条内容有没有接住“身份代入 / 认知减负 / 最小行动”中的至少一个
5. 如果这轮只能改一个点，最应该改什么，为什么

输出时只保留一个最高优先级改点，不要列一长串建议。
```

## 3. 快速初审模板

适用场景：

- 有脚本或视频初稿
- 想快速判断是否值得继续打磨

可直接使用模板：

```text
请对这条 AI 视频做快速初审。

请按下面结构输出：
1. 一句话结论：值不值得继续做
2. 当前主要问题层级
3. 开头是否有效
4. 中段是否像说明书
5. 结果是否可见
6. 配音与字幕是否先天就会拖后腿
7. 这轮只改一个点的话改哪里

要求：
- 不打分
- 不平均发力
- 只抓最高优先级问题
```

## 4. 正式交付审模板

适用场景：

- 准备交付“可发布测试”版本前

可直接使用模板：

```text
请把这条 AI 短视频按“正式交付审”来审。

必须逐条判断：
1. 脚本是否成立：像不像短视频，而不是说明书或项目汇报
2. 开头是否有效：前 3 秒能不能让人知道值不值得看
3. 前后变化是否可见：不是口头喊变化
4. 结构是否适配当前场景
5. 展示路由是否选对：当前更该 PPT、真人、录屏、案例图还是混合
6. 画面是否还像程序 demo
7. 配音是否还像系统播报
8. 字幕和配音是否协同，不出戏
9. 结尾是否有清楚落点

最后只输出：
- 是否达到“可发布测试”水位
- 如果还没达到，唯一最优先改点是什么
```

## 5. 抖音 90 分水位回审模板（项目内部质量简称）

这份模板专门用于统一“90 分标准”的项目内回审口径。

必须先明确三件事：

1. 这不是官方标准
2. 它是项目内部质量简称
3. 它适用于 AI / 知识类 / 无人出镜短视频

### A. 必须过线项

- 配音不再明显像系统播报
- 字幕与配音基本同步
- 开头 3 秒有效
- 内容不像说明书
- 画面不只是静态轮播
- 前后变化能被看懂
- 结尾有落点

### B. 一票否决项

- 明显机械配音
- 字幕和配音明显不同步
- 全程像 PPT 轮播 / demo 演示
- 开头 3 秒没有钩子
- 内容高度模板化、像批量生产
- 画面明显低质或 AI 瑕疵
- 看完整条仍不知道表达什么

### C. 加分项

- 配音有轻微情绪起伏
- 有真实案例或数据支撑
- 信息卡 / 字体 / 层次更接近原生短视频
- 节奏不是匀速播报
- 看起来像真人真的会发，而不是工具演示稿

可直接使用模板：

```text
请按“抖音 90 分水位回审模板（项目内部质量简称）”来审这条视频。

先明确：
- 这不是平台官方标准
- 这是项目内部对 AI / 知识类 / 无人出镜短视频质量水位的简称

请按三部分输出：
1. 必须过线项里，目前已经过了哪些、没过哪些
2. 有没有命中一票否决项；如果有，先只指出最致命的一条
3. 当前最明显的加分项有没有出现

最后只给一句结论：
- 当前是否已接近项目内部“90 分水位”
- 如果还没接近，最优先补哪一个点
```

## 6. 文案专项回审模板

适用场景：

- 主要问题怀疑在脚本文案

可直接使用模板：

```text
请只从“文案层”审这条视频脚本。

重点检查：
1. 开头是不是废话开头
2. 中段是不是说明书腔
3. 结果是不是说了等于没说
4. 句子是不是太书面
5. 有没有一句能真正命中目标用户身份
6. 有没有给到最小行动或掌控感

输出格式：
- 当前最拖后腿的句子
- 为什么拖后腿
- 应该改成什么方向
- 这轮只改这一处，不扩散
```

## 7A. 白领向纯 PPT 次级支路风格回审模板

适用场景：

- 当前评审对象是 pure PPT / 信息卡次级支路
- 想判断“白领风是否成立”
- 想确认内容有没有滑回培训班感、模板货感、学生作业感或低质 AI 感

可直接使用模板：

```text
请按《project_source/17_white_collar_ppt_style_rules.md》的口径，
对这条纯 PPT / 信息卡视频做“白领向母版风格回审”。

必须逐条判断：
1. 这条纯 PPT 视频是否显得专业、可信、体面
2. 是否出现培训班感 / 模板货感 / 学生作业感
3. 是否做到“信息高效、视觉克制、语气平等”
4. 是否因为视觉或文案问题滑回低质 AI 感
5. 如果这轮只能改一个点，优先改哪一个最值

输出时只保留：
- 一句话结论：白领风成立还是没成立
- 最拖后腿的一页 / 一段 / 一个信号
- 这轮唯一最优先改点
```

## 7B. 正式默认主线回审模板（人物 + 自录素材 + 少量 PPT / 图片）

适用场景：

- 当前评审对象是正式默认主线
- 想判断人物、自录素材、少量 PPT / 图片的分工是否成立
- 想确认有没有滑回 demo 感或 AI avatar 风险

可直接使用模板：

```text
请按“正式默认主线回审模板”来审这条视频。

必须逐条判断：
1. 开头的人物判断有没有真的接住人，而不是只是出个人
2. 中段自录素材 / 真实录屏有没有承担过程证据，而不是装饰 cutaway
3. 少量 PPT / 图片有没有只做辅助显影，没有重新抢回主承载
4. 全片是不是更像真人会发，而不是更像工具 demo
5. 有没有出现以下高风险信号：
   - 口型与语音不同步
   - 性别与文案不匹配
   - 动作与文案不匹配
   - 视觉人物与内容判断不一致
6. 如果这轮只能改一个点，最值的是人物判断、自录证据，还是辅助信息卡

输出时只保留：
- 当前默认主线是否成立
- 最拖后腿的一段
- 这轮唯一最优先改点
```

## 9. 场景 / 结构回审模板

适用场景：

- 怀疑问题根源不是一句话写得不好，而是场景判断或结构选错

可直接使用模板：

```text
请只从“场景与结构匹配度”审这条视频。

请判断：
1. 这条内容实际上属于哪个场景模式
2. 当前结构是不是选对了
3. 如果结构没选对，更适合换成哪一种
4. 当前内容最像：
   - 项目讲解
   - 方法分享
   - 学习实操
   - 案例拆解
   - 其他哪类
5. 结构有没有把最重要的变化讲出来

最后只输出一个结论：
- 这条视频要不要换结构
- 如果要换，换成什么结构最值
```

## 10. 展示路由专项回审模板

适用场景：

- 已怀疑当前问题在“展示路由是否选错”
- 想判断下一轮是继续同一路由，还是切路由

可直接使用模板：

```text
请只从“展示路由是否选对”来审这条视频。

必须输出：
1. 本条视频主路由策略是什么
2. 本条 block 路由表是什么
3. 为什么这样路由
4. 当前是否存在“展示路由错误”
5. 当前问题更像内容不过线，还是路由选错
6. 下一轮是否继续同一路由，还是切路由

并逐条判断：
- 这条本来该 PPT 却用了真人？
- 本来该真人却用了 PPT？
- 本来最适合混合，却被压成单一路线？

最后只保留一个结论：
- 这一轮最值的动作，是继续当前路由还是切路由
```

## 11. 改后复审模板

适用场景：

- 已按上一轮改过
- 想确认有没有真正往前走

可直接使用模板：

```text
请对这条视频做“改后复审”。

背景：
- 上一轮最优先改点是：[贴这里]
- 这一轮已经改完

请只回答：
1. 这个问题有没有真正改善
2. 改完后有没有引入新的更大问题
3. 现在新的最高优先级问题是什么

注意：
- 不回到大而全点评
- 只判断这次修改值不值
```

## 12. 用户只说“不对 / 有点空 / 差点意思 / 不像人话”时的诊断模板

适用场景：

- 用户反馈模糊
- 需要先把模糊感受压成问题层级

可直接使用模板：

```text
用户的反馈是：
“不对 / 有点空 / 差点意思 / 不像人话”

请不要泛泛解释，请你把这个模糊反馈拆成最可能的 4 类问题：
1. 场景没判准
2. 结构不成立
3. 心理机制没接住人
4. 表达或表现太像说明书 / demo

请对当前内容逐项判断：
- 哪一类最像真实问题
- 证据是什么
- 如果这轮只能改一个地方，最值的是哪一处

输出时只保留一个最优先诊断结论。
```

## 13. 心理机制诊断附加层

无论使用哪种回审模板，都建议额外补这一段：

```text
请额外判断：
这条内容有没有至少接住以下机制中的一个：
- 身份命中 / 自我代入
- 认知减负 / 处理流畅
- 信息缺口
- 自我效能感
- 最小行动
- 掌控感 / 胜任感

如果没有接住，请明确指出最值得补哪一个，而不是全补。
```

---

## 14. 结论

本项目的回审体系核心不是“越细越好”，而是：

- 先定位问题层级
- 再抓唯一最优先改点
- 命中质量问题时回到项目内部质量基线
- 命中展示问题时把路由当成正式变量来审
- 明确区分“必须过线项”“一票否决项”“加分项”

这样回审才不会继续把 demo 身份、质量样片和平台规则混掉。
