# 结果反推表 / 诊断逻辑表

## 1. 文件定位
本文件用于把“结果现象”翻译成“最可能问题层”。

注意：
它不是绝对因果表，而是优先排查表。

也就是说：
- 不负责给你一锤定音
- 负责告诉你先查哪一层最值

---

## 2. 标准诊断格式
以后默认按这个格式写：

- 结果现象：
- 最可能问题层：
- 次可能问题层：
- 优先排查变量：
- 当前判断状态：

---

## 3. 常用反推表

| 结果现象 | 最可能问题层 | 次可能问题层 | 优先排查变量 | 当前状态默认 |
|---|---|---|---|---|
| 播放低 | 选题层 | 表现层 | 标题 / 开头承诺是否打到人 | 待验证 |
| 前 3 秒掉得厉害 | 表现层 | 选题层 | 开头钩子类型 / 首屏信息 | 部分成立 |
| 前段留得住，中段明显掉 | 结构层 | 表达层 | 中段是否变成说明书、有没有推进感 | 部分成立 |
| 完播不差，但点赞低 | 表达层 | 选题层 | 语言是否太平、是否缺情感落点 | 部分成立 |
| 点赞有，但收藏低 | 选题层 | 结构层 | 内容有没有可复用价值、是不是只让人点头没让人想留 | 部分成立 |
| 收藏高，但转化低 | 结果层 | 表达层 | 结尾动作设计是否清楚、是否给到最小行动 | 部分成立 |
| 评论多，但停留差 | 结构层 | 选题层 | 是否有争议但内容本身不够吸引、节奏是否拖 | 部分成立 |
| 完播不差，但没人行动 | 结果层 | 表达层 | 结尾动作、掌控感、自我效能感 | 部分成立 |
| 各项都平平 | 选题层 | 表现层 | 切口是否太泛、开头是否不够强 | 待验证 |

## 4. 当前项目口径下，优先按什么层去看
### 选题层
优先看：
- 播放
- 点击 / 首屏吸引
- 收藏
- 用户是否一眼知道“这条和我有关”

### 结构层
优先看：
- 完播率
- 平均观看时长
- 中段流失点
- 前后变化是否真的立住

### 表达层
优先看：
- 点赞
- 评论里的“太空 / 太硬 / 太像说明书”
- 你自己回看时最别扭的句子

### 表现层
优先看：
- 开头 3 秒
- 配音自然度
- 字幕跟读感
- 画面 demo 感
- 切页与停顿是否协调

### 结果层
优先看：
- 私信 / 咨询 / 转粉 / CTA 是否发生
- 内容有价值，但为什么没人行动

---

## 5. 不能写成死结论的东西
以下都不要直接写成绝对因果：

- 播放低 = 选题差
- 收藏低 = 没价值
- 点赞低 = 表达差
- 转化低 = 结尾不行

更稳的写法是：
- 最可能
- 次可能
- 优先排查
- 当前状态

---

## 6. 一句话规则
结果不是答案本身，结果只是证据。
真正要问的是：
**这条结果现象，最像提示我下一轮该先查哪一层、先改哪个变量。**
