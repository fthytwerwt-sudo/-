# 24｜人物 + 自录素材 + 少量 PPT 路由规则（Human + Self Footage + Light PPT Routing Rules）

## 1. 文件定位
本文件是《视频工厂》当前新版主线的锚点文件。

它解决的是：
- 为什么默认主线要从 pure PPT / AI 人物风险路线切到这条新路线
- 人物、自录素材、少量 PPT / 图片各自应该承担什么
- 结构为什么必须跟着文案走
- 人物为什么有时只需要 1 次，有时需要 2 次
- 当前 talking avatar / 数字人口播为什么降级为非默认路线

它不负责：
- 具体某条视频的执行单
- 平台分发策略
- 商业化承接
- 代码实现细节

一句话：

**本文件负责把当前默认主线正式写死为“人物负责信任，自录素材负责主体，少量 PPT 负责总结”，并把这套路由写成可执行规则。**

---

## 2. 当前一句话总判断
当前知识类 / AI 类短视频更稳的主承载路线是：

**人物建立信任和判断，自录素材承担主体推进和过程证据，少量 PPT / 图片负责关键词显影、结构整理和结尾总结。**

这条路线当前优于以下两种默认做法：
- 纯 PPT / 信息卡全片承载
- AI talking avatar / 数字人口播全片承载

---

## 3. 为什么当前要切这条主线
原因不是“更好看”这么简单，而是更稳地解决过去已经暴露出来的失败点：

### 3.1 pure PPT / 信息卡容易累积的问题
- 容易 demo 感
- 容易说明书感
- 容易只有判断，没有过程证据
- 容易像课件，不像真人会发

### 3.2 AI talking avatar / 数字人口播当前暴露的问题
- 口型和语音对不上
- 性别和文案不匹配
- 动作和文案不匹配
- 视觉人物和内容判断不一致
- 画面容易假、容易出戏

### 3.3 新路线为什么更稳
- 人物解决“谁在说、为什么可信”
- 自录素材解决“我真的做过、这是过程证据”
- PPT / 图片解决“我怎么帮你快速看懂和记住”

---

## 4. 默认职责分工

## 4.1 人物负责什么
人物默认只承担这些职责：
- 开头进入感
- 信任建立
- 关键判断
- 转折句
- 结尾收束
- 最小行动

人物默认不承担：
- 中段大部分主体推进
- 全程解释型口播
- 代替真实过程证据

---

## 4.2 自录素材负责什么
用户自己的真实录制素材默认承担：
- 主体信息推进
- 操作过程
- 真实工作片段
- 过程证据
- 前后差异
- 现场感

自录素材默认不只是：
- 装饰 B-roll
- 背景切片
- 为了“显得丰富”而乱塞的碎片

---

## 4.3 少量 PPT / 图片负责什么
PPT / 图片默认承担：
- 关键词显影
- 结构整理
- 并列信息分布式总结
- 结果句 / 数字句强调
- 结尾总结

PPT / 图片默认不承担：
- 整条主体叙事
- 中段大篇幅讲道理
- 重新把视频拉回培训课件感

---

## 5. block 级路由原则
当前结构不能预设模板，而要按 block 判断。

默认顺序：

1. 先拆 block  
2. 再给每个 block 定职责  
3. 再决定这个 block 更需要先接住什么  
4. 最后才决定由谁承担

### 5.1 常见 block 职责
- 信任建立
- 问题 / 痛点激发
- 过程演示
- 关键判断 / 转折
- 分布式信息整理
- 结尾收束 / CTA

### 5.2 默认承载映射
- 信任建立 → 人物
- 痛点激发 → 人物或人物 + 素材
- 过程演示 → 自录素材
- 关键判断 / 转折 → 人物优先
- 分布式信息整理 → PPT / 图片
- 结尾收束 / CTA → 人物或总结卡

### 5.3 硬规则
- 人物次数是路由结果，不是前提
- 中段主体默认优先给自录素材
- PPT / 图片默认只做辅助整理与总结
- 不允许为了“更完整”把三种承载都塞满

---

## 6. 1 次人物 vs 2 次人物判断标准

## 6.1 更适合 1 次人物的情况
- 内容逻辑线很清楚
- 中段素材本身就能撑住主体
- 这条更像“看过程、看操作、看案例”
- 结尾用一张总结卡就能收住

典型模式：
- 开头人物
- 中段素材主体
- 结尾总结卡

---

## 6.2 更适合 2 次人物的情况
- 中段存在关键判断转折
- 素材之间有断裂，需要人物来接缝
- 结尾更适合用人物做人设收束
- 这条内容不只是“看过程”，还要“听判断”

典型模式：
- 开头人物
- 中段素材主体
- 中段或后段再来一次人物判断 / 收束
- 结尾总结卡或一句 CTA

---

## 6.3 不要预设的事
以下当前都不允许预设：
- “所有视频都只要 1 次人物”
- “所有视频都必须 2 次人物”
- “只要用了人物，就一定更高级”
- “有素材了，人物就完全不需要”

---

## 7. talking avatar / 数字人口播的当前正式口径
当前必须写死：

- talking avatar / 数字人口播不是默认主线
- 它当前只能是可选支线或待验证支线
- 若再次出现口型、语音、性别、动作、文案不匹配，应优先按一票否决风险理解
- 不允许为了“省事”把它重新写回默认主承载

---

## 8. 当前质量判断重点
当一条视频走这条新主线时，默认优先看：

1. 开头人物是不是值  
2. 中段素材是不是稳  
3. PPT / 图片是不是清楚但不过课件  
4. 整条是不是 still 像真人会发  
5. 有没有硬拼接感  
6. 有没有说明书感或 demo 感

---

## 9. 当前最常见失败模式
- 人物太多，拖节奏
- 人物有出场，但没有承担信任或判断
- 自录素材太碎，主体不稳
- PPT / 图片太重，重新滑回课件
- 结构没有跟着文案走，变成硬拼接
- 为了“更娱乐”乱加花活，反而破坏专业感

---

## 10. 当前一句话规则
**当前默认主线不是 pure PPT，也不是 AI 人物全片承载，而是：人物负责信任和判断，自录素材负责主体推进和过程证据，少量 PPT / 图片负责整理和总结；结构跟着文案走，人物次数由 block 路由决定。**
