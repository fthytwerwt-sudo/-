# 24｜API 真人 + 本地素材 + 少量 PPT / 图片主线规则

## 1. 文件定位

本文件是《视频工厂》当前正式默认主线的锚点文件。

它回答：

- 当前为什么切到这条主线
- 人物负责什么
- 用户真实录制素材负责什么
- 少量 PPT / 图片负责什么
- 结构为什么跟着文案走
- 人物出现 1 次还是 2 次，为什么是 block 路由结果
- pure PPT / 信息卡为什么降级为非默认路线
- AI talking avatar / 数字人口播为什么降级为非默认路线

它不负责：

- 单条视频执行单
- 样片验收结论
- 代码实现

## 2. 当前为什么切到这条主线

当前切到这条主线，不是为了“看起来更热闹”，而是为了同时解决以下问题：

### 2.1 pure PPT / 信息卡全片承载的常见失败点

- 容易 demo 感
- 容易说明书感
- 容易只有判断，没有过程证据
- 容易像课件，不像真人会发

### 2.2 AI talking avatar / 数字人口播的常见失败点

- 口型和语音对不上
- 性别和文案不匹配
- 动作和文案不匹配
- 视觉人物和内容判断不一致
- 画面容易假，容易出戏

### 2.3 这条主线为什么更稳

- API 真人承担“谁在说、为什么可信”
- 用户本地素材承担“我真的做过、这是过程证据”
- 少量 PPT / 图片承担“我怎么帮你快速看懂和记住”

## 3. 三类承载各自负责什么

### 3.1 API 真人负责什么

API 真人默认负责：

- 开头进入感
- 信任建立
- 关键判断
- 转折句
- 结尾收束
- 最小行动

API 真人默认不负责：

- 中段大部分主体推进
- 全程解释型口播
- 代替真实过程证据

### 3.2 用户本地录制素材负责什么

用户本地录制素材默认负责：

- 主体信息推进
- 操作过程
- 真实工作片段
- 过程证据
- 前后差异
- 现场感

它默认不是：

- 装饰 B-roll
- 为了显得丰富而乱塞的碎片

### 3.3 少量 PPT / 图片负责什么

少量 PPT / 图片默认负责：

- 关键词显影
- 结构整理
- 结果句 / 数字句强调
- 并列信息的分布式总结
- 结尾总结卡

它默认不负责：

- 整条主体叙事
- 中段大篇幅讲道理
- 把视频重新拉回培训课件感

## 4. 结构为什么跟着文案走

当前硬规则 `已确认`：

- 先拆文案 block
- 再判断每个 block 的职责
- 再决定每个 block 由谁承担

也就是说：

- 结构跟着文案走
- 不跟着预设模板走
- 不跟着“真人一定更高级”走
- 不跟着“PPT 一定更稳”走

## 5. 为什么 1 次人物 vs 2 次人物是 block 路由结果

### 更适合 1 次人物的情况

- 内容逻辑线已经很清楚
- 中段素材本身就能撑住主体
- 结尾用总结卡就能收住

常见模式：

- 开头人物
- 中段素材主体
- 结尾总结卡

### 更适合 2 次人物的情况

- 中段存在关键判断转折
- 素材之间有断裂，需要人物来接缝
- 结尾更适合用人物做人设收束

常见模式：

- 开头人物
- 中段素材主体
- 后段再来一次人物判断或收束
- 结尾总结卡或一句 CTA

### 当前不允许预设的事

- “所有视频都只要 1 次人物”
- “所有视频都必须 2 次人物”
- “只要用了人物，就一定更高级”
- “有素材了，人物就完全不需要”

## 6. pure PPT / 信息卡为什么降级为非默认路线

当前 pure PPT / 信息卡没有被删除，但已经降级为非默认路线。

原因 `已确认`：

- 它更适合承担结构化整理与总结
- 它不再适合默认承担整条主体叙事
- 一旦整条默认压回 pure PPT，更容易重新滑回 demo 感、说明书感、课件感

当前定位：

- 只保留为次级支路
- 只在 block 职责确实更适合结构整理时使用
- 不再把它写成全项目当前默认主线

## 7. AI talking avatar / 数字人口播为什么降级为非默认路线

当前 talking avatar / 数字人口播也没有被彻底删除，但已经降级为非默认路线。

原因 `已确认`：

- 它当前稳定性和可信度风险更高
- 一旦口型、语音、性别、动作、文案不匹配，就会直接破坏信任
- 它不适合作为当前默认主承载

当前定位：

- 只能是可选支线或待验证支线
- 不得为了省事把它重新写回默认主线

## 8. 当前执行含义

当前默认主线的执行含义 `已确认`：

- 正式 assembly 继续固定为北京区 `OSS + 云剪 cloud-only`
- `local preview` / `local mp4` 只能算辅助，不得冒充正式完成态
- 缺真实素材时，必须诚实写成 `待素材注入验证`

必须同时明确：

- 这份文件写的是“当前仓库正式主线口径”
- 不等于“新主线样片已经验证成立”

## 9. 当前一句话规则

当前正式默认主线不是 pure PPT，也不是 AI talking avatar，而是：API 真人负责信任和判断，用户本地录制素材负责主体推进和过程证据，少量 PPT / 图片负责关键词显影、结构整理和结尾总结，正式 assembly 继续走北京区 `OSS + 云剪 cloud-only`；结构跟着文案走，人物出现 1 次还是 2 次由 block 路由决定。
