# 当前主线锚点｜API生成真人 + 用户录制素材 + 少量PPT + 云端剪辑

> 说明：文件名沿用旧仓库命名，仅为兼容现有接手入口；正文口径以当前正式主线为准，不再代表旧版 human+self footage 口径。

## 1. 文件定位
本文件是当前正式主线的唯一锚点文件。

它只负责回答：
- 当前唯一正式主线是什么
- 4 个组成部分各自负责什么
- 为什么旧路线不能再默认回流
- `API生成真人段` 1 次还是 2 次怎么判断
- 当前最常见失败模式是什么

## 2. 当前唯一正式主线
`已确认` 当前唯一正式主线固定写成：

**API生成真人 + 用户录制素材 + 少量PPT + 云端剪辑**

这 4 项必须一起理解，缺一项就不是当前完整口径。

## 3. 为什么当前要锁这条主线
这次切换不是为了“形式更丰富”，而是为了更稳地压掉几类旧问题：

### 3.1 整条PPT/信息卡承载旧路线的问题
- 容易 demo 感
- 容易说明书感
- 容易只有结论，没有过程证据
- 容易像课件，不像真实短视频

### 3.2 `真人实拍出镜` 不能自动等于当前默认主线
- 它是独立分支，不是当前默认前提
- 当前主线不应依赖“用户必须真人实拍出镜”才成立

### 3.3 旧版数字人口播路线的问题
- 口型、动作、设定、文案容易断连
- 很容易出戏
- 容易把“像人”误当成“可信”

## 4. 四个组成部分的默认职责

### 4.1 `API生成真人`
默认负责：
- 开头进入
- 关键判断
- 转折衔接
- 结尾收束

默认不负责：
- 全条主体推进
- 长时间解释型口播
- 替代真实过程证据

### 4.2 `用户录制素材`
默认负责：
- 中段主体推进
- 操作过程
- 工作流证据
- 前后差值可视化
- 现场感

默认不负责：
- 只做背景切片
- 只做装饰性 B-roll
- 被降级成 PPT 的配角

### 4.3 `少量PPT`
默认负责：
- 关键词显影
- 结构整理
- 并列信息总结
- 结果句 / 数字句强调
- 结尾总结

默认不负责：
- 整条主体叙事
- 中段大篇幅讲道理
- 重新拉回整条PPT旧路线

### 4.4 `云端剪辑`
当前资料口径里的正式含义是：
- 当前正式主需求里的组装方向
- 当前统一文件集里的唯一正式写法

它当前不等于：
- 当前仓库执行链路已经稳定跑通
- 当前本地 / 云端实现已经全部验证完成

## 5. block 级路由原则
当前结构不能先定模板，而要先拆 block。

### 常见 block
- 开头进入
- 问题点破
- 过程演示
- 关键判断
- 结构整理
- 结尾收束 / CTA

### 默认承载映射
| block | 默认承载 |
|---|---|
| 开头进入 | `API生成真人` |
| 问题点破 | `API生成真人` 或 `API生成真人 + 用户录制素材` |
| 过程演示 | `用户录制素材` |
| 关键判断 | `API生成真人` |
| 结构整理 | `少量PPT` |
| 结尾收束 | `API生成真人` 或 `少量PPT` |

## 6. `API生成真人段` 1 次 vs 2 次判断

### 更适合 1 次的情况
- 中段素材本身已经能撑住主体
- 这条更像“看过程、看操作、看案例”
- 结尾用一张总结卡就能收住

典型模式：
- 开头 `API生成真人`
- 中段 `用户录制素材`
- 结尾 `少量PPT`

### 更适合 2 次的情况
- 中段存在关键判断转折
- 素材之间有断裂，需要一次判断段来接缝
- 结尾更适合用 `API生成真人` 收束
- 这条不只是在看过程，还要听一个明确判断

典型模式：
- 开头 `API生成真人`
- 中段 `用户录制素材`
- 中后段再来一次 `API生成真人`
- 结尾 `少量PPT` 或一句 CTA

### 当前不允许预设的事
- 所有视频都只要 1 次 `API生成真人段`
- 所有视频都必须 2 次 `API生成真人段`
- 只要有 `API生成真人` 就一定更高级
- 只要有 `用户录制素材` 就完全不需要判断段

## 7. 当前最常见失败模式
- `API生成真人段` 太多，拖节奏
- `API生成真人段` 有出场，但没有承担判断价值
- `用户录制素材` 太碎，主体不稳
- `少量PPT` 太重，重新滑回课件感
- 结构没有跟着文案走，变成硬拼接
- 把“云端剪辑方向”误写成“已稳定跑通事实”

## 8. 当前质量判断重点
当一条内容走这条主线时，默认优先看：
1. 开头 `API生成真人` 段值不值
2. 中段 `用户录制素材` 稳不稳
3. `少量PPT` 是否清楚但不过课件
4. 整条是否仍像真实会发的短视频
5. 有没有硬拼接感、说明书感或 demo 感

## 9. 一句话规则
**当前默认主线不是整条PPT旧路线，也不是 `真人实拍出镜` 默认主线，而是：`API生成真人` 负责进入和判断，`用户录制素材` 负责主体推进和过程证据，`少量PPT` 负责整理和总结，最终按 `云端剪辑` 口径理解正式组装方向。**
