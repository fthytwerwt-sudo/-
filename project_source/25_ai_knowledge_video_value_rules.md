# 25｜AI 知识类视频价值规则

## 1. 文件定位

本文件用于把《视频工厂》当前 AI 知识类视频的价值底线正式写死。

它解决的是：

- 什么叫“有价值”
- 什么叫“伪价值”
- 4 类内容分别最核心的价值交付是什么
- 进入样片执行前，内容价值闸门是什么

它不是：

- 市场已经验证成立的结论文档
- 平台官方规则
- 单条视频脚本模板

## 2. 什么叫“有价值”

当前 AI 知识类视频默认只有在满足以下条件时，才算“有价值”：

1. 用户看完后有收获
2. 用户看完后能做什么
3. 用户看完后能判断什么
4. 内容有证据支撑
5. 内容有最小行动 / 自检句 / 可执行步骤中的至少一种

一句话：

不是“讲了很多”，而是“用户看完后真的能带走东西”。

## 3. 什么叫“伪价值”

以下内容当前默认属于“伪价值”：

- 只说问题
- 只给观点
- 只有共鸣，没有帮助
- 只有情绪，没有动作
- 没有证据
- 看完后仍不知道下一步做什么

generation / assembly 成功，也不能把以上内容写成“已过线”。

## 4. 四类内容分别最核心的价值交付

### 4.1 AI 项目讲解

- 最核心交付：
  - 让用户判断项目卡点、关键取舍和最小下一步

### 4.2 AI 方法分享

- 最核心交付：
  - 让用户能试一个方法，并知道最容易错在哪里

### 4.3 AI 学习实操

- 最核心交付：
  - 让用户能照着做一个最小步骤，并知道如何自检

### 4.4 AI 案例拆解

- 最核心交付：
  - 让用户判断案例为什么成立，并知道如何迁移

## 5. 进入样片执行前的价值闸门

进入样片执行前，至少先过以下价值闸门：

1. 用户看完后能做什么
2. 用户看完后能判断什么
3. 这条内容的证据是什么
4. 这条内容的最小行动 / 自检句是什么

缺任一项，当前都不应直接进入样片执行。

## 6. 当前一句话规则

当前 AI 知识类视频的价值底线，不是“讲了很多”，而是“用户看完后能做什么 / 能判断什么 / 有没有证据支撑 / 有没有最小行动或自检句”。
