# 视觉动效与信息密度规则

## 1. 文件定位

本文件用于把当前这轮 45 秒 role handoff 样片回审里，已经收束的“视觉动效 / 信息密度”判断，正式写成项目脑里的阶段性稳定判断层。

它负责回答：

- 当前这类知识型 / AI 类 / PPT 信息卡视频，画面“差点意思”时，先看什么
- 当前这一轮最主要的问题层与次主要问题层怎么判断
- 哪些手法当前属于默认白名单
- 哪些手法当前属于默认黑名单

它不负责：

- 重写项目主线
- 代替单条视频的完整执行单
- 把单轮回审结论写成长期永久规律
- 把外部参考写成项目天然已确认事实

一句话：

本文件是当前内容阶段、当前这类 45 秒知识型样片的阶段性稳定判断层，不是全项目永久审美定律。

## 2. 适用场景

- `已确认`
  - 当前直接适用于：
    - 45 秒左右
    - AI 方法分享 / AI 项目讲解
    - pure PPT / 信息卡母版或以信息卡为主承载的知识类视频
    - 已经有完整片，本轮目标是回审收束与下一轮定向修改，不是整条重做
- `建议判断`
  - 后续若同类视频出现以下表述，可优先回到本文件：
    - “画面信息太少”
    - “转场有点样机感”
    - “内容能听懂，但画面不够有看头”
    - “重点内容没有被视觉接住”
- `外部参考`
  - 本文件内引用的 Perplexity 结论，只作为“这轮已收束采用的外部参考”使用，不自动上升为长期项目规律。

## 3. 当前典型失败模式

- `已确认`
  - 图片转场仍有回弹 / bounce 感。
- `已确认`
  - 图片内容太少，和配音相比，画面没有足够可看性。
- `已确认`
  - 剪辑吸引力不够，重点内容缺少字重放大、关键帧推进与视觉强调。
- `已确认`
  - 当前更根本的问题，不是“转场不好看”，而是“画面没有承担信息推进”。
- `建议判断`
  - 若观感被描述成“空”“平”“像课件”，先排查：
    - 页面是否只是整页静态切换
    - 重点是否跟着旁白逐步出现
    - 画面里是否存在支撑判断的视觉证据
    - 页面变化是否只停留在换页，而不是推进

## 4. 当前优先问题层判断口径

- `已确认`
  - 当前最主要问题层：
    - `表现层`
- `已确认`
  - 当前次主要问题层：
    - `字幕与配音节奏层`
- `已确认`
  - 当前唯一最优先改点：
    - 把整页静态切换改成“跟着旁白推进的分步 reveal + 关键词显影节奏”
- `建议判断`
  - 第二优先才是：
    - 把 `bounce / 回弹` 替换为更稳的直切 / 轻推进 / 轻淡入淡出
- `建议判断`
  - 第三优先才是：
    - 建立固定字号层级，让每页只有 1 个主判断被视觉突出

## 5. 默认白名单手法

- `已确认`
  - 默认优先手法：
    - 分步 reveal
    - 关键词显影
    - 让画面跟着旁白逐步承担信息推进
- `建议判断`
  - 当前同类片的稳妥增强顺序应优先理解为：
    - 先推进关系
    - 再稳转场
    - 最后统一字号层级
- `外部参考`
  - 当前已收束采用、且高性价比的可看性增强手法包括：
    - 分步 reveal
    - 关键词显影
    - 结果句 / 数字强调
    - 局部高亮 / 框选
    - 停顿点设计

## 6. 默认黑名单手法

- `外部参考`
  - 当前不建议把 `bounce / 回弹` 继续作为知识类 / AI 类 / PPT 信息卡母版的默认动效，因为它容易带来模板感、课件感、样机感。
- `外部参考`
  - 当前不建议的方向还包括：
    - 粒子 / 发光 / 3D 炫技
    - 把旁白全文搬到画面上
    - 全页同时强化多个重点
    - 为了可看性乱加素材填时间
    - 套营销广告式快节奏
- `已确认`
  - 本轮禁止误改方向：
    - 不要把这轮主改点误判成“先重做转场”
    - 不要把“画面信息太少”直接翻译成“每页加更多字”
    - 不要因为想补可看性，就把整条视频重写成另一种内容结构

## 7. 当前一句话规则

- `已确认`
  - 当这类 45 秒知识型样片被反馈为“画面空、回弹重、不够有看头”时，先不要把问题理解成纯转场问题；先让画面跟着旁白做分步 reveal + 关键词显影，等信息推进成立后，再处理稳转场和字号层级。
