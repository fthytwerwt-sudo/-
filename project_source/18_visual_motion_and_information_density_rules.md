# 视觉动效与信息密度规则

## 1. 文件定位
本文件是 `project_source/08_quality_baseline_and_90_score_rules.md` 的视觉补充规则。

它负责：
- 当用户觉得“空 / 平 / 像 demo / 像课件 / 拼得碎”时，优先先查什么
- 哪些视觉增强做法当前是白名单
- 哪些做法当前是黑名单

## 2. 当前优先问题层
当当前主线下出现“空 / 平 / 像 demo / 像课件 / 拼得碎”反馈时，默认优先按以下顺序判断：
1. `API 生成真人` 是否越权或失职
2. `用户录制素材` 是否真的承担中段主体推进
3. `少量 PPT / 图片` 是否越权抢主叙事
4. 字幕、配音、画面是否同步推进
5. 最后才看装饰性花活

## 3. 白名单做法
- 关键位置才放 `API 生成真人`
- 中段尽量让 `用户录制素材` 证明过程
- 讲到哪一步，画面推进到哪一步
- 关键词、结果句、数字句做单点显影
- 在关键结论处给处理时间

## 4. 黑名单做法
- 为了“有脸”而过度堆判断段
- 素材乱切，假装有节奏
- 把旁白全文搬上 PPT
- 一页同时强化多个重点
- 靠花活掩盖结构问题

## 5. 一句话规则
**当前视觉层优先解决的是“画面有没有承担信息推进”，不是“画面够不够热闹”；先让 `API 生成真人 / 用户录制素材 / 少量 PPT` 各司其职，再谈更花的视觉增强。**
