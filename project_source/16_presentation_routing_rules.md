# 展示路由规则

## 1. 文件定位
本文件用于把《视频工厂》的展示路由写成统一判断框架。

它负责：
- 一条视频整体应该偏什么主承载
- 每个 block 应该由谁承担
- 失败时如何回退
- 回审时如何判断是内容不过线，还是路由选错

## 2. 当前正式默认路由
当前展示路由的正式默认主线 `已确认` 为：
- `API 生成真人`
- `用户录制素材`
- `少量 PPT / 图片辅助`
- `云端剪辑`

固定理解：
- 结构跟着文案走
- `API生成真人段` 出现 1 次还是 2 次，是 block 路由结果
- hook / close 默认优先给 `API 生成真人`
- 中段主体默认优先给 `用户录制素材`
- `pure PPT / 信息卡` 只保留为次级支路
- 旧版数字人口播路线只保留为可选 / 待验证支线

## 3. 正式判断顺序
展示路由必须按以下顺序判断：
1. 先判内容类型
2. 再判整条视频的主价值
3. 再判观众最先需要接住什么
4. 再拆 block 职责
5. 再把每个 block 继续拆成 `segment`
6. 先判每个 `segment` 在干什么
7. 再判每个 `segment` 由谁承担
8. 最后才按字数 / 动作量 / 读速反推时长

明确禁止：
- 先按偏好选真人或 PPT
- 因为“真人更高级”就默认 `真人实拍出镜`
- 因为“PPT 更稳”就默认整条PPT旧路线
- 先把人物次数、结尾卡时长或录屏时长写死，再倒过来硬塞内容

## 4. 路由单位
展示路由不是只停在 `block`。

- `block` 是上层路由单位
- `segment` 是执行判断单位

也就是说：
- 先用 `block` 决定这一大段要承担什么职责
- 再用 `segment` 决定这一小段到底该给谁承载
- 执行时不允许只判到 block 就停

### 常见 block
- 开头进入
- 问题点破
- 过程演示
- 关键判断
- 结构整理
- 结尾收束

### 默认承载映射
| block | 默认承载 |
|---|---|
| 开头进入 | `API 生成真人` |
| 问题点破 | `API 生成真人` 或 `API 生成真人 + 用户录制素材` |
| 过程演示 | `用户录制素材` |
| 关键判断 / 转折 | `API 生成真人` |
| 结构整理 | `少量 PPT / 图片` |
| 结尾收束 / 最小行动 | `API 生成真人` 或 `summary_card` |

### segment 功能类型
当前执行时，每个 block 内的小段默认先分成下面 4 类：

- 判断节点
- 证据节点
- 整理节点
- 转场节点

### segment 判断顺序
每个 `segment` 默认按下面顺序判断：
1. 先判这小段是在下判断、给证据、做整理，还是只负责过渡
2. 再判这小段更适合谁来承载
3. 最后才按字数、动作量、可读性和观看负担反推时长

当前项目这一句必须作为正式默认理解：

**先判这一小段在干什么，再判这段字有多少、观众来不来得及看。功能决定承载，字数决定时长。**

### segment 默认承载映射
| segment 功能 | 默认承载 |
|---|---|
| 判断节点 | `API 生成真人` |
| 证据节点 | `用户录制素材` |
| 整理节点 | `少量 PPT / 总结卡` |
| 转场节点 | 看它更像判断还是整理，不单独乱加人物 |

## 5. 回退规则
- `API 生成真人` 不成立：
  - 优先回退到 `hybrid`
  - 若当前更缺结构而不是信任，再回退到 `ppt_or_image_secondary`
- `用户录制素材` 不成立：
  - 优先补真实素材
  - 若短期无法补齐，只能诚实标记 `待素材注入验证`
  - 不得直接偷换回整条PPT旧路线
- `少量 PPT` 不成立：
  - 优先减负，只保留关键词、结果句、总结卡
  - 不得扩成培训课件感主体叙事

## 6. 回审层必须问的问题
1. 这条内容更需要先接住“谁在说”，还是“过程证据”？
2. 每个 block 是否已经继续拆到 `segment`，而不是只停在 block 口号层？
3. 中段主体是不是已经交给 `用户录制素材` 承担？
4. `少量 PPT / 图片` 是不是只做了整理 / 显影 / 总结？
5. `API生成真人段` 次数是不是由判断节点数推出来的？
6. 当前节奏问题到底来自内容不过线，还是 `segment` 路由错误？

## 7. 一句话规则
**当前展示路由默认按“API 生成真人 + 用户录制素材 + 少量 PPT / 图片 + 云端剪辑”理解：先拆 block，再拆 `segment`；hook / close 默认优先给 `API 生成真人`，中段主体默认优先给 `用户录制素材`，整理节点默认给 `少量 PPT / 总结卡`，`API生成真人段` 次数由判断节点数决定。**
