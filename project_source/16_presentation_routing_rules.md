# 展示路由规则

## 1. 文件定位
本文件用于把《视频工厂》的展示路由写成统一判断框架。

它负责：
- 一条视频整体应该偏什么主承载
- 每个 block 应该由谁承担
- 失败时如何回退
- 回审时如何判断是内容不过线，还是路由选错

## 2. 当前正式默认路由
当前展示路由的正式默认主线 `已确认` 为：
- `API 生成真人`
- `用户录制素材`
- `少量 PPT / 图片辅助`
- `云端剪辑`

固定理解：
- 结构跟着文案走
- `API生成真人段` 出现 1 次还是 2 次，是 block 路由结果
- hook / close 默认优先给 `API 生成真人`
- 中段主体默认优先给 `用户录制素材`
- `pure PPT / 信息卡` 只保留为次级支路
- 旧版数字人口播路线只保留为可选 / 待验证支线

## 3. 正式判断顺序
展示路由必须按以下顺序判断：
1. 先判内容类型
2. 再判整条视频的主价值
3. 再判观众最先需要接住什么
4. 再拆 block 职责
5. 最后才判每个 block 由谁承担

明确禁止：
- 先按偏好选真人或 PPT
- 因为“真人更高级”就默认 `真人实拍出镜`
- 因为“PPT 更稳”就默认整条PPT旧路线

## 4. 路由单位
展示路由的正式执行单位是 `block`。

### 常见 block
- 开头进入
- 问题点破
- 过程演示
- 关键判断
- 结构整理
- 结尾收束

### 默认承载映射
| block | 默认承载 |
|---|---|
| 开头进入 | `API 生成真人` |
| 问题点破 | `API 生成真人` 或 `API 生成真人 + 用户录制素材` |
| 过程演示 | `用户录制素材` |
| 关键判断 / 转折 | `API 生成真人` |
| 结构整理 | `少量 PPT / 图片` |
| 结尾收束 / 最小行动 | `API 生成真人` 或 `summary_card` |

## 5. 回退规则
- `API 生成真人` 不成立：
  - 优先回退到 `hybrid`
  - 若当前更缺结构而不是信任，再回退到 `ppt_or_image_secondary`
- `用户录制素材` 不成立：
  - 优先补真实素材
  - 若短期无法补齐，只能诚实标记 `待素材注入验证`
  - 不得直接偷换回整条PPT旧路线
- `少量 PPT` 不成立：
  - 优先减负，只保留关键词、结果句、总结卡
  - 不得扩成培训课件感主体叙事

## 6. 回审层必须问的问题
1. 这条内容更需要先接住“谁在说”，还是“过程证据”？
2. 中段主体是不是已经交给 `用户录制素材` 承担？
3. `少量 PPT / 图片` 是不是只做了整理 / 显影 / 总结？
4. `API生成真人段` 次数是不是由 block 职责推出来的？
5. 当前问题到底是内容不过线，还是展示路由错误？

## 7. 一句话规则
**当前展示路由默认按“API 生成真人 + 用户录制素材 + 少量 PPT / 图片 + 云端剪辑”理解：hook / close 默认优先给 `API 生成真人`，中段主体默认优先给 `用户录制素材`，结构跟着文案走，`API生成真人段` 次数由 block 决定。**
