# 搜索层提示模板库

## 1. 文件定位
本文件用于给搜索层提供稳定提示模板。

它负责：
- 外部事实查证
- 类似案例和表达参考搜索
- 主线承载分工的外部验证
- 平台内容形态和质量观察

它不负责：
- 项目最终拍板
- 项目正式事实声明
- 直接代替主文件做判断

## 2. 固定背景前置块
以下前置块可直接贴给搜索层：

```text
你现在服务的项目是《视频工厂》。

当前正式主线固定为：
- API生成真人
- 用户录制素材
- 少量PPT
- 云端剪辑

当前主线职责分工固定为：
- API生成真人负责开头进入、关键判断、结尾收束
- 用户录制素材负责中段主体推进、过程证据、工作流证明
- 少量PPT负责关键词显影、结构整理、结尾总结

当前不再默认采用：
- 整条PPT/信息卡承载旧路线
- 真人实拍出镜默认主线
- 旧版数字人口播默认主线

硬规则：
- 不要把 API生成真人、真人实拍出镜、用户录制素材 混成一个词
- 不要把 云端剪辑 / cloud-only 写成“已稳定跑通”的事实
- 如果引用外部说法，请区分“外部观察”“建议判断”“已确认事实”
```

## 3. 最常用模板

### 模板 01：主线承载是否合理
```text
[固定背景前置块]

请判断“API生成真人 + 用户录制素材 + 少量PPT”是否是更稳的知识类 / AI 类短视频承载路线。

请分别回答：
1. API生成真人最适合承担什么
2. 用户录制素材最适合承担什么
3. 少量PPT最适合承担什么
4. 哪些做法会让它滑回 demo 感 / 说明书感 / 课件感 / 硬拼接感
5. 给我一句可直接写进项目文件的收束表述
```

### 模板 02：block 级承载职责研究
```text
[固定背景前置块]

我现在研究的不是固定模板，而是 block 级承载职责。

请围绕以下 block 研究更稳的承载分工：
- 开头进入
- 问题点破
- 过程演示
- 关键判断
- 结构整理
- 结尾收束

请输出：
1. block → 目标 → 最优承载映射表
2. API生成真人、用户录制素材、少量PPT各自最容易越权的错误
3. 一句话规则
```

### 模板 03：1 次 vs 2 次 API生成真人段判断
```text
[固定背景前置块]

请研究：
什么时候更适合 1 次 API生成真人段，
什么时候更适合 2 次 API生成真人段？

重点回答：
1. 什么情况下开头一次判断段就够
2. 什么情况下中段或结尾还需要第二次判断段
3. 什么情况下判断段太多会拖节奏
4. 什么情况下判断段太少又不够信任感
5. 给我一个可直接写进项目文件的判断清单
```

### 模板 04：旧版数字人口播路线风险
```text
[固定背景前置块]

请专项研究：
为什么旧版数字人口播路线当前不适合作为默认主线。

我已知的真实问题包括：
- 口型和语音对不上
- 动作和文案不匹配
- 设定和文案不匹配
- 画面容易假、容易出戏

请输出：
1. 这些问题为什么会直接破坏质量感
2. 它们更像信任风险、出戏风险还是 demo 感风险
3. 当前更稳的替代路线是什么
4. 一段能直接写进项目文件的正式表述
```


## 4. 使用规则
- 先贴固定背景前置块
- 再贴专项模板
- 再补当前卡点
- 最后由主文件或主对话收束

## 5. 一句话规则
**搜索层最优角色不是“给几条链接”，而是先把外部事实、案例和结构参考找回来，再由当前中文主文件体系统一收束。**
