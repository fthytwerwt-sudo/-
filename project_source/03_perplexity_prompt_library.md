# Perplexity Prompt 模板库

## 1. 定位

Perplexity 在本项目中不是低频补充工具，而是高频默认搜索层与补位层。

它优先用于：

- 外部信息查证
- 平台内容形态观察
- 类似案例与表达参考
- 质量增强路线比较
- 配音方案与字幕方案调研
- 画面路线与 demo 感治理
- 文案首稿与口语化素材
- skill / 工具 / 工作流路线调研

默认流程：

**先贴固定背景前置块 → 再贴专项模板 → 再补当前卡点 → 最后由 ChatGPT 收束**

## 2. 项目固定背景前置块

以下内容建议作为大多数查询的固定前置块：

```text
你现在服务的项目是“视频工厂：AI 垂类场景化视频内核”。

项目当前阶段：
- 已有一个本地 demo 跑通脚本、字幕、配音、视频这条链路
- 当前重点不是证明能不能生成视频，而是把质量压到可发布测试水位
- 当前允许本地生成 + 人工上传
- 当前不是平台发布 API 前置依赖
- 第一阶段默认主线是“人物判断 + 用户自录素材 + 少量 PPT / 图片辅助”

关于当前 demo：
- 当前 demo 只证明链路能跑通
- 当前 demo 是最小闭环验证件 / 运行锚点
- 当前 demo 不具备质量参考价值
- 当前 demo 不能继续被当作质量样片或质量基线

关于“抖音 90 分标准”：
- 这是项目内部质量简称，不是平台官方评分体系
- 它更接近创作者 / 操盘手对 AI 类、知识类、无人出镜短视频可推荐门槛的经验口径

关于当前第一优先路线：
- 当前第一优先不是平台发布 API、自动化运营、图生视频或数字人默认主承载
- 当前第一优先是用户现成可用的火山引擎 TTS API
- 当前目标是先解决机械配音 / 系统播报感
- 当前“走火山引擎一套”只表示供应商统一优先
- 本轮执行上只先接 TTS，不默认同时扩到 ASR、图生视频、平台 API 等其他能力

项目内容边界：
- 只做 AI 相关视频
- 当前高频核心场景：AI 项目讲解、AI 方法分享、AI 学习实操、AI 案例拆解
- 回答要尽量偏“小白可落地”“真实可执行”“像短视频内容”
- 避免空泛概念、营销腔、官网说明书式回答
```

## 3. 最常用 Top 5 模板

最常用 Top 5 模板建议优先记住：

1. 抖音 90 分标准判定模板
2. 质量增强 API 优先级模板
3. 类似案例 / 表达参考模板
4. 配音方案调研模板
5. 当前某条视频的专项查法

项目内固定说明：

- 如果当前问题是“demo 质量太差，不能当样片”，优先用“质量增强 API 优先级模板”
- 如果当前问题是“90 分标准到底怎么定义”，优先用“抖音 90 分标准判定模板”

## 4. 模板目录

### 模板 01：抖音 90 分标准判定模板

- 适用触发场景：
  - 想确认“抖音 90 分标准”是不是官方概念
  - 想把它压成项目内部可执行质量框架
- 解决什么问题：
  - 防止把行业经验误写成平台官方规则
- 可直接复制给 Perplexity 的模板正文：

```text
[固定背景前置块]

请帮我判断“抖音 90 分标准”这个说法在短视频语境里到底是什么。

要求：
1. 先判断它是不是平台官方概念、官方评分体系或公开官方标准
2. 如果不是，请明确说明它更像什么：创作者经验口径、操盘手内部说法、行业黑话，还是其他
3. 只围绕 AI / 知识类短视频与“API 真人 + 用户本地素材 + 少量辅助画面 + 云端剪辑”主线来收束，不要泛化到所有内容赛道
4. 最终请把结果压成一个可写进项目文件的内部质量框架

输出必须包含：
- 定义层：一句话说清它是不是官方概念
- 必须过线项：至少 5-7 条
- 一票否决项：至少 5-7 条
- 可直接写进项目文件的正式表述：1 段可复制文本

要求风格：
- 不要模糊表达
- 不要把猜测写成已确认事实
- 如果查不到官方依据，要明确写“不是已知官方概念”
```

- 理想输出长什么样：
  - 先给“非官方概念”判断
  - 再给项目内部可执行质量框架
- 查回来后如何接到 ChatGPT：
  - 让 ChatGPT 把结论收进 `08_quality_baseline_and_90_score_rules.md` 与回审模板
- 不适用边界：
  - 不适合只查单条视频选题时使用

### 模板 02：质量增强 API 优先级模板

- 适用触发场景：
  - 已确认 demo 能跑，但质量太差
  - 需要判断当前第一优先该补哪条增强路线
- 解决什么问题：
  - 防止一上来把平台发布、自动化运营或整套能力一起端上来
- 可直接复制给 Perplexity 的模板正文：

```text
[固定背景前置块]

请围绕“质量增强 API 优先级”帮我做一份收束判断。

已知前提：
- 当前 demo 只证明链路能跑通，不具备质量参考价值
- 当前问题不是能不能生成视频，而是 demo 感太重
- 当前只研究质量增强 API，不研究平台发布 API、自动化运营或账号运营能力

请重点比较：
- TTS
- 字幕对齐 / 字幕时序增强
- 图生视频
- 其他可能直接改善 demo 感的增强路线

要求：
1. 每条路线都说明它主要解决什么问题
2. 明确它更像“必须先做”还是“可以后做”
3. 优先判断哪条路线最适合先把最致命的一票否决项打掉
4. 如果当前用户已经有现成可用的火山引擎 TTS API，请把这点纳入优先级判断
5. 最终必须收束回答：当前第一优先是不是火山引擎 TTS，为什么

输出格式：
- 路线对比表
- 当前建议优先级排序
- 一句话结论
```

- 理想输出长什么样：
  - 能直接把“TTS 是否第一优先”说清楚
- 查回来后如何接到 ChatGPT：
  - 让 ChatGPT 只保留当前第一优先路线，并压成可交给 Codex 的执行任务
- 不适用边界：
  - 不适合只查文案表达时使用

### 模板 03：类似案例 / 类似表达参考

- 适用触发场景：
  - 想知道类似主题别人怎么讲更像短视频
- 解决什么问题：
  - 防止内容只有信息，没有表达参照
- 可直接复制给 Perplexity 的模板正文：

```text
[固定背景前置块]

请帮我找“类似主题的 AI 内容表达参考”。

我要的不是简单账号推荐，而是：
1. 找出类似创作者、类似案例或类似视频表达
2. 提炼他们常见的：
   - 开头方式
   - 结构方式
   - 节奏方式
   - 结尾落点
3. 告诉我哪些表达更像真人短视频，哪些明显像说明书或 demo 演示
4. 如果有平台差异，也请顺带指出
```

- 理想输出长什么样：
  - 3 到 8 个参考对象
  - 每个对象对应表达特征，而不是只有名字
- 查回来后如何接到 ChatGPT：
  - 让 ChatGPT 收束成“本条视频可借的表达动作”

### 模板 04：火山引擎 TTS 细化调研

- 适用触发场景：
  - 已确定火山引擎 TTS 是第一优先
  - 想进一步判断该怎么接才最值
- 解决什么问题：
  - 防止只知道“要接 TTS”，却不知道该怎么选声音、参数和接入方式
- 可直接复制给 Perplexity 的模板正文：

```text
[固定背景前置块]

请帮我调研“火山引擎 TTS 在 AI / 知识类短视频里最值得优先试的接法”。

重点想看：
1. 哪类中文声音更容易摆脱系统播报感
2. 语速、停顿、情绪幅度该怎么控制更自然
3. 知识类短视频适合什么声音风格，不适合什么声音风格
4. 如果当前目标只是先打掉机械配音，最小接入方案应该怎么定
5. 哪些做法会让配音听起来更像客服播报或系统提示音

输出请尽量给：
- 推荐思路
- 不推荐思路
- 当前最小可试方案
```

- 理想输出长什么样：
  - 可直接转成接入策略和验收点
- 查回来后如何接到 ChatGPT：
  - 让 ChatGPT 收束成“先接哪种声线 + 先看哪些参数”的执行单

### 模板 05：配音方案调研

- 适用触发场景：
  - 觉得当前配音太像系统播报
  - 想横向看不同中文 AI 配音路线
- 解决什么问题：
  - 判断是否真的需要换路线，还是先沿现路线优化
- 可直接复制给 Perplexity 的模板正文：

```text
[固定背景前置块]

请帮我调研“更像真人短视频而不是系统播报”的中文 AI 配音方案。

请重点比较：
1. 语音自然度
2. 停顿控制
3. 语速可控性
4. 情绪幅度是否适合知识向短视频
5. 小白可上手程度
6. 成本与稳定性

请输出：
- 当前最值得试的 3-5 个方案
- 每个方案最适合什么类型的视频
- 如果我当前已有火山引擎 TTS API，可否优先不换供应商
```

### 模板 06：字幕方案调研

- 适用触发场景：
  - 觉得字幕节奏、样式或可读性不够
- 解决什么问题：
  - 找到更适合短视频的字幕策略
- 可直接复制给 Perplexity 的模板正文：

```text
[固定背景前置块]

请帮我调研“适合 AI 知识型短视频”的中文字幕方案。

重点想看：
1. 字幕分段策略
2. 每段字数控制
3. 停顿与断句建议
4. 字幕与配音同步的常见做法
5. 知识向视频常见字幕问题

请输出：
- 最适合知识向短视频的 3 套字幕思路
- 各自适用场景
- 哪种最适合先配合 TTS 升级一起做
```

### 模板 07：视频模板 / 画面表达参考

- 适用触发场景：
  - 想提升画面，不再那么像程序 demo
- 解决什么问题：
  - 找到更合适的 PPT 信息卡、案例画面、旁白画面路线
- 可直接复制给 Perplexity 的模板正文：

```text
[固定背景前置块]

请帮我调研“适合 AI 知识型短视频、优先人物判断 + 自录素材证据主线”的画面表达路线。

至少覆盖：
- PPT 信息卡
- 信息卡 + 素材切片
- 屏幕演示 + 字幕
- 旁白 + 案例画面

请输出：
1. 每种路线适合什么场景
2. 哪种最容易先达到可发布测试水位
3. 哪种最容易做成说明书感或 demo 感
4. 画面层最常见失败点
5. 在 TTS 优先接入之后，下一步最值得补哪一种画面升级
```

### 模板 08：文案首稿

- 适用触发场景：
  - 已有主题，但想先拿一个首稿
- 解决什么问题：
  - 快速从空白到可改
- 可直接复制给 Perplexity 的模板正文：

```text
[固定背景前置块]

请基于以下主题，给我写 3 个方向不同的短视频首稿。

要求：
1. 先判断更适合哪个场景模式
2. 每个首稿都明确结构
3. 更像短视频表达，不像说明书或项目汇报
4. 优先知识向、中文短视频，并默认考虑人物判断 + 自录素材 + 少量辅助画面
5. 每个版本控制在 15-45 秒表达长度

当前主题：
[把具体主题贴在这里]
```

### 模板 09：当前某条视频的专项查法

- 适用触发场景：
  - 已经有一条具体视频卡住了
- 解决什么问题：
  - 对单条视频做定制化外部调研
- 可直接复制给 Perplexity 的模板正文：

```text
[固定背景前置块]

我现在有一条具体视频卡在这里：
[把卡点写清楚，例如：开头太弱 / 结果不够具体 / 画面太 demo / 配音太系统播报]

这条视频的主题是：
[贴主题]

当前结构是：
[贴结构]

请你只围绕这条视频当前卡点，给我最有帮助的外部参考和改法建议。

要求：
1. 不要泛泛讲原则
2. 优先给类似案例、类似表达、类似画面或类似节奏参考
3. 输出必须能让我带回 ChatGPT 后直接收束成修改方案
```

### 画面专项

## 6. 默认使用顺序

默认顺序固定为：

1. 先贴“项目固定背景前置块”
2. 再贴一个专项模板
3. 最后补上当前这条视频的具体卡点
4. 由 ChatGPT 收束成项目判断、项目文件文本或 Codex 执行任务

## 7. 结论

Perplexity 在本项目中的最优角色不是“补几条链接”，而是：

- 先确认外部事实和行业口径
- 再比较质量增强路线
- 再提供表达参考与首稿素材
- 最后由 ChatGPT 收束进项目脑或交给 Codex

这样才能避免把 demo 身份、质量基线和能力优先级继续混在一起。
