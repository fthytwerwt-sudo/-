# 用户可读仓库地图

## 1. 这份文件是干什么的

这不是给 Codex 的执行单，也不是项目脑总纲。

它只解决三件事：

- 帮非技术用户先看懂这个仓库怎么分层
- 帮你判断“我觉得哪里不对”大概落在哪一层
- 帮你更准确地把问题告诉 ChatGPT，而不是一上来就说“整体都不对”

如果你只记一句话：

- `project_source/` 管“这个项目是什么、这条内容该怎么判断”
- `codex_source/` 管“仓库任务怎么推进、什么时候开分支、写日志、commit、push、PR”
- `codex_log/` 管“最近到底做过什么，不用靠聊天记忆猜”
- `cases/` 和 `dist/` 管“当前输入是什么、产物到底出来了没有”

| 你现在最真实的卡点 | 更像哪一层 | 先看哪里 |
| --- | --- | --- |

| 我说不清这条视频到底是项目讲解、方法分享，还是案例拆解 | 结构问题 | `project_source/02_scene_mode_templates.md` |

| 画面太像 demo，或页与页之间太平，没有变化感 | 画面问题 | `project_source/04_review_templates.md` |
| 配音太像系统播报，字幕切得怪，节奏太赶或太平 | 节奏问题 | `project_source/04_review_templates.md` |

## 3. 用户最值得认识的关键文件

下面这些文件不是“全仓库所有文件”，而是最值得先认识的一组。

### 3.1 顶层入口与接手文件

| 文件 | 它主要管什么 | 什么时候应该看它 | 如果这里出问题，常见表面现象是什么 | 这更像哪类问题 | 你最适合怎么告诉 ChatGPT |
| --- | --- | --- | --- | --- | --- |
| `AGENTS.md` | 顶层入口。说明当前项目阶段、默认先读什么、仓库型任务默认走什么线路 | 新会话接手，或你不知道“现在到底该先读什么”时 | 新聊天框反复让你补背景；明明是仓库任务却没走分支 / PR 线路；把 demo 锚点说丢了 | 结构问题 / 执行问题 | “先按仓库入口接手，不要跳过 `AGENTS.md`。我这轮主要想改的是 [内容 / 执行 / 产物]。” |

## 4. 常见表面现象，通常该往哪层找

### 4.1 更像项目边界问题

常见现象：

- 本来在做视频，聊着聊着又变成做平台
- 明明是 demo 锚点任务，却开始往直播、增长、商业包装跑
- 把当前 15 秒 demo 误写成整个项目未来唯一形态

优先看：

- `project_source/00_project_brief.md`
- `AGENTS.md`

### 4.2 更像结构问题

常见现象：

- 你说不清这条视频到底属于哪个场景
- 片子有信息，但不像一条成立的短视频
- 每一句都没大错，但组合在一起不顺

优先看：

- `project_source/02_scene_mode_templates.md`
- `project_source/04_review_templates.md`

### 4.3 更像文案问题

常见现象：

- 句子太书面
- 开头像汇报，不像短视频
- 结果页说了等于没说

优先看：

- `dist/demo/script.txt`
- `project_source/04_review_templates.md`
- `project_source/05_psychology_execution_rules.md`

### 4.4 更像节奏问题

常见现象：

- 配音太赶或太平
- 字幕切分不顺
- 页切换和字幕、配音对不上

优先看：

- `dist/demo/captions.srt`
- `dist/demo/final.mp4`
- `project_source/04_review_templates.md`

### 4.5 更像执行问题

常见现象：

- 不知道为什么要先读文件、写日志、开分支
- 不知道这轮该不该 commit / push / PR
- 你感觉 ChatGPT 或 Codex 在流程上绕路，或者没按仓库协作方式推进

优先看：

- `codex_source/01_execution_rules.user_guide.md`
- `codex_source/01_execution_rules.md`
- `codex_log/latest.md`

### 4.6 更像产物问题

常见现象：

- 你怀疑“说是跑通了，其实只是部分文件出来了”
- `script.txt`、`captions.srt`、`voice.mp3`、`final.mp4` 里有缺项
- 你分不清是输入写法问题，还是执行链路没走完

优先看：

- `codex_source/05_runtime_and_artifact_rules.md`
- `cases/demo.md`
- `dist/demo/final.mp4`

## 5. 以后你可以这样告诉 ChatGPT 你觉得哪里不对

下面这些说法，通常比“整体不对”更有用。

### 如果你怀疑是项目边界问题

```text
这轮先不要扩项目边界。我觉得问题不是功能不够，而是已经偏离当前视频项目范围了。
请先按 `project_source/00_project_brief.md` 判断，这轮该收回到哪。
```

### 如果你怀疑是结构问题

```text
我觉得这轮最大问题更像结构，不一定是文案。
请先判断这条内容属于哪个场景，再只告诉我最值得换的结构点。
```

### 如果你怀疑是文案问题

```text
结构先别大改，我觉得主要是文案不像人话，太像说明书。
请先只从文案层定位最拖后腿的一句或一段。
```

### 如果你怀疑是节奏问题

```text
内容大意先不动，我觉得问题更像字幕和配音节奏。
请先判断最明显的不顺出在哪个时间段或哪一页。
```

### 如果你怀疑是画面问题

```text
脚本先别重写，我觉得这条片子最大的问题更像画面表现太像 demo。
请先告诉我最值得先改的画面问题在哪一页。
```

### 如果你怀疑是执行问题

```text
这轮先别讨论内容方向，我觉得问题在执行层。
请先判断是读取顺序、分支流程、日志节奏，还是验证口径不对。
```

### 如果你怀疑是产物问题

```text
请先不要默认判定“已跑通”。
我想先知道这次是输入层、执行链路，还是产物输出出了问题。
```

---

## 6. 最后一条提醒

你越能先说清楚“我想改的是哪一层”，ChatGPT 越不容易一上来就把所有层混在一起重写。

这份地图最核心的用途，不是让你背文件名，而是帮你把一句模糊的“哪里不对”，先压成一个更可讨论的问题。
