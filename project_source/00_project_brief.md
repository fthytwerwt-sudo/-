# 视频工厂：项目简介

## 1. 项目当前是什么

《视频工厂》当前是一个：

- 个人内部使用的
- Prompt 驱动协作的
- 可由 Codex 重复执行的
- 以视频项目为中心推进的
- 重点压内容质量、结构稳定与可复用性的

AI 知识类视频内核项目。

它当前不是：

- 大而全产品
- 平台发布系统
- 直播 / 增长 / 商业化项目
- “只差平台 API 就能成立”的叙事

## 2. 当前阶段

当前阶段 `已确认`：

- 技术闭环已跑通
- 当前重点不是“能不能生成视频”
- 当前重点是内容质量、结构稳定、可复用、可回审、可持续压质量
- 当前主阶段是内容阶段，正往试发阶段过渡

## 3. 当前正式默认主线

当前正式默认主线 `已确认` 为：

- 人物
- 用户自己的真实录制素材
- 少量 PPT / 图片辅助

当前正式主线口径同时固定为：

- 结构跟着文案走，不先预设整条视频的固定载体顺序
- 人物出现 1 次还是 2 次，是 block 路由结果，不是预设模板
- pure PPT / 信息卡全片承载，不再是默认主线
- AI talking avatar / 数字人口播，不再是默认主线
- 正式 assembly 继续固定为北京区 `OSS + 云剪 cloud-only`
- `local preview` / `local mp4` 只能算辅助，不得冒充正式完成态

## 4. demo 当前身份

当前 demo 的身份 `已确认` 为：

- demo 只是链路锚点
- demo 只证明最小链路闭环可运行
- demo 不是质量样片
- demo 不能再定义整个项目未来质量下限

## 5. 当前 AI 知识类视频价值底线

当前价值底线 `已确认` 为：

- 不能只说问题
- 不能只给观点
- 不能没有动作
- 不能没有证据
- 不能让用户看完后仍不知道下一步做什么
- generation / assembly 成功，不等于内容过线

当前高频内容继续按以下 4 类理解：

1. `AI 项目讲解`
2. `AI 方法分享`
3. `AI 学习实操`
4. `AI 案例拆解`

它们的价值交付、证据结构与结尾总结卡不再共用一种写法。

## 6. 当前必须诚实区分的两件事

以下两句话 `已确认` 不能混写：

- “仓库口径已同步”
- “新主线样片已验证成立”

前者只表示：

- 项目口径
- 执行口径
- 路由口径
- 接手口径

已经写回仓库。

后者必须依赖真实样片与回审结果，当前不得因为文档同步完成就写成已成立。

## 7. 当前一句话判断

当前《视频工厂》已经过了“技术可不可行”的阶段，正式默认主线已切到“人物 + 用户真实录制素材 + 少量 PPT / 图片”，结构跟着文案走，人物次数由 block 路由决定；demo 只保留链路锚点身份，不再充当质量样片。
