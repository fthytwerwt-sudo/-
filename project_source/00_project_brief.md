# 视频工厂｜GPT 正式资料总述

## 1. 文件定位
本文件是 `GPT 数据源` 根目录的总述文件。

它只负责说明 4 件事：
- 这套资料现在服务什么
- 当前唯一正式主需求是什么
- 哪些旧读法必须停止
- 当前正式事实、目标态计划、历史残片、临时执行说明分别放在哪里

它不负责：
- 每轮执行日志
- 仓库分支状态
- 代码实现细节
- 临时命令或一次性执行单

## 2. 当前唯一正式主需求
`已确认` 当前唯一正式主需求固定为：
- `API 生成真人`
- `用户录制素材`
- `少量 PPT`
- `云端剪辑`

一句话：

**当前正式主线必须统一写成“API 生成真人 + 用户录制素材 + 少量 PPT + 云端剪辑”。**

## 3. 当前默认理解
`已确认` 当前默认理解如下：
- `API 生成真人` 负责开头进入、关键判断、转场衔接、结尾收束
- `用户录制素材` 负责中段主体推进、过程证据、真实工作流证明
- `少量 PPT` 负责关键词显影、结构整理、结尾总结
- `云端剪辑` 是当前正式主需求里的剪辑与组装方向
- 结构跟着文案走，`API 生成真人段` 出现 1 次还是 2 次，是路由结果，不是预设模板

## 4. 当前明确不是默认主线的内容
`已确认` 以下内容不得再写成当前默认主线：
- `pure PPT / 信息卡全片承载`
- `真人实拍出镜默认主线`
- `旧版 talking avatar / 数字人口播路线默认主线`
- `local assembly / local mp4 默认正式交付口径`

## 5. 当前必须停掉的偷换
`已确认` 当前必须停掉以下偷换：
- 把 `API 生成真人` 偷换成 `真人实拍出镜`
- 把 `真人实拍出镜` 偷换成任何“只要像真人就算”的泛称
- 把 `用户录制素材` 偷换成“用户本人必须正脸出镜”
- 把 `少量 PPT` 偷换成“PPT 重新承担整条主体”
- 把 `云端剪辑` 偷换成“当前仓库或代码链路已经稳定跑通”

## 6. 四层分流
这套资料现在强制分成 4 层：

1. `当前正式事实`
   - 只写用户已拍板、当前资料包已经统一采用的正式口径
   - 统一看：`project_source/07_current_formal_facts.md`

2. `目标态计划`
   - 只写未来要达到、但不能冒充已成立的方向
   - 统一看：`project_source/09_target_state_plan.md`

3. `历史说明 / 旧口径`
   - 只允许作为历史说明保留在项目脑对应文件里
   - 不得再当成当前正式主线入口

4. `临时执行说明`
   - 不进入本资料包根目录
   - 继续留在仓库执行层、日志层或聊天上下文

## 7. 当前最小入口
新会话若只读最小集合，固定先读：
1. `project_source/00_project_brief.md`
2. `project_source/02_term_definitions_and_state_boundaries.md`
3. `project_source/07_current_formal_facts.md`
4. `project_source/24_human_self_footage_light_ppt_routing_rules.md`

## 8. 一句话入口
