# 录制素材与 Prompt 交付规则

## 1. 文件定位
本文件只负责 4 件事：
- 以后 GPT 给文案时，必须附带哪些录制交付
- 录制素材清单怎么写
- “录制动作说明”和“豆包输入 prompt”如何区分
- 录完后用户应回传哪些内容给 GPT 复审

它不负责：
- 单条样片的最终镜头表
- 发布文案全文
- 当前待发对象切换
- 代码实现

## 2. 以后 GPT 给文案时，必须附带哪些录制交付
只要当前输出的是 `内容表达文案`，默认必须同时交付：
1. 完整脚本
2. `block -> 承载分配`
3. 录制素材清单
4. 录制动作说明
5. 豆包输入 `prompt`
6. 结果差证明点
7. `PPT / 总结卡文案`
8. 录制验收清单

少其中任一项，都只能算部分交付。

## 3. 录制素材清单怎么写
录制素材清单默认至少写清：
- 要录哪一段
- 录什么页面或工具
- 这一段要证明什么
- 需要出现什么输入
- 需要出现什么结果
- 这一段录完后怎样判断“可用”

建议格式如下：
- 素材名
- 页面 / 工具
- 目标证明点
- 必出镜输入
- 必出镜结果
- 录制完成判定

## 4. “录制动作说明”和“豆包输入 prompt”如何区分
### 4.1 录制动作说明
录制动作说明写的是：
- 人要怎么操作
- 先开哪个页面
- 先点什么
- 先贴哪段内容
- 哪一步要停留
- 哪个结果一定要录到

它本质上是：
- 给拍摄与录屏动作看的执行单

### 4.2 豆包输入 `prompt`
豆包输入 `prompt` 写的是：
- 具体要贴给豆包的文本
- 输入结构怎么组织
- 普通输入版本是什么
- 工作包输入版本是什么
- 修结果版本是什么

它本质上是：
- 给工具看的输入内容

必须明确：
- 录制动作说明不等于豆包输入 `prompt`
- 豆包输入 `prompt` 也不能替代录制动作说明

## 5. 录完后用户应回传哪些内容给 GPT 复审
录完后，用户默认应回传：
1. 原始录屏或已裁好的关键片段
2. 实际贴入豆包的 `prompt`
3. 普通输入 vs 工作包输入的结果截图或文本
4. 需要保留的结果差证明点
5. 录制中遇到的问题或偏差
6. 当前认为可用 / 不可用的素材判断
7. 录制验收清单勾选结果

若这些信息缺失较多，GPT 只能做部分复审，不能冒充完整复审。

## 6. 一句话规则
**以后文案默认不是只交一版脚本，而是默认连“录制素材包 + 录制动作单 + 豆包 prompt 包 + 结果差证明点 + 验收清单”一起交；其中录制动作说明写的是人怎么录，豆包输入 `prompt` 写的是工具吃什么输入。**
