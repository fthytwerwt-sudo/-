# 文案路由规则

## 1. 文件定位
本文件不是统一文案模板，而是当前正式资料包的文案路由规则。

它负责：
- 先判断这轮文案属于哪一类
- 再判断这轮内容如何分配给 `API 生成真人`、`用户录制素材`、`少量 PPT`
- 再决定是否能进入样片执行

## 2. 文案一级分类
当前文案先分 4 类：
1. `内容表达文案`
2. `分发包装文案`
3. `关系建立文案`
4. `变现承接文案`

当前默认优先级：
1. `内容表达文案`
2. `分发包装文案`
3. `关系建立文案`
4. `变现承接文案`

## 3. 当前默认主流程只优先服务什么
当前根目录正式主线，默认只优先服务：
- `内容表达文案`

原因很简单：
- 当前要先把主线口径收稳
- 先把默认承载写清
- 不让分发、关系或变现文案反过来抢走主线

## 4. block 承载路由
`内容表达文案` 进入主流程后，必须继续拆 block。

### 常见 block
- 开头进入
- 问题点破
- 过程演示
- 关键判断
- 结构整理
- 结尾收束

### 默认承载映射
| block | 默认承载 |
|---|---|
| 开头进入 | `API 生成真人` |
| 问题点破 | `API 生成真人` 或 `API 生成真人 + 用户录制素材` |
| 过程演示 | `用户录制素材` |
| 关键判断 | `API 生成真人` |
| 结构整理 | `少量 PPT` |
| 结尾收束 | `API 生成真人` 或 `少量 PPT` |

## 5. `API 生成真人段` 的出现次数
当前只能这样写：
- `API 生成真人段 1 次`
- `API 生成真人段 2 次`

当前不能这样写：
- “只写 1 次，却不说明是 API 生成真人段”
- “只写 2 次，却不说明是 API 生成真人段”

规则：
- `API 生成真人段` 出现 1 次还是 2 次，是 block 路由结果
- 不是固定模板
- 也不是默认一定要更多“像人”的镜头

## 6. `真人实拍出镜` 的位置
若某条内容必须使用 `真人实拍出镜`，必须同时满足：
1. 显式写出 `真人实拍出镜`
2. 显式标注为非默认分支
3. 不得把它回写成当前唯一正式主线

## 7. 进入样片执行前的 7 条检查
1. 这轮文案是哪一类？
2. 当前是否仍属于 `内容表达文案` 主流程？
3. 开头是否明确交给 `API 生成真人`？
4. 中段是否真的有 `用户录制素材` 承担主体推进？
5. `少量 PPT` 是否只承担结构整理与收束？
6. 是否仍能在不默认 `真人实拍出镜` 的前提下成立？
7. 是否没有把 `云端剪辑` 写成“已跑通事实”？

## 8. 默认改稿顺序
当前默认按下面顺序改：
1. 先改开头判断句
2. 再改中段证据层
3. 再改 block 承载分配
4. 再改结尾动作或自检句
5. 最后才改语气和节奏

## 9. 一句话规则
**当前文案路由不是“先写一版再找东西塞进去”，而是先判断文案类型，再把开头交给 `API 生成真人`、把中段交给 `用户录制素材`、把总结交给 `少量 PPT`；凡是不定义承载就使用未定义人像称呼的做法，不再进入正式主文件。**
