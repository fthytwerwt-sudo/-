# 22｜文案模式路由规则

## 1. 文件定位

本文件用于规定：

- 不同类型 AI 知识类内容如何路由
- 不同类型内容为什么不能共用一种结构
- 不同类型内容为什么不能共用一种证据
- 不同类型内容为什么不能共用一种结尾总结卡

它不是：

- 固定统一模板
- 展示承载规则全文
- 单条样片执行单

## 2. 当前正式路由顺序

当前默认顺序固定为：

1. 先分内容类型
2. 再分场景
3. 再拆 block
4. 再决定每个 block 该由谁承担
5. 最后才决定结尾总结卡类型

明确禁止：

- 不分类型就直接套统一模板
- 不分证据类型就直接写文案
- 不分结尾目标就直接做总结卡

## 3. 四类内容不同结构 / 不同证据 / 不同结尾卡

### 3.1 AI 项目讲解

- 更适合的结构：
  - 问题 → 动作 → 结果
  - 现象 → 原因 → 做法
- 更适合的证据：
  - 项目卡点
  - 前后变化
  - 关键动作为什么成立
- 默认结尾卡：
  - `judgment_card + 最小行动`

### 3.2 AI 方法分享

- 更适合的结构：
  - 旧方式 → 新方式
  - 提问 → 拆解 → 结论
- 更适合的证据：
  - 方法步骤
  - 易错点
  - 方法适用边界
- 默认结尾卡：
  - `steps_error_card`

### 3.3 AI 学习实操

- 更适合的结构：
  - 问题 → 动作 → 结果
  - 最小步骤 → 关键动作 → 自检
- 更适合的证据：
  - 操作步骤
  - 关键界面 / 动作
  - 自检标准
- 默认结尾卡：
  - `steps_card + 自检句`

### 3.4 AI 案例拆解

- 更适合的结构：
  - 案例前 → 案例后
  - 问题 → 动作 → 结果
- 更适合的证据：
  - 案例前后变化
  - 关键动作
  - 可迁移条件
- 默认结尾卡：
  - `judgment_card + 可迁移句`

## 4. block 路由继续成立

当前 block 路由规则继续成立：

- 人物出现 1 次还是 2 次，不预设
- 中段主体默认优先给自录素材承担
- PPT / 图片只负责整理 / 显影 / 总结
- 不允许让 PPT / 图片重新抢回主体叙事

## 5. 文案执行检查单

任何进入样片执行的文案，至少先检查：

1. 用户看完后能做什么
2. 用户看完后能判断什么
3. 用户看完后能自检什么
4. 这条内容的证据是什么
5. 当前结尾总结卡类型是否与内容类型匹配

## 6. 当前一句话规则

当前文案路由不是统一模板，而是先分内容类型、再分证据和结尾卡；block 路由继续成立，中段主体默认优先给真实素材承担。
