# 三层 Prompt 交付与尾卡规则

## 1. 文件定位
本文件只负责：
- 定义为什么不再只交单一 `prompt`
- 定义三层 `prompt` 各自负责什么
- 定义 `Prompt 引用尾卡`

## 2. 三层 `prompt`
- 第 1 层：主 `prompt`
  - 把一句糊话压成更像可交付物的初稿
- 第 2 层：修结果 `prompt`
  - 把“看起来对但不能交”的版本补成更稳版本
- 第 3 层：展示与尾卡 `prompt`
  - 服务总结卡与 `Prompt 引用尾卡`

## 3. `Prompt 引用尾卡`
- 默认放在结尾总结之后
- 只承担产品单元引用
- 不承担主叙事
- 不代替中段证据

## 4. 与 4 段核心录制素材的关系
- 素材 1：证明为什么需要第 1 层主 `prompt`
- 素材 2：证明主 `prompt` 执行过程
- 素材 3：证明第 2 层修结果 `prompt`
- 素材 4：证明结果差与第 3 层展示 / 尾卡 `prompt`
