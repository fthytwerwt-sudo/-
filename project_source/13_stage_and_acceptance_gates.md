# 阶段与验收闸门规则

## 1. 文件定位

本文件用于写死当前项目在阶段推进与验收收尾时，哪些条件算过线，哪些条件还不能收口。

## 2. 当前验收闸门

当前项目至少区分以下 3 类闸门：

- 技术链闸门
- 质量闸门
- 样片交付闸门

## 3. 技术链闸门

技术链闸门通过，表示：

- TTS / 图片 / 视频 / assembly 等链路已跑通
- 但这不等于内容质量已过线
- 也不等于用户已经收到可见样片

## 4. 质量闸门

质量闸门通过，至少指以下任一状态成立：

- 项目内部“90 分水位”通过
- `quality_passed`
- 通过“可发布测试线”

但必须明确：

- 质量闸门通过，不自动等于验收完成
- 若还没有把可见样片交付给用户，当前收尾仍不完整

## 5. 样片交付闸门

当质量闸门通过后，必须立刻进入样片交付闸门。

样片交付闸门通过的最低要求是：

1. 已明确给出本轮样片路径
2. 已明确用户当前应优先看哪个样片文件
3. 若样片属于 `local_only`，已明确说明：
   - 不会上传到 GitHub
   - 但本地已生成
   - 当前本地路径是什么
4. 已说明该样片是否足以完成当前验收

未满足以上 4 条时，不得把收尾写成：

- “已完成验收”
- “已正式过线”
- “用户已可验收”

## 6. 当前默认样片交付物

当前默认样片交付物优先级写死为：

1. `dist/formal_api_demo/final.mp4`
2. 若需要辅助验收，再补：
   - `dist/formal_api_demo/review_frames/`

当前不要把样片交付物写得过散。

## 7. 当前收尾最小回报字段

若本轮达到 90 分 / `quality_passed` / 可发布测试线，收尾必须额外回报：

1. 本轮样片路径
2. 是否属于 `local_only`
3. 用户当前应优先看哪一个样片文件
4. 若 GitHub 无法直接看，必须给本地可见路径

## 8. 当前一句话规则

达到 90 分 / `quality_passed` / 可发布测试线，只表示质量闸门通过；只有把可见样片明确交付给用户后，当前验收收尾才完整。
