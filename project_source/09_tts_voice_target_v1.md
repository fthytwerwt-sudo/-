# TTS 声音目标稿 v1

## 1. 这份文件解决什么问题

这份文件用于统一 `formal_api_demo` 当前阿里 TTS 子线的声音目标。

它回答的是：

- 当前想要什么声音
- 当前明确不要什么声音
- 当前默认技术路线是什么
- 当前哪些误判不成立
- 当前一句话目标是什么

它不负责：

- 代码实现
- API 调用细节
- 执行命令
- assembly 或视频生成

这些属于 `codex_source/` 和代码层。

## 2. 当前声音目标是什么

目标不是仿某个具体博主本人声音，而是做出“军事鉴定 / 装备拆解 / 判断型解说”那种中文男声表达感。

当前目标特征：

- 年轻中文男声，约 25–35 岁感
- 中低音区
- 声线干净利落
- 略带一点轻微磨砂感
- 语速中偏快
- 短句推进
- 重点判断词前有轻微停顿
- 句尾收干净，不上扬，不拖音
- 情绪冷静克制
- 有判断感
- 可带一点轻微讽刺底色
- 信息密度高，废话少

## 3. 当前明确不要什么

当前负向排除：

- 不要播音腔
- 不要客服腔
- 不要广告腔
- 不要温柔陪伴感
- 不要字正腔圆式朗诵
- 不要激情澎湃
- 不要高亢
- 不要句尾上扬
- 不要拖音
- 不要明显演讲感

## 4. 当前默认技术路线是什么

当前默认技术路线先不重开：

- `provider.name = aliyun_bailian`
- `tts.api_route_family = aliyun_bailian_cosyvoice`
- `tts.model = cosyvoice-v3-flash`
- `tts.voice = longanyang`

当前执行口径：

- 已接通阿里 TTS
- 已能真实落出音频文件
- 当前只推进 TTS 子线
- 当前不推进视频生成
- 当前不推进 assembly

## 5. 当前已确认的阶段判断

当前不是“路线错了”的问题，而是：

- 路线已经接通
- 旧 A 方向更对
- 但还不完全对
- 当前主问题是 instruction、节奏和细节参数还要继续压实

因此当前默认动作不是换大路线，而是：

- 先围绕旧 A 做更窄微调
- 先改 instruction 写法
- 再微调 `speech_rate / pitch_rate / volume`
- 只有明确证明当前承载力不够时，才考虑换 voice

## 6. 当前不成立的误判

以下判断当前都不成立：

- “阿里路线本身不对”
- “现在该先切别的模型族”
- “现在该先推进 assembly”
- “现在该先推进视频生成”
- “只要有声音就算完成”
- “只要换 voice 就一定更接近目标”

当前真正成立的是：

- 当前路线已通
- 当前目标已明确
- 当前要做的是把声音标准和执行标准压进文件，并在旧 A 周围继续收窄
- 当前更具体的中文 instruction 并不一定能直接被阿里当前 route / model / voice 接受
- 若 richer instruction 被远端 `InvalidParameter / 428` 拦下，问题首先属于 instruction 承载边界，不等于整条路线无效

## 7. 当前一句话目标

用 `longanyang + cosyvoice-v3-flash` 在阿里 CosyVoice 路线上做出更接近“冷静、利落、判断型解说”的中文男声，让下一轮只剩最后收口，而不是重新定义方向。
