# Docker 工作台与录制减负规则

## 1. 文件定位
本文件只负责：
- 定义 Docker 最小工作台的角色
- 定义中间层工作流顺序
- 定义录制减负的默认做法

## 2. Docker 工作台的正确角色
当前 Docker 工作台只负责把：
1. `01_场景简报.md`
2. `02_工作包正文.md`
3. `03_Prompt包.md`
4. `04_录制计划.md`
5. `05_回审清单.md`

串成最小中间层工作台。

它不是：
- 替代判断
- 替代用户录制
- 全自动成片系统

## 3. 录制减负默认规则
- 人只录 `4 段核心录制素材`
- AI 先整理工作包正文、三层 prompt 包、结果差骨架与 `Prompt 引用尾卡`
- 当前只承诺中间层 scaffold，不承诺最终成片自动化

## 4. Compose 最小要求
- 至少提供一个 `workbench` 服务
- 用 bind mount 挂载整个仓库
- 可补一个本地状态 volume
- 当前不接外部 API
- 当前不接 Perplexity
