# 场景模式模板

## 1. 总原则
本项目按“场景模式”运行，而不是按固定唯一结构运行。

当前高频核心仍是前 4 类：
1. AI项目讲解
2. AI方法分享
3. AI学习实操
4. AI案例拆解

统一前提：
- 结构跟着文案走
- `API生成真人段` 出现 1 次还是 2 次，是路由结果，不是预设模板
- 中段主体默认优先交给 `用户录制素材`

## 2. AI项目讲解（P1）
- 最适合讲：
  - 某个 AI 项目解决什么问题
  - 一个模糊想法如何被压成可执行方案
  - 一个项目从卡住到推进的关键转折
- 推荐结构：
  - 问题 → 动作 → 结果
  - 旧方式 → 新方式
  - 现象 → 原因 → 做法
- 当前更适合的表现方式：
  - `API生成真人 + 项目过程素材 + 少量总结卡`
  - `API生成真人 + 项目截图/录屏 + 少量PPT`
- 默认判断段策略：
  - 常见为 2 次
  - 第 1 次负责进入和可信度
  - 第 2 次负责关键判断或收束

## 3. AI方法分享（P1）
- 最适合讲：
  - 一个可复用的方法
  - 一个降低混乱、返工或上手门槛的做法
- 推荐结构：
  - 旧方式 → 新方式
  - 提问 → 拆解 → 结论
  - 结论先行 → 过程补充
- 当前更适合的表现方式：
  - `API生成真人` 开头 + `用户录制素材` 主体 + `少量PPT` 收束
- 默认判断段策略：
  - 常见为 1 次或 2 次
  - 如果中段证据非常顺，1 次就够
  - 如果中段有关键转折，2 次更稳

## 4. AI学习实操（P1）
- 最适合讲：
  - 新手最容易卡住的环节
  - 从不会到能做出结果的最小实操路径
- 推荐结构：
  - 提问 → 拆解 → 结论
  - 现象 → 原因 → 做法
- 当前更适合的表现方式：
  - `API生成真人` 轻开头 + 屏幕演示主体 + 结尾总结卡
- 默认判断段策略：
  - 通常 1 次更常见
  - 开头负责设置预期
  - 中段尽量交给 `用户录制素材`

## 5. AI案例拆解（P1）
- 最适合讲：
  - 一个案例是怎么成立的
  - 案例前后差异
  - 某个结果背后的关键动作
- 推荐结构：
  - 案例前 → 案例后
  - 问题 → 动作 → 结果
- 当前更适合的表现方式：
  - `API生成真人 + 案例截图/录屏 + 拆解总结卡`
- 默认判断段策略：
  - 1 次或 2 次都可成立
  - 案例本身很强时，1 次足够
  - 案例有多个判断转折时，2 次更稳

## 6. AI工具实测（P2）
- 更适合：
  - `API生成真人` 给判断
  - `用户录制素材` 承担工具过程
  - `少量PPT` 承担对比总结

## 7. AI问题回答（P2）
- 更适合：
  - `API生成真人` 短答
  - 局部演示或截图补证据
  - 一张结论卡收束

## 8. AI观点拆解（P2）
- 更适合：
  - `API生成真人` 给边界判断
  - 少量案例图 / 结果卡补证据

## 9. AI日常工作流分享（P2）
- 更适合：
  - `API生成真人` 开头进入
  - 工作流录屏 / 真实工作片段承载中段
  - `少量PPT` 做结尾整理

## 10. 生活片段辅助的 AI 表达型内容（P2）
- 更适合：
  - 局部生活切片增强代入
  - 但不应抢走当前主线
- 硬规则：
  - 生活切片只能做辅助
  - 不能把项目主轴带偏

## 11. 一句话规则
**每条视频都先判场景，再选结构，再定承载；默认优先考虑 `API生成真人` 承担什么、`用户录制素材` 承担什么、`少量PPT` 只在哪些位置最值。**
