# 选题与文案规则

## 0A. 正式运营约束

当前阶段为 `formal_operation_active`。选题与文案修改必须先读取 `codex_log/current_operation_target.md`、`review_loop/operation_records_index.md` 和 `codex_log/current_data_goal_anchor.md`；数据不足时不得把正式运营样本写成方向成立或内容通过。

若选题 / 文案任务继续指向做视频、产视频、发片候选、运营内容或下一条视频，交付基线必须切换到 `publish_candidate（可发布候选片）` 或 `blocked（阻断）`。文案准备包、内容路由卡、时间线映射、TTS 韵律锚点、剪辑 / 装配决策包只属于执行前必备条件，不能被写成视频交付完成；技术预览、无声预览、横屏技术包或只交 JSON / Markdown 包只能作为内部诊断。

## 1. 文件定位
本文件只解决 3 件事：
- 当前什么选题值得进入主流程
- 文案怎样先压成适配当前正式主线的版本
- 当前用户偏好的最关键稳定结论是什么

它不负责：
- 当前待发对象切换
- Docker / compose 技术细节
- 最终成片执行命令

## 2. 当前文案前置层唯一适配要求
一条内容要进入当前主流程，至少要同时满足：
1. 能安排 `API 生成真人段`
2. 能提供 `用户录制素材`
3. 能用 `少量 PPT` 收住关键词或总结
4. 不需要把 `真人实拍出镜` 当默认前提
5. 不需要把 `pure PPT` 拉回主体叙事
6. 能提供 `真实 AI 使用经验`、`工作提效证据`、`前后变化` 中的至少一项强证明点

## 3. 当前阶段优先主打的内容类型

### 3.1 `AI 职场任务现场拆解`
当前优先。

更适合的原因：
- 更能展示“把一句糊话压成可执行结构”的真实能力
- 更容易放进具体任务、具体动作、具体结果
- 更容易让 `用户录制素材` 提供中段强证据
- 更容易讲清楚哪里节省了重复劳动、哪里出现了真实结果差

### 3.2 `AI 问题点破`
当前优先。

更适合的原因：
- 更容易从真实误判切入
- 更容易让开头判断句成立
- 更容易把“问题为什么会发生”压成一个可执行动作

## 4. 当前高优先级选题来源
- 高频具体职场场景
- 真实失败体验
- 有清楚前后差值的题
- 能真实录到中段证据的题
- 能自然拆成 `真实动作 + 结果差 + 最小动作`，并在必要时再沉淀工作包的题

当前更适合优先考虑：
- 模糊需求压稿
- 汇报 / 周报 / 项目方案
- 会议纪要变待办
- 材料整理与结构重写
- 为什么你用了 AI 最后还是自己重写

## 5. 当前不优先的题
- `AI 日常工作流分享`
- 纯工具新闻
- 纯模型更新播报
- 纯抽象方法论
- 只能靠 `pure PPT` 承载的题
- 必须默认 `真人实拍出镜` 才成立的题

## 6. 当前稳定文案偏好结论
以下是从旧 `30-36` 文案知识库吸收出的最关键稳定结论：

### 6.1 开头偏好
- 用户更偏好：`结论先拍桌`
- 次优先：`结果先交代`
- 可用补充：`对比拉开型`
- 暂不默认：长叙事、重悬念式开头

### 6.2 判断句偏好
- 用户偏好明显有立场的判断句
- 更像“先定性 -> 再比较 -> 再给选择”
- 不喜欢书面报告腔

### 6.3 过渡偏好
- 更稳的是：
  - `问题推进型`
  - `举例切换型`
  - `编号展开型`
- 不追求漂亮辞藻，更追求顺

### 6.4 收尾偏好
- 更偏好：
  - 一句话收束观点
  - 给一个低门槛动作
- 不默认强卖课、强关注、强会员导向

### 6.5 当前一句风格理解
**用户偏好 `有判断、有人味、有动作` 的表达。**

## 6A. 后续默认文案生产流程

以后《视频工厂》AI 知识类视频文案默认按以下流程推进：

1. `Perplexity（外部参考检索 / 研究工具）` 只负责输出 `reference pack（参考包）` 和 `raw feeling draft（原感初稿）`。
   - 作用：提供外部参考、表达方向、案例、梗感和原始感觉。
   - 边界：不是最终稿，不得直接进入视频、字幕或声音执行。
2. 用户根据参考方向录制素材。
   - 素材可以是录屏、网页操作、结果对比、过程证据、关键截图。
   - 素材是最终文案的硬依据。
3. `Codex（执行代理）` 做素材技术检查与细节汇报。
   - `Codex（执行代理）` 不负责最终文案定稿。
   - `Codex（执行代理）` 必须把素材里可写进最终稿的细节报告给 `ChatGPT（最终落稿与复审入口）`。
   - 汇报不能只写“文件存在、时长正常、技术通过”，必须写清真实素材里有什么、在哪一秒、发生了什么、能证明什么。
4. `ChatGPT（最终落稿与复审入口）` 基于 `Perplexity（外部参考检索 / 研究工具）` 参考包、用户素材和 `Codex（执行代理）` 细节报告写最终落稿。
   - 最终稿唯一默认出口是 `ChatGPT（最终落稿与复审入口）`。
   - 可以重排、压缩、改写、加口语、加微反转、加真实细节。
   - 必须贴真实素材，不能写素材里没有的动作、页面和画面。
5. `Codex（执行代理）` 再按 `ChatGPT（最终落稿与复审入口）` 最终稿执行视频、字幕、音频、卡片和审片包。
   - 执行后回到 `ChatGPT（最终落稿与复审入口）` 复审。

硬规则：
- `Perplexity（外部参考检索 / 研究工具）` 初稿不能被当成最终稿。
- `Codex（执行代理）` 素材检查不能被当成 `Codex（执行代理）` 写最终文案。
- `ChatGPT（最终落稿与复审入口）` 不能在缺少素材细节时直接写最终落稿。

## 6B. 当前最终文案风格锚点

当前最终风格锚点为：
- 用户确认的“用字更自然版长稿”
- 20260427 B 版“停顿梗感”试听方向

稳定风格要求：
1. 生活观察起手。
2. 微反转，不做极限反转。
3. 说话带一点梗，但不要刻意装幽默。
4. 自然口语，像真人能直接说出口。
5. 有轻吐槽，但不攻击用户。
6. 保留真实细节，不写空泛概念。
7. 避免 AI 感硬词。
8. 不写课程腔。
9. 不写广告腔。
10. 不写鸡血短视频腔。

明确禁用或慎用：
- `下一步从哪打`
- `开门钥匙`
- `把路铲出来`
- `一团雾`
- `强势赋能`
- `精准击穿`
- `底层逻辑`
- `高效闭环`
- `抓手`
- `撬动`
- `价值闭环`
- `强势破局`

推荐保留的表达方向：
- `陪资料加班`
- `第一行没开`
- `看着像方案，用起来像空气`
- `永远不会错，也永远没用`
- `普通问法给废话，Prompt（提示词）调对给初稿`
- `别再跟空白页硬刚`
- `先帮你把顺序理出来`
- `下一步从哪开始`

## 7. 当前 vNext 文案壳要求
- 正式主线结构不变，但文案壳可以升级
- 当前默认同时考虑：
  - `opening_route_decision（开头路由判断）`
  - 开头人物壳
  - 中段结果差证据
  - 结尾总结卡
  - 结尾总结后的 `Prompt 引用尾卡`
- 开头不再只有元素娃娃路线；选题进入主流程前，必须先判断本条更适合哪种 opening route。
- 普通讲解 / 复盘 / 低压陪伴型选题，可以继续使用 `element_doll_opening（元素娃娃开头）`。
- 争议 / 情绪 / 商业焦虑型选题，且开头超过 `3s`，可优先考虑 `meme_gif_opening_hook（梗图 GIF 开场钩子）`。
- 主题一句话足够强时，可考虑 `direct_question_title_card（直接问题标题卡）`。
- 开头真实画面本身有强结果差或强证据时，可考虑 `screen_first_opening（录屏现场先行开头）`。
- 结尾总结壳仍可默认统一使用：`Minecraft-inspired 原创体素方块风`。

## 7A. 开头路线判断边界

`opening_route_decision（开头路由判断）` 必须先回答：

1. 开头时长是 `<= 3s` 还是 `> 3s`？
2. 话题情绪强度是 low / medium / high？
3. 争议程度是 low / medium / high？
4. 开头证据强度是否足以直接用录屏先行？
5. 本条是否强依赖品牌一致性和轻陪伴进入？
6. 核心问题能否一句话钉住？

边界：

- `element_doll_opening（元素娃娃开头）` 保留，但不得写成所有内容唯一默认开头。
- `meme_gif_opening_hook（梗图 GIF 开场钩子）` 只在需要抓眼、起情绪、抛问题时优先，不得写成新默认死流程。
- 开头 GIF / 梗图只负责停留和问题，不负责证明。
- 强证据仍来自 `用户录制素材`、前后对比、步骤截图、结果截图或平台数据。

## 8. 进入文案阶段前必须先回答的 10 个问题
1. 用户看完后最小能做什么，或最小能判断什么？
2. 这条内容最关键的强证据是什么？
3. `opening_route_decision（开头路由判断）` 选择哪一种开头路线，为什么？
4. 中段最值的一段证据，是否真的能由 `用户录制素材` 提供？
5. 结尾是否能被 `少量 PPT` 或一句清楚收束句收住？
6. 这条内容是否在没有 `真人实拍出镜` 的前提下仍然成立？
7. 这条是否有 `真实 AI 使用经验`？
8. 这条是否有 `工作提效证据`？
9. 这条是否有可感知的 `前后变化`？
10. 这条内容是否值得沉淀为工作包，还是当前只做内容验证？

## 9. 一句话规则
**当前选题和文案，不是先找一个“看起来能讲”的题，而是先找一个能同时接住“开头路线判断 + 用户录制素材中段证据 + 少量 PPT 结尾收束”，并且能录出 `真实 AI 使用经验`、`工作提效证据`、`结果差` 与 `最小动作` 的题；开头固定机制、不固定素材，若后续出现明确复用价值，再沉淀为工作包。**

正式运营阶段的文案若进入视频交付，必须能支撑竖屏 9:16、有口播或明确可发布音轨、有字幕、清楚开头、中段证据、结尾收束、基础人感质量和发布前复审；做不到这些基础线时，文案链路应返回 `blocked_publish_candidate_unavailable`，不得继续下发低标准技术预览。
