# 项目回审模板

## 1. 回审总原则
本项目的回审不是为了泛泛打分，而是为了定位问题层级并只收一个最高优先级改点。

当前回审必须同时检查：
- `API生成真人段` 是否真的承担了进入或判断
- `用户录制素材` 是否真的承担了主体推进
- `少量PPT` 是否只做整理与总结
- 整条是否还像 demo、说明书或课件

## 2. 最小可用回审模板
```text
请帮我对这条 AI 短视频做“最小可用回审”。

你不要打分，只回答下面 6 个问题：
1. 主要问题更像出在哪一层：
   - 场景
   - 结构
   - 心理机制
   - 画面表现
   - 字幕 / 配音 / 节奏
2. API生成真人段是否真的承担了信任或关键判断，还是只是空出场
3. 中段主体是否真的由用户录制素材承担，还是素材很碎 / 很虚
4. 少量PPT有没有越权抢主叙事，重新滑回课件感
5. 观众最可能在哪一刻失去兴趣
6. 如果这轮只能改一个点，最该改什么，为什么

输出时只保留一个最高优先级改点。
```

## 3. 快速初审模板
```text
请对这条 AI 视频做快速初审。

请按下面结构输出：
1. 一句话结论：值不值得继续做
2. 当前主要问题层级
3. 开头是否有效
4. API生成真人段出场是否有价值
5. 中段主体是否成立
6. 少量PPT是否清楚但不过课件
7. 这轮只改一个点的话改哪里
```

## 4. 正式交付审模板
```text
请把这条 AI 短视频按“正式交付审”来审。

必须逐条判断：
1. 脚本是否成立
2. 开头是否有效
3. API生成真人段是否承担了信任、判断或收束
4. 用户录制素材是否承担了主体推进和过程证据
5. 少量PPT是否只做关键词显影、结构整理和结尾总结
6. 前后变化是否可见
7. 结构是否适配当前场景
8. 心理机制是否真正接住人
9. 字幕和配音是否协同
10. 整条是否还像 demo / 说明书 / 硬拼接

最后只输出：
- 是否达到“可发布测试”水位
- 如果还没达到，唯一最优先改点是什么
```

## 5. 文案专项回审模板
```text
请只从“文案层”审这条视频脚本。

重点检查：
1. 开头是不是废话开头
2. 中段是不是说明书腔
3. API生成真人段是不是承担了真实判断，而不是口播填空
4. 用户录制素材段是不是有过程感和现场感
5. 结果是不是说了等于没说
6. 有没有一句能真正命中目标用户身份
7. 有没有给到最小行动或掌控感
```

## 6. 画面专项回审模板
```text
请只从“画面层”审这条视频。

重点检查：
1. API生成真人段出场是不是有进入感 / 信任感，而不是摆件
2. 用户录制素材有没有承担主体推进，而不是碎片切换
3. 少量PPT是不是清楚、有条理，但不过课件
4. 整条有没有硬拼接感
5. 辅助素材是否加分还是抢戏
6. 画面有没有承接脚本里的前后变化
```

## 7. 字幕与配音节奏专项回审模板
```text
请只从“字幕与配音节奏层”审这条视频。

重点检查：
1. 配音是不是太像系统播报
2. 停顿是否合理
3. 字幕分段是否利于理解
4. 字幕出现与 API生成真人段 / 用户录制素材 / 少量PPT 的切换是否协调
5. 是否有哪一段显得太赶、太平或太碎
6. 节奏有没有帮助观众处理信息，而不是增加负担
```

## 8. 场景与结构回审模板
```text
请只从“场景与结构匹配度”审这条视频。

请判断：
1. 这条内容实际上属于哪个场景模式
2. 当前结构是不是选对了
3. 当前 API生成真人段、用户录制素材、少量PPT 的职责分工是不是合理
4. 这条更适合 1 次 API生成真人段还是 2 次 API生成真人段
5. 当前最像哪类场景
6. 结构有没有把最重要的变化讲出来
```

## 9. 改后复审模板
```text
请对这条视频做“改后复审”。

背景：
- 上一轮最优先改点是：[贴这里]
- 这一轮已经改完

请只回答：
1. 这个问题有没有真正改善
2. API生成真人段 / 用户录制素材 / 少量PPT 的职责分工有没有更清楚
3. 改完后有没有引入新的更大问题
4. 现在新的最高优先级问题是什么
```

## 10. 模糊反馈诊断模板
```text
用户的反馈是：
“不对 / 有点空 / 差点意思 / 不像人话 / 还是像 demo / 有点像课件 / 拼得有点碎”

请不要泛泛解释，请把这个模糊反馈拆成最可能的 5 类问题：
1. 场景没判准
2. 结构不成立
3. API生成真人段没有承担信任或判断
4. 用户录制素材没有承担主体推进
5. 少量PPT太重、太硬、太课件
```

## 11. 一句话规则
**回审的核心不是列更多建议，而是先定位层级，再只抓一个最优先改点，同时强制检查 `API生成真人段 / 用户录制素材 / 少量PPT` 的职责分工。**
