# 术语定义与状态边界

## 1. 文件定位
本文件只负责两件事：
- 锁清项目内哪些词现在可以正式使用
- 锁清哪些状态现在可以正式声明

## 2. 核心术语

| 术语 | 当前允许的正式含义 | 当前不允许被偷换成 |
| --- | --- | --- |
| `API 生成真人` | 用于承担开头进入、关键判断、转折衔接、结尾收束的人像段 | `真人实拍出镜`、未定义人像泛称 |
| `真人实拍出镜` | 真实拍摄设备采集的人像段 | `API 生成真人`、当前默认主线 |
| `用户录制素材` | 用户录制的工作流证据素材，包括录屏、操作过程、真实工作片段、现场过程证据 | 用户本人必须正脸出镜、`API 生成真人` |
| `少量 PPT` | 只承担关键词显影、结构整理、结尾总结的辅助承载 | `pure PPT` 全片承载 |
| `云端剪辑` | 当前正式主需求里的剪辑与组装方向表述 | “当前链路已稳定跑通”的事实 |
| `证明壳 / 入口壳` | 视频在产品结构里的默认角色 | `产品单元` |
| `产品单元` | 默认承接变现与交付的工作包单元 | 单条视频本身 |
| `三层 prompt 包` | `主 prompt + 修结果 prompt + 展示 / 尾卡 prompt` 的组合交付 | 单一 `prompt` |
| `Prompt 引用尾卡` | 放在结尾总结之后、只承担产品单元引用的一张尾卡 | 主叙事 / 中段证据 |
| `4 段核心录制素材` | 为减轻录制负担而收束出的 4 段最值素材 | 全量碎片化录制任务 |
| `Minecraft-inspired 原创体素方块风` | inspired-only 的原创体素视觉外壳 | 官方资产复用方案 |
| `CPS` | 副线路径级的建议型商业化判断 | 当前已验证成立的主商业化模型 |

## 3. 状态边界

### `已确认`
- 已完成真实文件落地或真实执行验证
- 不依赖聊天补解释
- 当前资料包已经统一采用

### `部分成立`
- 方向已成立，但仍有一部分没闭环
- 或信息层已落地，但执行链路还没跑通

### `待验证`
- 方向、建议、目标态可以写
- 但没有足够执行证据

### `推测`
- 当前只是猜测或风险提示
- 不得冒充事实

### `technical_validation`
- 只回答：技术链路、脚手架、配置、脚本、命令是否通过
- 不等于内容好看

### `content_validation`
- 只回答：内容表达、结构、尾卡、声音路线是否成立
- 不等于技术链路已经跑通

## 4. 当前禁用写法
- 使用未定义人像泛称
- 把 `API 生成真人` 写成 `真人实拍出镜`
- 把 `三层 prompt 包` 写成单 prompt
- 把 `Prompt 引用尾卡` 写成主体叙事
- 把 `Minecraft-inspired` 写成官方 Minecraft 资产复用
- 把 `云端剪辑`、Docker 工作台或声音路线写成“已稳定跑通 / 已最终确定”而没有执行证据

## 5. 一句话规则
**以后只要进入《视频工厂》的正式根目录文件，就必须先把主线术语、vNext 术语和状态标记写清；任何偷换术语、偷换状态、偷换 inspired 边界的表达，都不再允许进入正式资料。**
