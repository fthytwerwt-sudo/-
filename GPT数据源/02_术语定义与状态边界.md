# 术语定义与状态边界

## 1. 文件定位
本文件专门解决两个问题：
- 哪些词现在可以正式使用
- 哪些状态现在可以正式声明

它是根目录所有正式主文件的共用词典。

## 2. 允许使用的核心术语

| 术语 | 当前允许的正式含义 | 当前不允许被偷换成 |
|---|---|---|
| `API 生成真人` | 由 API 生成的人体呈现段，用来承担开头进入、关键判断、段落衔接、结尾收束 | `真人实拍出镜`、未定义的人像泛称 |
| `真人实拍出镜` | 真实人像由真实拍摄设备采集的出镜段 | `API 生成真人`、未定义的人像泛称、当前默认主线 |
| `用户录制素材` | 用户录制的工作流证据素材，包括录屏、操作过程、真实工作片段、现场过程证据 | `用户本人必须正脸出镜`、`API 生成真人` |
| `少量 PPT` | 只承担关键词显影、结构整理、结尾总结的辅助承载 | `整条主体叙事`、`pure PPT 默认主线` |
| `云端剪辑` | 当前正式主需求里的剪辑与组装方向表述 | “当前仓库链路已稳定跑通”的事实 |
| `旧版 talking avatar / 数字人口播路线` | 旧资料里出现过的旧称，只能作为历史口径讨论 | 当前默认主线、`API 生成真人` 的自动同义词 |

## 3. 禁用写法
以下写法现在都不允许直接进入正式主文件：
- 使用未定义的人像泛称
- 把不同人像承载混成一个词
- 不说明承载类型就直接写“人像段 + 素材 + PPT”
- 写 `cloud-only 已经稳定跑通`
- 写 `云端剪辑 = 当前已验证成立`

## 4. 推荐写法模板

### 当你指 `API 生成真人`
写法：
- `API 生成真人负责开头判断段`
- `本条视频默认由 API 生成真人承担收束`

### 当你指 `真人实拍出镜`
写法：
- `若本条走真人实拍出镜分支，必须显式标注为非默认分支`

### 当你指 `用户录制素材`
写法：
- `用户录制素材负责中段过程证据`

### 当你指 `云端剪辑`
写法：
- `云端剪辑是当前正式主需求里的剪辑方向`
- `云端剪辑是否已经在仓库链路稳定跑通，仍需执行证据`

## 5. 状态边界

### `当前正式事实`
定义：
- 用户已拍板
- 当前资料包根目录已统一采用
- 不依赖聊天补解释

统一看：
- `08_当前正式事实.md`

### `目标态计划`
定义：
- 未来要达到
- 不能冒充已成立
- 允许写方向，但必须写边界

统一看：
- `09_目标态计划.md`

### `历史说明 / 旧口径`
定义：
- 已被吸收进当前中文文件集的对应主题文件
- 不再独立留旧文件

### `临时执行说明`
定义：
- 一次性命令、当前轮次执行单、临时产物说明
- 不进入当前正式资料包根目录

## 6. 一句话规则
**以后只要进入当前正式主文件，就必须先把“API 生成真人 / 真人实拍出镜 / 用户录制素材 / 少量 PPT / 云端剪辑”写清楚；任何使用未定义人像泛称的做法，都不再允许进入正式资料。**
