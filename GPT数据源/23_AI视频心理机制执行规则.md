# 23｜AI视频心理机制执行规则

## 1. 为什么要正式接入心理机制层
本项目后续不再只看：

场景层 → 结构层 → 表现层

而要看：

**场景层 → 结构层 → 心理机制层 → 表现层**

原因：
- 很多 AI 视频不是没信息，而是没打进人脑子
- 很多内容不是技术不对，而是没有让观众快速完成“这跟我有关”“我能听懂”“我知道下一步”的内部判断

心理机制层的作用，不是“拿捏用户”，而是帮助内容更容易被理解、被代入、被行动化。

## 2. 当前优先使用的机制
推荐机制：
1. 身份命中 / 自我代入
2. 认知减负 / 处理流畅
3. 自我效能感
4. 最小行动
5. 掌控感 / 胜任感

次推荐机制：
1. 信息缺口
2. 前后变化对比
3. 轻度损失显性化
4. 轻度新奇感

谨慎使用机制：
1. 泛化焦虑
2. 夸张损失
3. 空洞社会证明
4. 伪权威感
5. 过度制造稀缺感

## 3. 执行型规则

### EXEC-001：先命中“这和我有关”，再给信息
- 开头优先命中一个具体人群、具体卡点或具体状态
- 不要一上来先讲一堆名词

### EXEC-002：优先降低理解负担，而不是堆信息量
- 一屏只保一个主判断
- 一句只传一个核心动作
- 页面切换要有层级变化

### EXEC-003：每条视频至少要给观众一个“我也能做”的感觉
- 中段或结尾必须给一个可执行、低门槛的动作
- 不要把所有建议都抬成“大工程”

### EXEC-004：信息缺口要为理解服务，不要为吊人胃口而吊胃口
- 用在“这件事为什么会卡住”“关键差异是什么”这种位置
- 不要硬造悬念
- 不要标题党

### EXEC-005：前后变化必须能被看见，而不是只被说出来
- 结果页必须有具体变化
- 尽量用对比、列表、前后状态或案例变化承载

### EXEC-006：最小行动优先于完整方案
- 结尾优先落在最小行动
- 不要在一条短视频里塞完所有动作

### EXEC-007：掌控感比焦虑感更适合做长期项目主线
- 更常用“你可以先这样处理”
- 少用“再不做就完了”

### EXEC-008：心理机制必须服从场景，不许反客为主
- 先定场景
- 再定结构
- 再决定补哪一个心理机制最值

## 4. 各层最适合承载什么机制
### 标题层
- 身份命中
- 信息缺口
- 前后变化

### 开头 3 秒
- 身份代入
- 具体卡点
- 处理流畅

### 脚本结构层
- 掌控感
- 自我效能感
- 前后变化

### 字幕分段层
- 认知减负
- 节奏层级

### 配音节奏层
- 处理流畅
- 轻度情绪承接
- 掌控感

### PPT 信息卡或画面层
- 前后变化
- 关键对比
- 认知减负

### 结尾行动层
- 最小行动
- 自我效能感
- 掌控感

## 5. 边界警示
本项目明确警惕：
- 伪心理学包装
- 泛化焦虑
- 空洞社会证明

## 6. 当前阶段最适合优先强化的机制顺序
1. 身份命中
2. 认知减负
3. 自我效能感
4. 最小行动
5. 前后变化可见

## 7. 一句话结论
本项目引入心理机制层，不是为了把 AI 视频变成话术机器，而是为了让内容更容易被看懂、被代入、被行动化，同时始终保留边界感、证据意识和克制感。
