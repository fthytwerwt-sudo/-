# 三层 Prompt 交付与尾卡规则

## 1. 文件定位
本文件只负责 4 件事：
- 定义为什么不再只交单一 `prompt`
- 定义三层 `prompt` 各自负责什么
- 定义 `Prompt 引用尾卡`
- 定义它们与 4 段核心录制素材的关系

它不负责：
- 最终镜头表
- 当前待发对象切换
- TTS 或 provider 选型

## 2. 为什么不再只交单一 `prompt`
当前从“单 prompt”升级到“三层 prompt 包”，原因是：
- 单一 `prompt` 容易把“主输入、修结果、视频展示”混成一团
- 用户会不知道到底该先贴什么、后贴什么
- 视频尾卡也容易没有独立引用层

## 3. 三层 `prompt` 的定义

### 3.1 第 1 层：主 `prompt`
- 作用：先把一句糊话压成更像可交付物的初稿
- 默认要写清：
  - 目标
  - 边界
  - 判断口径
  - 动作顺序
  - 输出格式

### 3.2 第 2 层：修结果 `prompt`
- 作用：把“看起来对但不能交”的版本继续补成更稳的版本
- 默认要补：
  - 缺失判断
  - 缺失边界
  - 缺失风险提醒
  - 缺失输出模板

### 3.3 第 3 层：展示与尾卡 `prompt`
- 作用：服务视频展示、总结卡与 `Prompt 引用尾卡`
- 默认要压：
  - 3 到 5 个总结判断点
  - 可直接显示在尾卡里的最小动作
  - 不抢主叙事的引用句

## 4. `Prompt 引用尾卡`
`Prompt 引用尾卡` 默认放在结尾总结之后。

它负责：
- 引用这套工作包在解决什么问题
- 引用三层 prompt 的职责分工
- 引用用户下一步最小动作

它不负责：
- 再讲一遍整条理论
- 代替中段证据
- 代替结尾总结

## 5. 与 4 段核心录制素材的关系
当前三层 `prompt` 包与 `4 段核心录制素材` 默认一一咬合：
- 素材 1：证明为什么需要第 1 层主 `prompt`
- 素材 2：证明主 `prompt` 执行过程
- 素材 3：证明第 2 层修结果 `prompt`
- 素材 4：证明结果差与第 3 层展示 / 尾卡 `prompt`

## 6. 一句话规则
**以后每条《视频工厂》内容表达文案默认不再只交一个 `prompt`，而是交“主 prompt + 修结果 prompt + 展示与尾卡 prompt”的三层 prompt 包，并在结尾总结之后允许新增一张只承担产品单元引用的 `Prompt 引用尾卡`。**
