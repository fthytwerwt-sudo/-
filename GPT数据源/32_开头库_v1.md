# 开头库 v1

## 当前主结论

- 当前用户偏好的开头，不是先铺背景，而是先把结论、问题或结果扔出来
- 最适合当前项目的开头，是 `结论先拍桌` 和 `结果先交代`
- `极端假设 / 戏剧化开幕` 很抓人，但更适合长版，不适合作为当前主版默认开头

## 使用边界

- 本文件只基于当前“喜欢样本”得出
- 当前属于初步判断，后续可继续补强
- “只喜欢哪一段” 还没补，所以这里先按整条样本观察

## 1. 结论先拍桌型

- 样本来源：`喜欢 1 / 6 / 7`
- 典型开头：
  - “推荐六个……”
  - “技术选型不是炫技，是业务决策。”
  - “记住我这两条结论……”
- 这一类的特点：
  - 上来就给观点，不绕
  - 观众马上知道这条内容值不值得继续看
  - 后面天然容易接编号、框架、标准
- 哪类更适合当前项目：
  - 工具推荐
  - 方法选择
  - 路线判断
  - 知识库 / 工作流 / 表达框架类内容

## 2. 结果先交代型

- 样本来源：`喜欢 9 / 10`
- 典型开头：
  - “我非常震惊……”
  - “我最近每天用三十秒……坚持了一个月……”
- 这一类的特点：
  - 先抛变化、收益或感受
  - 很像真人先说“我到底遇到了什么 / 得到了什么”
  - 后面容易转成“为什么会这样”和“你也可以怎么做”
- 哪类更适合当前项目：
  - 经验分享
  - 方法验证
  - 口语化教程
  - “我试过，所以我告诉你”类表达

## 3. 对比拉开型

- 样本来源：`喜欢 2 / 5 / 9`
- 典型开头：
  - “当别人还在……你早就……”
  - “四大 AI 随便聊一聊吧……”
- 这一类的特点：
  - 先把对象拉开差距
  - 天然带选择感和立场感
  - 很容易把内容做得有判断，不像中性百科
- 哪类更适合当前项目：
  - 工具对比
  - 方案选择
  - 人群分层判断
- 当前使用提醒：
  - 可用
  - 但不要滑进“夸张承诺”或“搞钱话术”

## 4. 极端假设 / 戏剧化开幕型

- 样本来源：`喜欢 3`
- 典型动作：
  - 先丢一个离谱问题
  - 再用反问把观众留住
  - 最后才揭示真正要讲的主题
- 这一类的特点：
  - 抓人非常猛
  - 适合长版叙事
  - 对口播能力和后续铺陈要求高
- 哪类更适合长版，不适合当前主版：
  - 历史 / 叙事 / 大议题解释
  - 需要大量背景展开的内容

## 当前最值得直接复用的开头方向

- 第一优先：`结论先拍桌型`
- 第二优先：`结果先交代型`
- 补充位：`对比拉开型`
- 观察位：`极端假设 / 戏剧化开幕型`

## 当前一句话理解

- 当前项目最稳的开头，不是炫技巧，而是让观众第一句就知道“这条要讲什么、你站什么立场、我为什么要继续听”
