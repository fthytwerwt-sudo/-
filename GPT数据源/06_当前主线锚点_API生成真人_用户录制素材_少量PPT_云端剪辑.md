# 当前主线锚点｜API 生成真人 + 用户录制素材 + 少量 PPT + 云端剪辑

## 1. 文件定位
本文件只负责回答：
- 当前唯一正式主线是什么
- 4 个组成部分各自负责什么
- 为什么旧路线不能再默认回流
- `API 生成真人段` 1 次还是 2 次怎么判断
- 当前 vNext 外壳和声音路线怎么挂到这条主线上

## 2. 当前唯一正式主线
当前唯一正式主线固定写成：

**API 生成真人 + 用户录制素材 + 少量 PPT + 云端剪辑**

这 4 项必须一起理解，缺一项就不是当前完整口径。

## 3. 四个组成部分的默认职责

### `API 生成真人`
默认负责：
- 开头进入
- 关键判断
- 转折衔接
- 结尾收束

### `用户录制素材`
默认负责：
- 中段主体推进
- 操作过程
- 工作流证据
- 前后差值可视化

### `少量 PPT`
默认负责：
- 关键词显影
- 结构整理
- 并列信息总结
- 结尾总结

### `云端剪辑`
当前正式含义是：
- 当前正式主需求里的组装方向
- 不等于“当前链路已经稳定跑通”

## 4. 当前更适合的执行形态
- 当前更适合承载：`AI 职场任务现场拆解`
- `AI 问题点破` 也更适合接入这条主线
- 当前默认结构更适合：`问题 -> 动作 -> 结果`
- 当前中段更适合由 `用户录制素材` 提供真实操作录屏证据

## 5. `API 生成真人段` 1 次 vs 2 次

### 更适合 1 次
- 中段素材本身已经能撑住主体
- 这条更像“看过程、看操作、看案例”
- 结尾用一张总结卡就能收住

### 更适合 2 次
- 中段存在关键判断转折
- 素材之间有断裂，需要一次判断段来接缝
- 结尾更适合用 `API 生成真人` 收束

当前不允许预设：
- 所有视频都只要 1 次
- 所有视频都必须 2 次

  - 更像主持娃娃
  - 不像普通游戏 NPC
  - 不像只是站着说话的僵硬数字人

## 7. 当前 inspired 合法边界
当前只允许：
- inspired 原创体素方块风
- 原创角色
- 原创体素工作台场景
- 原创体素面板语言

当前明确禁止：
- 直接复用官方 `logo`
- 直接复用官方 `字体`
- 直接复用官方 `textures`
- 直接复用官方 `images`
- 直接复用官方 `models`
- 直接复用官方 `sounds`

## 8. 当前声音路线
当前默认声音路线只允许写成建议：
- 平静
- 轻陪伴
- 低压
- 像游戏向导
- 不像 AI 播报
- 不是幼稚夹子音
- 不是新闻播音

参数建议只允许写成：
- 语速略低于默认
- 句间停顿略长于默认
- 情绪强度低到中等

当前外部资料包桥接后的方向判断 `已确认` 为：
- 单模型（直接 API 调用 + 默认音色）默认不够用于可发布状态
- 主要瓶颈不在音色本身，而在：
  - 韵律
  - 停顿
  - 情绪弧线
  - 句间过渡
- 推荐主链路当前只能写成方向判断：
  - 文案韵律重写（Prompt Engineering）
  - 豆包语音合成 2.0（语音指令 + 可爱女生音色）
  - 后期轻处理（EQ + 压缩 + 去齿音）
- 低门槛备选路线当前只能写成方向判断：
  - Azure TTS `zh-CN-XiaoxiaoNeural`
  - `SSML` 精控
  - 后期轻处理
- 当前不需要默认再叠 `Voice Conversion`
- 当前需要默认增加：同批次音色一致性检查

当前工程顺序建议 `待验证` 为：
- 由于当前用户已有阿里 API，可先做“阿里优先验证”
- 这只代表：
  - 工程起手顺序建议
  - 不代表阿里已成为最终路线
  - 不代表 Perplexity 原报告已被推翻

必须明确：
- 这不代表最终 TTS 已选定
- 这不代表已经完成内容验证

## 9. 当前最常见失败模式
- `API 生成真人段` 太多，拖节奏
- `用户录制素材` 太碎，主体不稳
- `少量 PPT` 太重，重新滑回课件感
- 结构没有跟着文案走，变成硬拼接
- 把 `Prompt 引用尾卡` 做成第二个主结尾
- 把 inspired 外壳误写成官方资产复用
- 声音方向跑偏，变成夹子音、播音腔或新闻播音
- 主持娃娃像普通 NPC，看不出“主持壳”职责
- 动作过僵、过闹、过大，像在抢戏
- 口型、表情、头部、轻微 bounce、手势各做各的，不能一起服务文案
- 主持壳越权抢中段主体，让 `用户录制素材` 失去主证据位置

## 10. 一句话规则
**当前默认主线不是整条 PPT 旧路线，也不是 `真人实拍出镜` 默认主线，而是：`API 生成真人` 负责进入和判断，`用户录制素材` 负责主体推进和过程证据，`少量 PPT` 负责整理和总结，最终按 `云端剪辑` 理解正式组装方向；vNext 当前在这条主线外面统一加上 inspired 原创体素外壳、`可爱元素主持娃娃` 主持壳、`可爱女生向导音` 目标方向和一张 `Prompt 引用尾卡`，但这些都还不能偷写成已完成内容验证。**
